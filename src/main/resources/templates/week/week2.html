第2週: オブジェクト指向プログラミング

Javaのオブジェクト指向プログラミングの基礎から応用までを学び、実践的なコードの書き方を習得する週です。

週の目標と概要

この週の学習目標

  • クラスとオブジェクトの基本概念を理解し、適切に設計できるようになる
  • カプセル化、継承、ポリモーフィズムというオブジェクト指向の3大原則を習得する
  • 抽象クラスとインターフェースの違いと使い分けを理解する
  • 例外処理の仕組みとベストプラクティスを学ぶ
  • Java 17で導入された新機能の基礎を理解する

オブジェクト指向の主要概念

カプセル化

データ（属性）とその操作（メソッド）を一つのユニット内にまとめて、外部からのアクセスを制限する概念。

主な特徴:

  • データの隠蔽（private修飾子）
  • 公開インターフェース（public修飾子）
  • getter/setterによる制御されたアクセス
  • データの整合性・一貫性の維持

継承

既存のクラス（親クラス・スーパークラス）の特性を新しいクラス（子クラス・サブクラス）が引き継ぐ仕組み。

主な特徴:

  • コードの再利用性向上
  • is-a関係の実現
  • 階層的なクラス設計
  • メソッドのオーバーライド
  • superキーワードの活用

ポリモーフィズム

同じインターフェースを持つ異なるオブジェクトが、それぞれ独自の方法で振る舞うことができる性質。

主な特徴:

  • 共通インターフェースによる抽象化
  • サブクラスによる固有実装
  • メソッドのオーバーライドと動的バインディング
  • 拡張性・柔軟性の向上
  • コード結合度の低減

抽象化

複雑な現実を単純化し、必要な本質的な特性だけを抽出する概念。

主な特徴:

  • 抽象クラス（abstract class）
  • インターフェース（interface）
  • 実装と使用の分離
  • API設計の基盤
  • 複雑性の隠蔽と管理

週末課題プロジェクト

シンプル図書管理システム

この週の学んだオブジェクト指向の概念を活かして、シンプルな図書管理システムを実装してください。

要件:

  1. 少なくとも以下のクラスを含める: Book, User, Library
  2. 継承を使って、Bookの派生クラス（例: FictionBook, NonFictionBook）を作成する
  3. 適切なカプセル化（アクセス修飾子、getter/setter）を実装する
  4. ポリモーフィズムを活用したメソッドを作成する
  5. 例外処理を適切に組み込む（例: 存在しない本の貸出処理など）
  6. 少なくとも1つのインターフェースを定義し実装する
  7. Java 17の新機能（レコードクラスなど）を活用する

このプロジェクトは、次週のはじめに提出し、コードレビューを行います。質問がある場合は、質疑応答の時間やオンラインフォーラムで質問してください。
