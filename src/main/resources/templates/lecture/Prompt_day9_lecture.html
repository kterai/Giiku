✅ 改善後の指針

計算量の説明強化

  • • O(1) → 「一定時間（高速）」
  • • O(n) → 「要素数に比例（要素1000個なら最大1000回処理）」
  • • O(log n) → 「対数時間（要素1000個なら約10回処理）」

実用例の追加

  • • ArrayList: 「顧客リスト、商品カタログ（ID検索が頻繁）」
  • • LinkedList: 「待ち行列、undo履歴（先頭・末尾操作が頻繁）」
  • • HashSet: 「重複チェック、タグ管理（存在確認が重要）」
  • • TreeSet: 「ランキング、ソート済み一覧（順序維持が重要）」
