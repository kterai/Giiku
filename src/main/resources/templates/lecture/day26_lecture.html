1. DOM操作の基礎

1.1 DOMとは？

DOM（Document Object Model）とは、HTML文書をJavaScriptから操作するためのプログラミングインターフェースです。 まるで家の設計図のように、Webページの構造を階層的に表現しています。

🏠 家との比較で理解するDOM

  • • 家全体 → document（HTML文書全体）
  • • 部屋 → div、section（コンテナ要素）
  • • 家具 → p、h1、img（コンテンツ要素）
  • • 装飾品 → class、style（属性）

1.2 要素の取得方法

// IDで要素を取得（最も確実）
const titleElement = document.getElementById('page-title');

// クラス名で要素を取得（複数の場合は配列的に扱う）
const buttons = document.getElementsByClassName('btn');

// タグ名で要素を取得
const paragraphs = document.getElementsByTagName('p');

// CSS セレクターで取得（最も柔軟）
const firstButton = document.querySelector('.btn');
const allButtons = document.querySelectorAll('.btn');

実務のポイント：現代の開発ではquerySelectorとquerySelectorAllが主流です。CSSセレクターの知識がそのまま使えるためです。

1.3 要素の操作

📝 テキスト・HTML操作

// テキストのみ変更
element.textContent = '新しいテキスト';

// HTMLも含めて変更
element.innerHTML = '<strong>太字</strong>';

// 属性の操作
element.setAttribute('class', 'new-class');
element.getAttribute('id');
element.removeAttribute('data-old');

🎨 スタイル操作

// 直接スタイル指定
element.style.color = 'red';
element.style.backgroundColor = 'yellow';

// クラスの操作（推奨）
element.classList.add('active');
element.classList.remove('inactive');
element.classList.toggle('hidden');
element.classList.contains('visible');

2. イベント処理の実装

2.1 イベントとは？

イベントとは、ユーザーの操作（クリック、キー入力など）やブラウザの状態変化（ページ読み込み完了など）を表す「合図」のようなものです。 まるでドアベルのように、「何かが起こったよ！」と教えてくれます。

🖱️ マウスイベント

  • • click（クリック）
  • • mouseover（マウス乗せ）
  • • mouseout（マウス離れ）
  • • mousedown/mouseup

⌨️ キーボードイベント

  • • keydown（キー押下）
  • • keyup（キー離し）
  • • keypress（文字入力）

📝 フォームイベント

  • • submit（送信）
  • • change（値変更）
  • • input（入力中）
  • • focus/blur（フォーカス）

2.2 イベントリスナーの設定

// 基本的なイベントリスナーの設定
button.addEventListener('click', function() {
    alert('ボタンがクリックされました！');
});

// アロー関数を使った記述（現代的）
button.addEventListener('click', () => {
    console.log('ボタンクリック');
});

// 名前付き関数を使用（再利用可能）
function handleButtonClick(event) {
    console.log('クリックされた要素:', event.target);
    console.log('イベントの種類:', event.type);
}
button.addEventListener('click', handleButtonClick);

ベストプラクティス：複数の要素に同じ処理を適用する場合は、名前付き関数を使用して再利用性を高めましょう。

3. 動的表示切替の実装

3.1 表示・非表示の基本パターン

🔄 シンプルな切り替え

function toggleVisibility(elementId) {
    const element = document.getElementById(elementId);
    
    if (element.style.display === 'none') {
        element.style.display = 'block';
    } else {
        element.style.display = 'none';
    }
}

✨ CSSクラスを使用（推奨）

// CSS: .hidden { display: none; }
function toggleVisibility(elementId) {
    const element = document.getElementById(elementId);
    element.classList.toggle('hidden');
}

推奨方法：CSSクラスを使用する方法が保守性・パフォーマンス・デザインの一貫性の観点から優れています。

3.2 アニメーション付き表示切替

// CSS トランジション用のクラス
// .fade-in { opacity: 1; transition: opacity 0.3s ease; }
// .fade-out { opacity: 0; transition: opacity 0.3s ease; }

function fadeToggle(elementId) {
    const element = document.getElementById(elementId);
    
    if (element.classList.contains('fade-out')) {
        element.classList.remove('fade-out');
        element.classList.add('fade-in');
        element.style.display = 'block';
    } else {
        element.classList.remove('fade-in');
        element.classList.add('fade-out');
        
        // アニメーション完了後に非表示
        setTimeout(() => {
            if (element.classList.contains('fade-out')) {
                element.style.display = 'none';
            }
        }, 300);
    }
}

解説： CSSセレクターを使用できるため、ID、クラス、属性など様々な条件で要素を取得できます。 
function toggleAnswer(questionId) {

4. フォーム制御とバリデーション

4.1 リアルタイムバリデーション

📧 実践例：メール入力フォーム

const emailInput = document.getElementById('email-input');
const validationDiv = document.getElementById('email-validation');

emailInput.addEventListener('input', (event) => {
    const email = event.target.value;
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    
    if (email === '') {
        validationDiv.textContent = '';
        emailInput.classList.remove('border-red-500', 'border-green-500');
    } else if (emailRegex.test(email)) {
        validationDiv.textContent = '✅ 有効なメールアドレスです';
        validationDiv.className = 'text-sm text-green-600';
        emailInput.classList.remove('border-red-500');
        emailInput.classList.add('border-green-500');
    } else {
        validationDiv.textContent = '❌ 正しいメールアドレスを入力してください';
        validationDiv.className = 'text-sm text-red-600';
        emailInput.classList.remove('border-green-500');
        emailInput.classList.add('border-red-500');
    }
});

4.2 動的フォーム要素の制御

// セレクトボックスの連動制御
function updateCityOptions(prefecture) {
    const citySelect = document.getElementById('city-select');
    const cities = {
        '東京都': ['新宿区', '渋谷区', '港区', '中央区'],
        '大阪府': ['大阪市', '堺市', '高槻市', '枚方市'],
        '愛知県': ['名古屋市', '豊田市', '岡崎市', '一宮市']
    };
    
    // 既存のオプションをクリア
    citySelect.innerHTML = '<option value="">市区町村を選択</option>';
    
    // 新しいオプションを追加
    if (cities[prefecture]) {
        cities[prefecture].forEach(city => {
            const option = document.createElement('option');
            option.value = city;
            option.textContent = city;
            citySelect.appendChild(option);
        });
        citySelect.disabled = false;
    } else {
        citySelect.disabled = true;
    }
}

理解度チェック

問題1: DOM操作の基本

ID が "myButton" の要素を取得し、クリックイベントリスナーを設定する正しいコードは？

正解: B

解説: document.getElementById()で要素を取得し、addEventListener()でイベントリスナーを設定するのが標準的な方法です。この方法により、複数のイベントリスナーを同じ要素に設定することも可能です。

問題2: 要素の表示切替

要素の表示・非表示を切り替える最も推奨される方法は？

正解: B

解説: CSSクラスを使用する方法が保守性、再利用性、パフォーマンスの面で優れています。スタイルとロジックの分離も実現でき、アニメーション効果も簡単に追加できます。

問題3: イベントオブジェクト

イベントハンドラー内で、イベントが発生した要素を取得するプロパティは？

正解: B

解説: event.targetは、イベントが実際に発生した要素を表しています。event.currentTargetは、イベントリスナーが設定された要素を表すため、状況によって使い分けが必要です。

実践演習問題

💡 演習問題1: ToDo リスト機能の実装

以下の要件を満たすシンプルなToDoリスト機能を実装してください：

  • テキスト入力フィールドと「追加」ボタン
  • 入力されたタスクをリストに動的に追加
  • 各タスクに「完了」ボタンを設置
  • 完了ボタンクリックで取り消し線を表示
  • 空の入力の場合はアラート表示

解答例:

// HTML
<div>
    <input type="text" id="taskInput" placeholder="新しいタスクを入力">
    <button onclick="addTask()">追加</button>
    <ul id="taskList"></ul>
</div>

// JavaScript
function addTask() {
    const input = document.getElementById('taskInput');
    const taskText = input.value.trim();
    
    if (taskText === '') {
        alert('タスクを入力してください');
        return;
    }
    
    const taskList = document.getElementById('taskList');
    const listItem = document.createElement('li');
    listItem.innerHTML = `
        <span>${taskText}</span>
        <button onclick="toggleComplete(this)" class="ml-2 bg-blue-500 text-white px-2 py-1 rounded">完了</button>
    `;
    
    taskList.appendChild(listItem);
    input.value = '';
}

function toggleComplete(button) {
    const taskSpan = button.previousElementSibling;
    if (taskSpan.style.textDecoration === 'line-through') {
        taskSpan.style.textDecoration = 'none';
        button.textContent = '完了';
    } else {
        taskSpan.style.textDecoration = 'line-through';
        button.textContent = '取消';
    }
}

ポイント: createElement()とinnerHTMLを使い分け、ユーザビリティを考慮した入力検証を実装。実際の開発では、配列でデータを管理し、状態管理を行うことが重要です。

解答例（抜粋）:

// CSS
.modal {
    display: none;
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    background: rgba(0,0,0,0.8);
    z-index: 1000;
    opacity: 0;
    transition: opacity 0.3s ease;
}
.modal.show {
    display: flex;
    opacity: 1;
}

// JavaScript
function openModal(imageSrc) {
    const modal = document.getElementById('imageModal');
    const modalImg = document.getElementById('modalImage');
    
    modalImg.src = imageSrc;
    modal.classList.add('show');
    
    // ESCキーで閉じる
    document.addEventListener('keydown', closeOnEscape);
}

function closeModal() {
    const modal = document.getElementById('imageModal');
    modal.classList.remove('show');
    document.removeEventListener('keydown', closeOnEscape);
}

function closeOnEscape(event) {
    if (event.key === 'Escape') {
        closeModal();
    }
}

// 背景クリックで閉じる
document.getElementById('imageModal').addEventListener('click', (event) => {
    if (event.target === event.currentTarget) {
        closeModal();
    }
});

ポイント: CSSトランジションとJavaScriptの組み合わせでスムーズなアニメーション。イベントリスナーの適切な追加・削除でメモリリークを防止。
