サンプル問題と解説

問題1: Stream活用による条件抽出

以下の要件を満たすStreamを用いた処理を実装してください：

学生リストから「年齢が20歳以上」かつ「成績がB以上」の学生の名前を取得し、アルファベット順でソートして出力

解答例：
students.stream()
    .filter(student -> student.getAge() >= 20)
    .filter(student -> student.getGrade().compareTo(Grade.B) >= 0)
    .map(Student::getName)
    .sorted()
    .collect(Collectors.toList());
解説：
  • • filter()で2つの条件を順次適用
  • • map()で学生オブジェクトから名前を抽出
  • • sorted()でアルファベット順にソート
  • • collect()でリストとして収集

問題2: ジェネリクスを使ったユーティリティクラス設計

型安全性を確保したペア（2つの値を保持）クラスを設計してください：

解答例：
public class Pair<T, U> {
    private final T first;
    private final U second;
    
    public Pair(T first, U second) {
        this.first = first;
        this.second = second;
    }
    
    public T getFirst() { return first; }
    public U getSecond() { return second; }
    
    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}
注意点：
  • • 2つの異なる型パラメータT, Uを使用
  • • finalフィールドでイミュータブル設計
  • • toStringメソッドでデバッグしやすく

総合演習問題

総合演習: 社員管理システム

以下の要件を満たす社員管理システムを実装してください：

要件：
  • 1. Employeeクラス（ID、名前、部署、給与を持つ）
  • 2. 部署ごとに社員をグループ化
  • 3. 各部署の平均給与を計算
  • 4. 給与が平均以上の社員のみを抽出
  • 5. 結果を部署名でソートして表示
解答例：
// Employeeクラス
public class Employee {
    private final int id;
    private final String name;
    private final String department;
    private final double salary;
    
    // コンストラクタとgetterメソッド省略
}

// メイン処理
Map<String, Double> avgSalaryByDept = employees.stream()
    .collect(Collectors.groupingBy(
        Employee::getDepartment,
        Collectors.averagingDouble(Employee::getSalary)
    ));

Map<String, List<Employee>> result = employees.stream()
    .filter(emp -> emp.getSalary() >= avgSalaryByDept.get(emp.getDepartment()))
    .collect(Collectors.groupingBy(Employee::getDepartment))
    .entrySet().stream()
    .sorted(Map.Entry.comparingByKey())
    .collect(Collectors.toMap(
        Map.Entry::getKey,
        Map.Entry::getValue,
        (e1, e2) -> e1,
        LinkedHashMap::new
    ));
解説：
  • • groupingBy()で部署ごとのグループ化
  • • averagingDouble()で平均給与計算
  • • filter()で条件に合う社員を抽出
  • • LinkedHashMapでソート順序を保持

模擬テスト③

制限時間: 120分（本試験と同じ形式）

問題1: コレクションの性能特性

ArrayList と LinkedList の挿入・削除性能に関する説明で正しいものはどれですか？

正解: B

解説:

LinkedList はリンク構造のため、先頭や末尾への挿入・削除は O(1) で高速です。一方、ArrayList は末尾への挿入は O(1) ですが、途中への挿入は要素のシフトが必要で O(n) かかります。ランダムアクセスは ArrayList が O(1)、LinkedList が O(n) です。

問題2: ジェネリクスの境界型

<T extends Number> が示すものはどれですか？

正解: A

解説:

<T extends Number> は上限境界を表し、T は Number クラスまたはその子クラス（Integer、Double など）に制限されます。これにより Number クラスのメソッドを安全に使用できます。

問題3: ラムダ式と外部変数

ラムダ式で外部変数を参照する際の制約はどれですか？

正解: A

解説:

ラムダ式内で外部変数を参照する場合、その変数は final または実質的final（一度だけ代入され、その後変更されない）でなければなりません。これは変数キャプチャの仕組みによるものです。

問題4: Stream の操作

Stream の filter と map を組み合わせた処理の目的は何ですか？

正解: C

解説:

filter() で条件に合う要素を絞り込み、その後 map() で要素を別の形に変換するのが一般的なパターンです。例：学生リストから成績優秀者を抽出し、名前だけを取得する処理など。

問題5: コードの実行結果

以下のコードの出力は何ですか？

Stream.of(1,2,3,4,5)
      .filter(n -> n % 2 == 0)
      .count();

正解: B

解説:

1,2,3,4,5 の中で偶数（n % 2 == 0）は 2 と 4 の2つです。filter() で偶数のみが残り、count() でその個数を返すため、結果は 2 になります。
