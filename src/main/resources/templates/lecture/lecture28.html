1. 暗号化技術

1.1 暗号化の基本概念

暗号化とは

平文（Plaintext）を暗号文（Ciphertext）に変換し、データを保護する技術です。機密性（Confidentiality）を確保する重要な手段です。

基本式:

暗号文 = 暗号化(平文, 鍵)

平文 = 復号化(暗号文, 鍵)

暗号化の目的

  • 機密性: 情報の秘匿
  • 完全性: データの改ざん検出
  • 認証性: 送信者の確認
  • 否認防止: 送信の否認を防止

1.2 暗号方式の分類

分類 共通鍵暗号 公開鍵暗号
鍵の種類 暗号化・復号化に同じ鍵を使用 公開鍵（暗号化）と秘密鍵（復号化）
処理速度 高速 低速
鍵配送問題 あり なし
代表例 AES, DES, 3DES RSA, ECC, DSA
主な用途 大容量データの暗号化 鍵交換、デジタル署名

講師向け指導ポイント

実際のシステムでは「ハイブリッド暗号」を使用することが多い。公開鍵暗号で共通鍵を配送し、共通鍵暗号でデータを暗号化する方式を具体例で説明しましょう。

1.3 主要な暗号アルゴリズム

AES (Advanced Encryption Standard)

  • 鍵長: 128/192/256bit
  • ブロックサイズ: 128bit
  • 特徴: 現在の標準暗号
  • 用途: Wi-Fi、VPN、ファイル暗号化

RSA

  • 鍵長: 1024/2048/4096bit
  • 基盤: 素因数分解問題
  • 特徴: 公開鍵暗号の代表
  • 用途: デジタル署名、SSL/TLS

SHA-256

  • ハッシュ長: 256bit
  • 種類: ハッシュ関数
  • 特徴: 一方向性、改ざん検出
  • 用途: パスワード保存、ブロックチェーン

2. 認証技術

2.1 認証の3要素

知識認証 (Something You Know)

パスワード、PIN、秘密の質問

最も一般的

所有認証 (Something You Have)

ICカード、トークン、スマートフォン

物理的証明

生体認証 (Something You Are)

指紋、虹彩、顔、音声

高セキュリティ

多要素認証 (MFA/2FA)

複数の認証要素を組み合わせることで、セキュリティを大幅に向上させる手法。例：パスワード + SMS認証、生体認証 + ICカード

2.2 認証プロトコル

プロトコル 特徴 使用場面 セキュリティレベル
Basic認証 Base64エンコードのみ 簡易なWebサイト 低
Digest認証 ハッシュ化してパスワード送信 HTTPアクセス制御 中
Kerberos チケットベース認証 Active Directory 高
OAuth 2.0 認可フレームワーク API認証、SNS連携 高
SAML XMLベースSSO 企業間連携 高

3. セキュリティ脆弱性

3.1 OWASP Top 10 (2021)

A01: Broken Access Control (アクセス制御の不備)

適切な権限制御が行われていない脆弱性。他人のデータにアクセス可能。

対策: 最小権限の原則、認可チェックの実装

A02: Cryptographic Failures (暗号化の失敗)

弱い暗号化や暗号化されていないデータの送信。

対策: 強力な暗号化、HTTPS通信の徹底

A03: Injection (インジェクション)

SQLインジェクション、NoSQLインジェクション、コマンドインジェクション。

対策: パラメータ化クエリ、入力値検証

A04: Insecure Design (安全でない設計)

設計段階でのセキュリティ考慮不足。

対策: セキュアデザイン、脅威モデリング

A05: Security Misconfiguration (セキュリティ設定ミス)

デフォルト設定の使用、不適切な権限設定。

対策: 設定レビュー、定期的な監査

4.2 ネットワーク攻撃

攻撃手法 概要 影響 対策
DDoS攻撃 大量のリクエストでサーバーを過負荷状態に サービス停止 負荷分散、WAF導入

5. 演習問題

演習問題1: 暗号方式の選択

以下のシナリオで最適な暗号方式を選択し、理由を説明してください：

  1. 10GBの動画ファイルを暗号化して保存する場合
  2. WebサイトでのSSL/TLS通信における鍵交換
  3. パスワードをデータベースに保存する場合
  4. デジタル署名を実装する場合

解答例:

  1. AES（共通鍵暗号） - 大容量データの高速暗号化に適している
  2. RSA（公開鍵暗号） - 鍵配送問題を解決し、共通鍵を安全に交換
  3. SHA-256 + Salt（ハッシュ関数） - 一方向性でパスワード復元を防ぐ
  4. RSA または ECDSA（公開鍵暗号） - 否認防止機能を提供

講師向けポイント: 実際のシステムでは複数の暗号技術を組み合わせて使用することを強調しましょう。

演習問題2: 脆弱性対策

以下のコードの脆弱性を特定し、適切な対策を提案してください：

// Java Servlet例
String userId = request.getParameter("userId");
String query = "SELECT * FROM users WHERE id = " + userId;
Statement stmt = connection.createStatement();
ResultSet rs = stmt.executeQuery(query);

脆弱性と対策:

脆弱性: SQLインジェクション

ユーザー入力を直接SQL文に結合しているため、悪意のあるSQL文を実行される可能性。

対策コード:
// PreparedStatementを使用
String query = "SELECT * FROM users WHERE id = ?";
PreparedStatement pstmt = connection.prepareStatement(query);
pstmt.setString(1, userId);
ResultSet rs = pstmt.executeQuery();
追加対策:
  • 入力値検証（数値チェック、長さ制限）
  • 最小権限でのDB接続
  • エラーメッセージの適切な制御
