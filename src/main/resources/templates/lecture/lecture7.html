1. アクセス修飾子（Access Modifiers）

🔐 アクセス修飾子とは

アクセス修飾子（アクセス制御子）は、クラスのメンバー（フィールドやメソッド）に対する 「誰がアクセスできるか」を制御する仕組みです。 これは金庫の鍵のように、大切な情報を適切に保護するために使用されます。

📊 アクセス修飾子詳細比較表

修飾子 アクセス範囲 具体的な意味 実用例 使用場面
private 同一クラス内のみ 金庫の中身のように、外部から直接見ることはできない 銀行口座の残高フィールド、パスワード 機密データ、内部処理用メソッド
（デフォルト） 同一パッケージ内 同じ部署の人だけがアクセス可能 パッケージ内の共通ユーティリティ パッケージ内での共有
protected 継承クラス + 同一パッケージ 家族（継承関係）には公開される家族の秘密 親クラスの共通メソッド、テンプレート 継承での拡張を想定
public どこからでも 図書館の本のように、誰でも利用可能 API公開メソッド、getter/setter 外部に公開するインターフェース

💻 実践例：銀行口座クラス

public class BankAccount {
    // private: 口座残高は外部から直接変更できない（金庫の中身）
    private double balance;
    
    // private: 口座番号も外部から変更不可
    private String accountNumber;
    
    // protected: 継承クラス（特別口座など）からは利用可能
    protected String accountType;
    
    // public: 外部からアクセス可能なメソッド（窓口業務）
    public double getBalance() {
        return balance;  // 残高照会は可能
    }
    
    // public: 入金メソッド（安全な操作のみ公開）
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;  // 内部でのみ残高変更
        }
    }
    
    // private: 内部処理用（外部からは呼び出せない）
    private boolean validateTransaction(double amount) {
        return amount > 0 && balance >= amount;
    }
}

👨‍🏫 講師向け指導ポイント

  • なぜprivateにするのか：「残高を直接変更されたら困るから」という理由を強調
  • 比喩の活用：「金庫」「家族の秘密」「図書館」など身近な例で説明
  • 段階的理解：まずprivateとpublicから始め、徐々にprotectedを追加
  • 実習重視：実際にアクセス制限を破ろうとしてエラーを体験させる

2. コンストラクタと継承の詳細

🏗️ コンストラクタ呼び出し順序

継承関係にあるクラスでは、コンストラクタは「親クラス → 子クラス」の順序で呼び出されます。 これは建物を建てる時に「基礎 → 1階 → 2階」の順序で作るのと同じです。

📈 コンストラクタ呼び出し順序図

①
Vehicleクラス
（親クラス）
②
Carクラス
（子クラス）
③
SportsCarクラス
（孫クラス）

実行順序の詳細：

  1. SportsCarのコンストラクタが呼ばれる
  2. 最初にsuper()でCarのコンストラクタを呼び出す
  3. Carのコンストラクタ内でsuper()によりVehicleのコンストラクタを呼び出す
  4. Vehicleのコンストラクタが実行される
  5. Carのコンストラクタが実行される
  6. 最後にSportsCarのコンストラクタが実行される

💻 実践例：Vehicle継承チェーン

// ① 親クラス
class Vehicle {
    protected String brand;
    
    public Vehicle(String brand) {
        this.brand = brand;
        System.out.println("①Vehicle コンストラクタ実行: " + brand);
    }
}

// ② 子クラス
class Car extends Vehicle {
    protected int doors;
    
    public Car(String brand, int doors) {
        super(brand);  // ← 必ず最初の行で親のコンストラクタを呼び出し
        this.doors = doors;
        System.out.println("②Car コンストラクタ実行: " + doors + "ドア");
    }
}

// ③ 孫クラス
class SportsCar extends Car {
    private boolean turbo;
    
    public SportsCar(String brand, boolean turbo) {
        super(brand, 2);  // ← スポーツカーは2ドア固定
        this.turbo = turbo;
        System.out.println("③SportsCar コンストラクタ実行: ターボ" + turbo);
    }
}

// 実行例
SportsCar myCar = new SportsCar("Ferrari", true);
// 出力:
// ①Vehicle コンストラクタ実行: Ferrari
// ②Car コンストラクタ実行: 2ドア  
// ③SportsCar コンストラクタ実行: ターボtrue

⚠️ super()の重要なルール

  • 1. 最初の行で呼び出し：super()は必ずコンストラクタの最初の行に記述
  • 2. 自動挿入：super()を書かない場合、コンパイラが自動的にsuper()を挿入
  • 3. 引数つきsuper()：親クラスに引数なしコンストラクタがない場合は必須
  • 4. this()との併用不可：this()とsuper()は同時に使用できない

3. thisキーワードの詳細活用

🎯 thisキーワードとは

thisキーワードは「現在のオブジェクト自身」を指す参照です。 鏡を見て自分を指差すように、オブジェクトが自分自身を参照するために使用されます。

📋 thisキーワード使用場面詳細表

使用場面 記述方法 理由・目的 実用例
フィールドと引数の区別 this.name = name; 同名のフィールドと引数を区別するため setter メソッド、コンストラクタ
他のコンストラクタ呼び出し this(defaultValue); 同一クラス内の別のコンストラクタを呼び出し オーバーロードされたコンストラクタ
メソッドチェーン return this; 連続したメソッド呼び出しを可能にする Builder パターン、流れるような API
自分自身の参照渡し callback.notify(this); 他のオブジェクトに自分自身を渡す イベント処理、コールバック

💻 実践例：thisキーワードの活用

public class Student {
    private String name;
    private int age;
    private String grade;
    
    // ① フィールドと引数の区別
    public Student(String name, int age) {
        this.name = name;  // this.name（フィールド）= name（引数）
        this.age = age;    // this.age（フィールド）= age（引数）
    }
    
    // ② 他のコンストラクタ呼び出し
    public Student(String name) {
        this(name, 0);  // 上記のコンストラクタを呼び出し
    }
    
    // ③ setter でのthis使用
    public Student setGrade(String grade) {
        this.grade = grade;
        return this;  // 自分自身を返す（メソッドチェーン）
    }
    
    public Student setAge(int age) {
        this.age = age;
        return this;  // 自分自身を返す
    }
    
    // ④ メソッドチェーンの利用例
    public static void main(String[] args) {
        Student student = new Student("田中太郎")
            .setAge(20)      // thisが返されるので続けて呼び出し可能
            .setGrade("A");  // さらに続けて呼び出し可能
        
        System.out.println(student.name + " (" + student.age + "歳) 成績:" + student.grade);
    }
}

⚠️ よくある間違いと対策

❌ 間違い：this を付け忘れ
public void setName(String name) {
    name = name;  // ← これは引数 = 引数 になってしまう
}
✅ 正解：this で明示的に指定
public void setName(String name) {
    this.name = name;  // ← フィールド = 引数
}

4. カプセル化（Encapsulation）の実践

🏆 カプセル化とは

カプセル化（データ隠蔽）は「データと処理を一体化し、外部から直接アクセスできないよう隠蔽する仕組み」です。 薬のカプセルのように、中身（データ）を保護し、安全な方法（メソッド）でのみアクセスを許可します。

📊 カプセル化のメリット詳細表

メリット 説明 具体例 実務での効果
データ保護 不正な値の設定を防ぐ 年齢に負の値を設定することを防ぐ バグの早期発見、データ整合性確保
内部実装の隠蔽 内部の処理方法を変更しても外部に影響しない 配列からListに実装を変更 メンテナンス性向上、影響範囲限定
アクセス制御 必要な機能のみを公開 残高照会は可能、直接変更は不可 セキュリティ向上、誤用防止
検証・ログ 値の設定時に検証やログ出力が可能 パスワード設定時の強度チェック 品質向上、トラブルシューティング

💻 実践例：完全なカプセル化の実装

public class Employee {
    // private フィールド（外部から直接アクセス不可）
    private String name;
    private int age;
    private double salary;
    private String department;
    
    // コンストラクタ
    public Employee(String name, int age, double salary, String department) {
        setName(name);        // setter を使って検証
        setAge(age);          // setter を使って検証
        setSalary(salary);    // setter を使って検証
        this.department = department;
    }
    
    // getter: 読み取り専用アクセス
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
    
    public double getSalary() {
        // 機密情報は制限付きで公開
        return salary * 0.8;  // 手取り概算を返す
    }
    
    // setter: 検証付きの安全な設定
    public void setName(String name) {
        if (name == null || name.trim().isEmpty()) {
            throw new IllegalArgumentException("名前は必須です");
        }
        this.name = name.trim();
    }
    
    public void setAge(int age) {
        if (age < 0 || age > 100) {
            throw new IllegalArgumentException("年齢は0-100の範囲で設定してください");
        }
        this.age = age;
    }
    
    public void setSalary(double salary) {
        if (salary < 0) {
            throw new IllegalArgumentException("給与は0以上である必要があります");
        }
        // ログ出力（実際の変更を記録）
        System.out.println("給与変更: " + this.salary + " → " + salary);
        this.salary = salary;
    }
    
    // 業務メソッド（計算ロジックも隠蔽）
    public double calculateAnnualSalary() {
        return salary * 12;  // 内部実装の詳細は隠蔽
    }
    
    public boolean isEligibleForPromotion() {
        return age >= 25 && salary >= 300000;  // 昇進条件も内部で判定
    }
}

📝 演習問題

問題1: アクセス修飾子の選択

以下のBookクラスの各メンバーに適切なアクセス修飾子を選択し、その理由を説明してください。 

class Book {
    _____ String title;           // 書籍タイトル
    _____ String isbn;            // ISBN番号
    _____ double price;           // 価格
    _____ int publishYear;        // 出版年
    _____ boolean isAvailable;    // 貸出可能フラグ
    
    _____ String getTitle() { return title; }
    _____ void setPrice(double price) { this.price = price; }
    _____ void validateISBN(String isbn) { /* 検証処理 */ }
    _____ void updateAvailability() { /* 内部処理 */ }
}

💡 回答例

class Book {
    private String title;           // 外部から直接変更されたくない
    private String isbn;            // 一度設定したら変更不可にすべき
    private double price;           // 価格は慎重に管理すべき
    protected int publishYear;      // 継承クラス（特別版など）で利用
    private boolean isAvailable;    // 内部状態は隠蔽
    
    public String getTitle() { return title; }        // 外部からの読み取りは許可
    public void setPrice(double price) { this.price = price; }  // 公開API
    private void validateISBN(String isbn) { /* 検証処理 */ }   // 内部処理
    private void updateAvailability() { /* 内部処理 */ }        // 内部処理
}
選択理由：
  • private: title, isbn, price, isAvailable - 重要なデータは保護
  • protected: publishYear - 継承クラスでの利用を想定
  • public: getter/setter - 外部との公開インターフェース
  • private: validate系メソッド - 内部実装の詳細

問題2: コンストラクタ呼び出し順序

以下のコードの実行結果を予測し、コンストラクタの呼び出し順序を説明してください。 

class Animal {
    public Animal() {
        System.out.println("Animal コンストラクタ");
    }
}

class Dog extends Animal {
    public Dog() {
        System.out.println("Dog コンストラクタ");
    }
    
    public Dog(String name) {
        this();  // 同一クラスの別のコンストラクタを呼び出し
        System.out.println("Dog 名前付きコンストラクタ: " + name);
    }
}

// 実行
Dog myDog = new Dog("ポチ");

💡 回答例

実行結果：
Animal コンストラクタ
Dog コンストラクタ
Dog 名前付きコンストラクタ: ポチ
呼び出し順序の詳細：
  1. Dog(String name) コンストラクタが呼ばれる
  2. this() により Dog() コンストラクタが呼ばれる
  3. Dog() 内で暗黙的に super() により Animal() が呼ばれる
  4. Animal コンストラクタが実行される
  5. Dog() コンストラクタが実行される
  6. 最後に Dog(String name) の残り部分が実行される

問題3: thisキーワードの活用

以下の要件を満たすPersonクラスを作成してください：

  • name, age, email の private フィールドを持つ
  • 全ての引数を受け取るコンストラクタと、nameのみを受け取るコンストラクタ（age=0, email=""でデフォルト設定）
  • メソッドチェーンで連続して値を設定できるsetterメソッド
  • 適切な検証機能付き

💡 回答例

public class Person {
    private String name;
    private int age;
    private String email;
    
    // 全引数コンストラクタ
    public Person(String name, int age, String email) {
        this.name = name;
        this.age = age;
        this.email = email;
    }
    
    // 名前のみコンストラクタ（thisで他のコンストラクタを呼び出し）
    public Person(String name) {
        this(name, 0, "");  // デフォルト値で全引数コンストラクタを呼び出し
    }
    
    // メソッドチェーン対応setter
    public Person setAge(int age) {
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("年齢は0-150の範囲で設定してください");
        }
        this.age = age;
        return this;  // 自分自身を返してメソッドチェーンを可能に
    }
    
    public Person setEmail(String email) {
        if (email != null && !email.contains("@")) {
            throw new IllegalArgumentException("有効なメールアドレスを入力してください");
        }
        this.email = email;
        return this;  // 自分自身を返す
    }
    
    // 利用例
    public static void main(String[] args) {
        Person person = new Person("田中太郎")
            .setAge(25)
            .setEmail("tanaka@example.com");
    }
}

🧠 理解度クイズ

クイズ1: アクセス修飾子

protectedフィールドにアクセスできるのはどれですか？（複数選択可）

正解：A、B、C

protectedは「同一クラス内」「同一パッケージ内」「継承クラス（パッケージが異なっても可）」からアクセス可能です。 全く関係のないクラス（継承関係もなく、パッケージも異なる）からはアクセスできません。

クイズ2: super()の呼び出し

子クラスのコンストラクタでsuper()を明示的に書かなかった場合、どうなりますか？

正解：C

子クラスのコンストラクタでsuper()を明示的に書かない場合、 コンパイラが自動的に引数なしのsuper()を最初の行に挿入します。 ただし、親クラスに引数なしのコンストラクタが存在しない場合はコンパイルエラーになります。

クイズ3: thisキーワード

以下のコードの実行結果はどうなりますか？ 

class Test {
    private String name = "クラス";
    
    public void method(String name) {
        System.out.println("引数: " + name);
        System.out.println("フィールド: " + this.name);
        System.out.println("ローカル: " + name);
    }
}

Test test = new Test();
test.method("メソッド");

実行結果：

引数: メソッド
フィールド: クラス
ローカル: メソッド

thisキーワードを使うことで、同名の引数（ローカル変数）とフィールドを明確に区別できます。 this.nameはクラスのフィールドを、nameは引数（ローカル変数）を指します。

📚 JavaSilver試験対策

頻出問題パターン

パターン1: アクセス修飾子の範囲

どのクラスからアクセス可能かを問う問題

package com.example;
class A {
    protected int value;
}

package com.other;
class B extends A {
    void method() {
        value = 10;  // これは可能？
    }
}

→ 可能（異なるパッケージでも継承関係があればOK）

パターン2: コンストラクタ呼び出し順序

継承チェーンでのコンストラクタ実行順序

class A { A() { System.out.print("A"); } }
class B extends A { B() { System.out.print("B"); } }
class C extends B { C() { System.out.print("C"); } }

new C();  // 出力は？

→ "ABC"（親から子の順）

試験攻略のコツ

✅ 重要ポイント

  • protectedは継承関係があれば異なるパッケージでもアクセス可能
  • super()は必ず最初の行、省略時は自動挿入
  • this()とsuper()は同時使用不可
  • コンストラクタは親→子の順序で実行

⚠️ 頻出の引っかけ

  • defaultアクセスとprotectedの違い
  • 継承関係のないクラスからのprotectedアクセス
  • this()呼び出し後のsuper()呼び出し
  • フィールドと引数の同名問題

📋 本日のまとめ

✅ 習得できたスキル

  • privateで隠蔽し、getterで安全にアクセスできるカプセル化の実装
  • protectedは継承クラスでのみ利用可能で、API設計に有効
  • super()で親コンストラクタを最初に呼び出す継承の仕組み
  • thisで同名フィールドと引数を区別でき、メソッドチェーンも実現

🚀 実務での活用

  • 機密データ（パスワード、個人情報）の適切な保護
  • APIの公開範囲制御によるセキュリティ向上
  • 継承を活用した保守性の高いクラス設計
  • メソッドチェーンによる流れるようなAPI設計
