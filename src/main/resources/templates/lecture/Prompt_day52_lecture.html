テスト手法の説明強化

  • Unit Test → 「単体テスト（個別メソッドの動作確認）」
  • Integration Test → 「統合テスト（複数コンポーネントの連携確認）」
  • E2E Test → 「エンドツーエンドテスト（全体の動作確認）」

具体例の追加

  • 「単体テスト」→「計算メソッドの正確性確認」
  • 「統合テスト」→「データベース連携処理の確認」
  • 「E2Eテスト」→「ユーザー登録から完了まで全工程確認」
