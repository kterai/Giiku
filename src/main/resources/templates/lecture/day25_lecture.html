Java基礎問題（問題1-8）

問題1: オブジェクト指向プログラミング

次のJavaコードについて、実行結果として正しいものはどれか。

class Animal {
    protected String name = "動物";
    public void speak() {
        System.out.println(name + "が鳴いています");
    }
}

class Dog extends Animal {
    protected String name = "犬";
    @Override
    public void speak() {
        System.out.println(name + "がワンワン鳴いています");
    }
}

public class Test {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.speak();
    }
}

正解: イ) 犬がワンワン鳴いています

解説:

  • ポリモーフィズムにより、実際のオブジェクト（Dog）のメソッドが呼ばれる
  • Dog クラスでオーバーライドされた speak() メソッドが実行される
  • Dog クラスの name フィールド（"犬"）が使用される
  • 結果: "犬がワンワン鳴いています"が出力される

重要ポイント: 動的バインディングにより、参照型ではなく実際のオブジェクト型のメソッドが実行される

問題2: コレクションフレームワーク

ArrayListとLinkedListの特徴について、正しい記述はどれか。

正解: イ) ArrayListはランダムアクセスが高速、LinkedListは要素の挿入・削除が高速

解説:

操作 ArrayList LinkedList
ランダムアクセス O(1) - 配列なので高速 O(n) - 順次アクセス必要
挿入・削除 O(n) - 要素移動が必要 O(1) - ポインタ変更のみ

実務での選択基準: 検索重視ならArrayList、挿入・削除重視ならLinkedList

アルゴリズム問題（問題9-12）

問題3: 計算量

次のアルゴリズムの時間計算量として正しいものはどれか。

public static int search(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;
        }
    }
    return -1;
}

正解: ウ) O(n)

解説:

  • これは線形探索（リニアサーチ）のアルゴリズム
  • 配列の先頭から順番に目的の値を探索する
  • 最悪の場合、配列の全要素（n個）を調べる必要がある
  • したがって時間計算量は O(n) となる

比較:

  • O(1): 配列の直接アクセス（arr[index]）
  • O(log n): 二分探索（事前にソート済み配列が必要）
  • O(n²): 二重ループ（例：バブルソート）

統合問題（問題13-20）

問題4: データベース正規化

次のテーブルを第3正規形まで正規化する場合、適切な分割方法はどれか。

社員番号 社員名 部署番号 部署名 給与
001 田中 10 営業部 300000
002 佐藤 20 開発部 350000

正解: イ) 社員テーブル（社員番号、社員名、部署番号、給与）、部署テーブル（部署番号、部署名）

解説:

  • 問題点: 部署番号 → 部署名の推移的関数従属性が存在
  • 第3正規形: 非キー属性間の推移的関数従属性を排除
  • 解決策: 部署情報を別テーブルに分離

正規化後:

社員テーブル: 社員番号（PK）、社員名、部署番号（FK）、給与

部署テーブル: 部署番号（PK）、部署名

メリット: 部署名変更時の更新異常を防止、データの重複排除

問題5: ネットワーク

IPアドレス 192.168.10.0/24 のネットワークについて、正しい記述はどれか。

正解: ア) ホスト部は8ビットで、最大254台のホストを接続できる

解説:

  • /24 は CIDR記法で、ネットワーク部が24ビット
  • IPv4アドレスは32ビットなので、ホスト部は 32-24 = 8ビット
  • 8ビットで表現可能な値: 2⁸ = 256通り
  • ネットワークアドレス（.0）とブロードキャストアドレス（.255）を除く
  • 実際に使用可能なホスト数: 256 - 2 = 254台

具体例:

ネットワークアドレス: 192.168.10.0

使用可能範囲: 192.168.10.1 ～ 192.168.10.254

ブロードキャストアドレス: 192.168.10.255

演習1: Java実装問題

以下の要件を満たすJavaプログラムを作成してください。

要件:

  • 従業員情報を管理するEmployeeクラス
  • フィールド: id（整数）、name（文字列）、salary（整数）
  • コンストラクタ、getter、setterメソッド
  • 給与に基づくComparableインターフェースの実装
  • toString()メソッドのオーバーライド
public class Employee implements Comparable<Employee> {
    private int id;
    private String name;
    private int salary;
    
    // コンストラクタ
    public Employee(int id, String name, int salary) {
        this.id = id;
        this.name = name;
        this.salary = salary;
    }
    
    // Getter メソッド
    public int getId() { return id; }
    public String getName() { return name; }
    public int getSalary() { return salary; }
    
    // Setter メソッド
    public void setId(int id) { this.id = id; }
    public void setName(String name) { this.name = name; }
    public void setSalary(int salary) { this.salary = salary; }
    
    // Comparable インターフェースの実装（給与昇順）
    @Override
    public int compareTo(Employee other) {
        return Integer.compare(this.salary, other.salary);
    }
    
    // toString メソッドのオーバーライド
    @Override
    public String toString() {
        return String.format("Employee{id=%d, name='%s', salary=%d}", 
                           id, name, salary);
    }
    
    // equals と hashCode のオーバーライド（推奨）
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Employee employee = (Employee) obj;
        return id == employee.id;
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}

ポイント:

  • Comparableインターフェースで自然順序付けを定義
  • equals/hashCodeも実装してHashSetやHashMapで正しく動作
  • toString()で可読性の高い文字列表現を提供
  • Integer.compare()でint型の安全比較を実現

演習2: アルゴリズム設計問題

効率的なデータ構造とアルゴリズムを選択してください。

シナリオ:

1万件の学生データ（学籍番号、氏名、成績）があり、以下の操作を頻繁に行う：

  • 学籍番号による学生情報の高速検索
  • 成績による学生の順序付け表示
  • 新しい学生データの追加・削除

問題: 最適なデータ構造を選択し、その理由を説明してください。

推奨解決策: HashMap + TreeSet の組み合わせ

1. 学籍番号による高速検索

データ構造: HashMap<String, Student>

理由: O(1)の高速検索が可能

key: 学籍番号, value: Studentオブジェクト

2. 成績による順序付け

データ構造: TreeSet<Student>

理由: 自動ソート機能（Comparable実装）

Student クラスでcompareTo()を成績基準で実装

3. 追加・削除操作

HashMap: O(1)で追加・削除

TreeSet: O(log n)で追加・削除

両方のデータ構造を同期して更新

実装のポイント:

  • Studentクラスでequals/hashCodeを学籍番号基準で実装
  • compareTo()メソッドを成績基準で実装
  • データ追加・削除時は両データ構造を同期更新
  • メモリ使用量は多いが、操作性能を重視した設計
