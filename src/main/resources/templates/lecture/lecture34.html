Day34: 基本情報技術者試験 模擬試験（午後）

午後試験本番形式・時間配分・解答戦略・総合力チェック

所要時間: 3時間

対象: 基本情報技術者試験受験者

本日の学習目標

知識目標

  • 午後試験の出題形式と時間配分を理解する
  • プログラミング問題の解答戦略を習得する
  • アルゴリズム問題のアプローチ方法を身につける
  • 情報セキュリティ問題の解法パターンを理解する

実践目標

  • 本番と同じ時間配分で模擬試験を完答する
  • 各問題の解答時間を効率的に管理できる
  • 部分点を意識した解答戦略を実践できる
  • 見直し時間を確保した時間管理ができる

午後試験概要

試験形式

  • 時間：150分（2時間30分）
  • 問題数：11問中5問選択
  • 配点：100点満点
  • 合格基準：60点以上

出題分野

  • 問1：情報セキュリティ（必須）
  • 問2-3：ハードウェア・ソフトウェア
  • 問4-5：データベース
  • 問6-7：ネットワーク
  • 問8-11：アルゴリズム・プログラミング

講師向け指導ポイント

午後試験は知識の暗記だけでなく、問題文を正確に読み解き、論理的に思考する能力が問われます。受講者には時間配分の重要性を強調し、部分点を意識した解答方法を指導してください。特に、プログラミング問題では完璧を目指さず、確実に点数を取れる部分から解答することの重要性を説明してください。

時間配分戦略

推奨時間配分（150分）

段階 時間 内容 ポイント
問題選択 10分 全問題を概観し、解答する5問を選択 得意分野を優先、難易度を見極める
問1（必須） 25分 情報セキュリティ問題 確実に点数を取る、部分点を意識
問2-7選択 80分 選択問題3問（各約25-30分） 得意な問題から順番に解答
プログラミング 25分 アルゴリズム問題1問 トレース実行、論理的思考
見直し 10分 解答の確認、記入漏れチェック ケアレスミスを防ぐ

時間管理のコツ

  • 25分ルール：1問あたり25分を目安とし、時間になったら次の問題へ
  • 部分点戦略：完璧を求めず、確実に点数が取れる部分を優先
  • 見直し時間確保：最後10分は必ず見直し時間として確保
  • 優先順位：得意分野・確実に解ける問題から取り組む

解答戦略

情報セキュリティ問題（問1）

  • 読解重視：問題文を丁寧に読み、状況を正確に把握
  • 用語確認：専門用語の定義を正確に理解
  • 対策手順：脅威→脆弱性→対策の流れで考える
  • 具体例：具体的な対策案を記述する

データベース問題

  • ER図理解：エンティティ間の関係を正確に把握
  • 正規化：段階的な正規化プロセスを理解
  • SQL記述：SELECT文の構文を正確に記述
  • 結合処理：内部結合・外部結合の使い分け

ネットワーク問題

  • プロトコル：各層のプロトコルの役割を理解
  • アドレス計算：IPアドレス・サブネットマスク計算
  • ルーティング：経路選択の仕組みを理解
  • セキュリティ：ファイアウォール・VPNの設定

アルゴリズム問題

  • 疑似コード読解：アルゴリズムの動作を正確に理解
  • トレース実行：変数の値の変化を追跡
  • 計算量：時間計算量・空間計算量を評価
  • 最適化：効率的なアルゴリズムへの改良案

プログラミング問題

  • 言語理解：Java・Python・Cの基本構文
  • 空欄補充：文脈から適切なコードを推測
  • デバッグ：エラーの原因と修正方法
  • テストケース：実行結果を予測・検証

共通戦略

  • 部分点狙い：完全でなくても部分的な解答を記述
  • 図表活用：問題の図表から情報を読み取る
  • 計算確認：数値計算は必ず検算する
  • 用語統一：問題文と同じ専門用語を使用

模擬試験開始

制限時間：150分（2時間30分）

以下の注意事項を確認してから開始してください：

  • • 問1（情報セキュリティ）は必須、問2-11から4問選択
  • • 時計を確認し、時間配分を意識する
  • • 解答は部分点を意識して記述する
  • • 見直し時間を10分確保する

問1：情報セキュリティ（必須・25分）

以下の状況を読んで、情報セキュリティの観点から設問に答えなさい。

【状況】

A社は従業員200名の中規模IT企業である。最近、在宅勤務の普及に伴い、社外からの業務システムアクセスが増加している。 現在、従業員は個人のPCから直接インターネット経由で社内システムにVPN接続を行っているが、 セキュリティ責任者のB氏は以下の課題を抱えている。

  • 従業員の個人PCのセキュリティ対策レベルがばらばらである
  • VPNの認証は従来のID・パスワードのみで、多要素認証が導入されていない
  • 社内システムへのアクセスログの監視体制が不十分である
  • 従業員へのセキュリティ教育が形式的で実効性に疑問がある

設問1-1

従業員の個人PCからのリモートアクセスに関するセキュリティリスクを3つ挙げ、 それぞれに対する具体的な対策を述べなさい。

解答例：
リスク1：マルウェア感染リスク

対策：エンドポイント保護ソフトウェアの導入、定期的なウイルス定義ファイル更新の自動化

リスク2：認証情報の漏洩リスク

対策：多要素認証（MFA）の導入、生体認証やワンタイムパスワードの活用

リスク3：データ漏洩リスク

対策：仮想デスクトップ環境（VDI）の構築、ローカルへのデータ保存禁止

設問1-2

多要素認証を導入する場合、「知識要素」「所持要素」「生体要素」の3つの要素のうち、 どの組み合わせが最も効果的か理由と共に述べなさい。

解答例：

推奨組み合わせ：知識要素 + 所持要素
理由：生体要素は高いセキュリティを提供するが、コストが高く、在宅勤務環境での導入は困難。 知識要素（パスワード）と所持要素（スマートフォンのワンタイムパスワード）の組み合わせが、 セキュリティと利便性のバランスが良く、導入コストも抑えられるため最も実用的である。

設問1-3

効果的なセキュリティ教育を実施するための方法を3つ挙げ、 それぞれの実施方法と期待される効果を説明しなさい。

解答例：
1. 標的型攻撃メール訓練

実施方法：模擬的な不審メールを送信し、クリック率を測定
期待効果：実践的な経験により、不審メールの識別能力向上

2. インシデント事例学習

実施方法：実際の情報漏洩事例を分析し、原因と対策を検討
期待効果：リスクの具体的理解と当事者意識の醸成

3. 定期的な理解度テスト

実施方法：四半期ごとにセキュリティ知識のオンラインテストを実施
期待効果：知識の定着確認と継続的な学習意識の維持

問8：アルゴリズム（選択・25分）

以下の疑似コードを読んで設問に答えなさい。

【アルゴリズム】二分探索

procedure BinarySearch(A[], n, key)
    left ← 0
    right ← n - 1
    
    while left ≤ right do
        mid ← (left + right) / 2  ※整数除算
        
        if A[mid] = key then
            return mid
        else if A[mid] > key then
            right ← mid - 1
        else
            left ← mid + 1
        end if
    end while
    
    return -1  ※見つからない場合
end procedure

設問8-1

配列A = [2, 5, 8, 12, 16, 23, 38, 45, 67, 78]（n=10）に対して、 key=23を探索する場合の実行過程を示しなさい。 各ステップでのleft, right, midの値と比較結果を記述すること。

解答：

ステップ1: left=0, right=9, mid=4, A[4]=16 < 23 → left=5

ステップ2: left=5, right=9, mid=7, A[7]=45 > 23 → right=6

ステップ3: left=5, right=6, mid=5, A[5]=23 = 23 → return 5

結果: インデックス5が返される

設問8-2

この二分探索アルゴリズムの時間計算量をO記法で表し、 線形探索O(n)と比較してその優位性を説明しなさい。

解答：

時間計算量： O(log n)
優位性： 二分探索は毎回探索範囲を半分に縮小するため、 最大でもlog₂n回の比較で結果が得られる。 例えば要素数1000の配列では、線形探索が最大1000回の比較が必要なのに対し、 二分探索では最大10回程度で済み、大幅に効率的である。 ただし、配列がソート済みである必要がある。

問4：データベース（選択・25分）

以下のテーブル構造を参考にして、SQL文を作成しなさい。

商品テーブル（Product）

列名 データ型
product_idINT
product_nameVARCHAR(50)
priceINT
category_idINT

カテゴリテーブル（Category）

列名 データ型
category_idINT
category_nameVARCHAR(30)

設問4-1

価格が1000円以上3000円以下の商品について、商品名、価格、カテゴリ名を 価格の昇順で表示するSQL文を作成しなさい。

解答：
SELECT p.product_name, p.price, c.category_name
FROM Product p
INNER JOIN Category c ON p.category_id = c.category_id
WHERE p.price BETWEEN 1000 AND 3000
ORDER BY p.price ASC;

設問4-2

各カテゴリの商品数と平均価格を求めるSQL文を作成しなさい。 カテゴリに商品が存在しない場合も0として表示すること。

解答：
SELECT c.category_name,
       COUNT(p.product_id) as product_count,
       COALESCE(AVG(p.price), 0) as avg_price
FROM Category c
LEFT JOIN Product p ON c.category_id = p.category_id
GROUP BY c.category_id, c.category_name
ORDER BY c.category_name;
