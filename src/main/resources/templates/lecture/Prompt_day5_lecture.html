✅ 改善後の指針

オブジェクト指向3大原則の説明強化

  • • カプセル化 → 「データと処理を一体化し、外部から直接触れないようにする（銀行のATMのように）」
  • • 継承 → 「親クラスの機能を子クラスが引き継ぐ（動物→犬→柴犬のような関係）」
  • • ポリモーフィズム → 「同じ命令でも、対象によって異なる動作をする（「鳴け」で犬は"ワン"、猫は"ニャー"）」

手続き型との比較強化

  • • データ管理: 手続き型「バラバラに管理」→ OOP「クラス内で一元管理」
  • • 再利用性: 手続き型「コピー&ペースト」→ OOP「継承で効率的」
  • • 保守性: 手続き型「影響範囲が不明」→ OOP「変更影響を局所化」
