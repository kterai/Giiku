1. プリミティブ型（基本データ型）の詳細解説

プリミティブ型とは

プリミティブ型（primitive type）とは、Javaが標準で提供する8つの基本データ型のことです。 箱に例えると、それぞれ異なるサイズ・用途に特化した専用の箱のようなものです。

プリミティブ型詳細比較表（改良版）

なぜ複数のデータ型があるのか？

メモリ効率と処理速度のバランスが理由です。byte（1バイト）で済む年齢にlong（8バイト）を使うのは無駄ですし、 大きな数値にbyteを使うと桁あふれ（オーバーフロー）が発生します。適切な型選択が重要です。

2. 変数宣言と初期化

基本的な変数宣言

// 宣言のみ（初期値はデフォルト値）
int age;        // 0が自動設定
double price;   // 0.0が自動設定
boolean flag;   // falseが自動設定

// 宣言と同時に初期化（推奨）
int studentCount = 25;
double productPrice = 1980.5;
boolean isValid = true;
char grade = 'A';

変数命名規則（重要）

camelCase（キャメルケース）:
studentName, totalPrice
意味のある名前:
int x; → int customerAge;
避けるべき: 数字開始、予約語、日本語

実践的な変数使用例 

// 顧客管理システムの例
int customerId = 12345;           // 顧客ID
String customerName = "田中太郎";  // 顧客名
int customerAge = 35;             // 年齢
double totalPurchase = 15800.0;   // 累計購入額
boolean isPremiumMember = true;   // プレミアム会員フラグ

// ECサイトの商品情報
long productCode = 4901234567890L;  // JANコード（Lを付ける）
float discountRate = 0.15f;        // 割引率（fを付ける）
char categoryCode = 'A';            // カテゴリコード

3. 型変換（キャスト）

暗黙的型変換（拡大変換）

小さい型から大きい型への変換は自動で行われます（データ損失なし）

byte b = 100;
int i = b;        // byte → int（自動変換）
double d = i;     // int → double（自動変換）

// 計算時の型変換
int num1 = 10;
double num2 = 3.5;
double result = num1 + num2;  // int → double
変換順序: byte → short → int → long → float → double

明示的型変換（縮小変換）

大きい型から小さい型への変換は明示的に指定が必要（データ損失の可能性）

double d = 123.456;
int i = (int)d;           // 123（小数部切り捨て）

long l = 2200000000L;     // 22億
int i2 = (int)l;          // データ損失の危険！

// 安全な変換例
double price = 1980.75;
int yen = (int)Math.round(price);  // 四捨五入
注意: 範囲を超えるとオーバーフロー発生

型変換でよくある間違い 

// 間違い例
int a = 10;
int b = 3;
double result = a / b;    // 結果: 3.0 (整数除算のため)

// 正しい例
double result = (double)a / b;  // 結果: 3.3333...

4. 演算子の詳細

算術演算子

演算子 名称 例 結果 注意点
+ 加算 5 + 3 8 文字列結合にも使用
- 減算 8 - 3 5 負数作成にも使用
* 乗算 4 * 5 20 オーバーフロー注意
/ 除算 10 / 3 3（整数） 整数除算は小数切り捨て
% 剰余 10 % 3 1 偶数判定で活用

比較演算子

演算子 名称 例 結果
== 等しい 5 == 5 true
!= 等しくない 5 != 3 true
> より大きい 8 > 5 true

重要な注意点

  • 文字列比較: ==ではなく.equals()使用
  • 浮動小数点: ==比較は危険（誤差あり）
  • 結果は必ずboolean型（true/false）

論理演算子

演算子 名称 例 結果
&& AND（論理積） true && false false
|| OR（論理和） true || false true
! NOT（否定） !true false

短絡評価（重要）

// &&の場合：左がfalseなら右は評価されない
boolean result = false && someMethod();

// ||の場合：左がtrueなら右は評価されない  
boolean result = true || someMethod();

効率化とエラー防止に重要な機能

5. 演算子優先順位表（改良版）

なぜ優先順位が重要？

数学と同様、計算順序によって結果が変わるからです。 例：5 + 3 * 2 は 5 + 6 = 11 であり、 (5 + 3) * 2 = 16 ではありません。

優先順位 演算子 名称 結合性 計算例 実行結果
1位（最高） ( ) 括弧 左から右 (2 + 3) * 4 5 * 4 = 20
2位 ++ -- インクリメント/デクリメント 右から左 ++a, a++ 前置/後置で差異
3位 * / % 乗算・除算・剰余 左から右 2 * 3 / 4 6 / 4 = 1（整数）
4位 + - 加算・減算 左から右 5 + 3 * 2 5 + 6 = 11
5位 < > <= >= 関係演算子 左から右 x > 5 true/false
6位 == != 等価演算子 左から右 a == 10 true/false
7位 && 論理AND 左から右 a > 5 && b < 10 短絡評価
8位 || 論理OR 左から右 a == 1 || b == 2 短絡評価
9位（最低） = += -= *= 代入演算子 右から左 a = b = 5 bに5、aにbの値

推奨される書き方 

// 括弧で明示的に順序指定
int result = (a + b) * (c - d);

// 複雑な条件は分割
boolean isValidAge = (age >= 18) && (age <= 65);
boolean hasPermission = (role.equals("admin")) || (isOwner);

混乱しやすい例 

// 読みにくい
boolean result = age > 18 && score >= 80 || isVIP;

// 読みやすく改善  
boolean result = ((age > 18) && (score >= 80)) || isVIP;

📝 演習問題

問題1: データ型選択

以下の値を格納するのに最適なデータ型を選択し、理由を説明してください。

  • • 人の年齢 (0-150)
  • • 商品価格 (小数点2桁まで)
  • • 在庫数 (0-1000000)
  • • ログイン状態 (ログイン中/未ログイン)
  • • 郵便番号 (1234567)

回答例と解説

• 人の年齢: byte - 範囲0-150なので-128～127で十分、メモリ効率が良い

• 商品価格: double - 小数点計算が必要、精度も重要

• 在庫数: int - 100万までなら21億の範囲で十分、一般的

• ログイン状態: boolean - true/falseの2値のみ

• 郵便番号: int - 数値だが計算しないならStringも可

問題2: 型変換

以下のコードの実行結果を予測し、その理由を説明してください。

int a = 10;
double b = 3.5;
int result1 = (int)(a + b);
double result2 = a / 3;
System.out.println("result1: " + result1);
System.out.println("result2: " + result2);

回答例と解説

result1: 13

理由: a(int) + b(double) → 10.0 + 3.5 = 13.5 → (int)で13に変換（小数部切り捨て）

result2: 3.0

理由: a(int) / 3(int) → 整数除算で10/3=3 → doubleに自動変換で3.0

注意: 小数結果が欲しい場合は (double)a / 3 または a / 3.0

問題3: 演算子優先順位

以下の式の計算順序と最終結果を求めてください。

int x = 5 + 3 * 2 - 1;
boolean y = x > 10 && x < 15 || x == 5;
System.out.println("x = " + x);
System.out.println("y = " + y);

回答例と解説

x の計算順序:

1. 3 * 2 = 6 （乗算が最優先）

2. 5 + 6 = 11 （加算）

3. 11 - 1 = 10 （減算）

結果: x = 10

y の計算順序:

1. x > 10 → 10 > 10 → false

2. x < 15 → 10 < 15 → true

3. false && true → false

4. x == 5 → 10 == 5 → false

5. false || false → false

結果: y = false

🧠 理解度クイズ

クイズ1: データ型のサイズ

int型のサイズと値の範囲として正しいものはどれですか？

正解: B) 32bit、-2,147,483,648 ～ 2,147,483,647

int型は32bit（4バイト）で、約-21億～21億の範囲です。これは世界人口の約3倍に相当し、 一般的な整数処理には十分な範囲を提供します。A)はshort型、C)はlong型、D)はbyte型の仕様です。

クイズ2: 型変換

次のコードの実行結果として正しいものはどれですか？

double d = 123.789;
int i = (int)d;
System.out.println(i);

正解: B) 123

明示的型変換（キャスト）で double → int に変換する際は、小数部が切り捨てられます。 四捨五入ではないので 123.789 → 123 となります。四捨五入したい場合は Math.round() メソッドを使用します。

クイズ3: 演算子優先順位（JavaSilver頻出）

次の式の計算結果として正しいものはどれですか？

int result = 2 + 3 * 4 - 1;
System.out.println(result);

正解: B) 13

計算順序:

1. 3 * 4 = 12 （乗算が最優先）

2. 2 + 12 = 14 （加算）

3. 14 - 1 = 13 （減算）

JavaSilver試験のポイント: 演算子優先順位は頻出問題です。乗除が加減より優先されることを確実に覚えましょう。

📚 JavaSilver試験対策

頻出問題パターン

パターン1: データ型のサイズ・範囲

各プリミティブ型のサイズ（bit）と値の範囲を正確に覚える

パターン2: 型変換の可否

暗黙的変換と明示的変換の判別、データ損失の有無

パターン3: 演算子優先順位

複雑な式の計算順序と最終結果の予測

試験攻略のコツ

暗記のコツ: int(32bit)を基準に、byte(8bit)、short(16bit)、long(64bit)と覚える
計算のコツ: 必ず乗除を先に計算、不安なら括弧で順序確認
型変換のコツ: 「大→小は明示的、小→大は自動」と覚える
時間短縮: 選択肢から逆算、明らかに間違いなものを除外

よくある間違いパターン

整数除算の罠

10 / 3 → 3（小数部切り捨て）

オーバーフローの見落とし

int の最大値を超えると負数になる

浮動小数点の精度

0.1 + 0.2 == 0.3 → false

文字列比較

String は == でなく .equals()
