1. OSI参照モデル（Open Systems Interconnection）

身近な例で理解：郵便システムに例えると

OSI参照モデルは、異なるメーカーの機器が通信できるように定められた「共通ルール」です。

これは郵便システムに似ています：手紙を書く人（アプリケーション）から、実際に配達する人（物理層）まで、それぞれが決められた役割を果たすことで確実に手紙が届きます。

👨‍🏫 講師向け指導ポイント

  • 段階的説明：まず「なぜ階層に分けるのか」から説明し、分業の効率性を強調
  • 身近な例：会社組織や学校の部活動の役割分担と関連付けて説明
  • 実例重視：各層で実際に使われているプロトコルやアプリケーションを具体的に示す
  • 相互関係：各層がどのように連携するかを「データの流れ」で視覚的に説明

OSI参照モデル7層構造

第1層：物理層（Physical Layer）
第2層：データリンク層（Data Link Layer）
第3層：ネットワーク層（Network Layer）
第4層：トランスポート層（Transport Layer）
第5層：セッション層（Session Layer）
第6層：プレゼンテーション層（Presentation Layer）
第7層：アプリケーション層（Application Layer）

詳細解説表：各層の役割と具体例

層 役割 身近な例 主要プロトコル/技術 実際の機器例
第7層
アプリケーション層		 ユーザーが直接操作するアプリケーションサービス ブラウザでWebサイト閲覧、メールソフトでメール送信 HTTP, HTTPS, FTP, SMTP, POP3 Webブラウザ、メールソフト
第6層
プレゼンテーション層		 データの形式変換、暗号化・復号化 写真をJPEGに圧縮、パスワードを暗号化 SSL/TLS, JPEG, MPEG, ASCII 暗号化ソフトウェア
第5層
セッション層		 通信の開始・維持・終了を管理 電話の発信・通話・切断の管理 NetBIOS, RPC, SQL セッション管理ソフトウェア
第4層
トランスポート層		 信頼性のあるデータ転送の保証 宅配便の確実な配達保証、受取確認 TCP, UDP OS内のTCP/IPスタック
第3層
ネットワーク層		 異なるネットワーク間の経路制御 郵便の住所による配送ルート決定 IP, ICMP, ARP ルータ

データリンク層		 同一ネットワーク内での確実なデータ転送 同じ建物内での手紙の受け渡し Ethernet, Wi-Fi, PPP スイッチ、無線アクセスポイント
第1層
物理層		 電気信号や光信号によるデータの物理的伝送 道路や線路などの物理的なインフラ 電気信号、光信号、電波 ケーブル、ハブ、リピータ

2. TCP/IPプロトコルスイート

身近な例で理解：国際会議の通訳システム

TCP/IPは、インターネットで使われる「共通言語」のセットです。世界中の異なるコンピュータが理解できる「標準語」として機能し、国際会議で各国の代表が英語で話すように、全てのコンピュータがTCP/IPで通信します。

TCP/IP階層モデル

TCP/IP層 OSI対応層 主な役割 代表的プロトコル 身近な例
アプリケーション層 第5-7層 ユーザーが利用するサービス HTTP, HTTPS, FTP, SMTP, DNS Webブラウザ、メールソフト
トランスポート層 第4層 信頼性のあるデータ転送 TCP, UDP 宅配便（TCP）、郵便（UDP）
インターネット層 第3層 ネットワーク間の経路制御 IP, ICMP, ARP 住所による配送ルート決定
ネットワークインターフェース層 第1-2層 物理的なデータ転送 Ethernet, Wi-Fi 道路、鉄道などのインフラ

TCP vs UDP：詳細比較

特徴 TCP（Transmission Control Protocol） UDP（User Datagram Protocol）
正式名称 伝送制御プロトコル ユーザーデータグラムプロトコル
信頼性 高い（配達確認あり） 低い（配達確認なし）
速度 普通（確認処理で少し遅い） 高速（確認処理なし）
身近な例 書留郵便（受取確認必要） 普通郵便（受取確認不要）
適用場面 Webブラウジング、メール送信、ファイル転送 動画配信、オンラインゲーム、DNS照会
使用例 HTTP, HTTPS, FTP, SMTP 動画ストリーミング、VoIP, DHCP

👨‍🏫 講師向け指導ポイント

  • プロトコル理解のコツ：「なぜそのプロトコルが必要か」を実際のインターネット利用場面で説明
  • TCP/UDP使い分け：動画配信でUDPが使われる理由を「リアルタイム性重視」で説明
  • 実演効果：可能であればWebブラウザの開発者ツールでHTTP通信を実際に見せる
  • 試験対策：プロトコルの特徴は比較表で整理し、どちらがどの場面で適しているかを重点的に

3. IPv4アドレッシング方式

身近な例で理解：住所システム

IPアドレスは「インターネット上の住所」です。郵便番号（ネットワーク部）で地域を特定し、番地（ホスト部）で具体的な建物を特定する仕組みと同じです。

例： 192.168.1.100

・192.168.1 → 郵便番号（どのネットワークか）

・100 → 番地（そのネットワーク内のどの機器か）

IPv4アドレスクラス分類

クラス 範囲 ネットワーク数 ホスト数 用途・特徴 実際の使用例
クラスA 1.0.0.0 ～ 126.255.255.255 126個 約1677万台 大規模組織向け（巨大企業、国家レベル） 大手通信会社、政府機関
クラスB 128.0.0.0 ～ 191.255.255.255 約16,000個 約65,000台 中規模組織向け（大学、中堅企業） 大学、中規模企業
クラスC 192.0.0.0 ～ 223.255.255.255 約200万個 254台 小規模組織向け（一般企業、家庭） 中小企業、家庭用ルータ
クラスD 224.0.0.0 ～ 239.255.255.255 - - マルチキャスト用（同時配信） 動画配信、会議システム
クラスE 240.0.0.0 ～ 255.255.255.255 - - 実験用（一般利用不可） 研究機関での実験

プライベートアドレスとパブリックアドレス

プライベートアドレス（私有アドレス）

インターネットで使えない「内部専用住所」

  • クラスA: 10.0.0.0 ～ 10.255.255.255
  • クラスB: 172.16.0.0 ～ 172.31.255.255
  • クラスC: 192.168.0.0 ～ 192.168.255.255
身近な例： 家庭のWi-Fiルータが配布するアドレス（192.168.1.1など）

パブリックアドレス（グローバルアドレス）

インターネットで使える「公式住所」

  • 世界で唯一のアドレス
  • インターネット上で直接通信可能
  • ISP（プロバイダ）から割り当て
  • 有限の資源（IPv4は約43億個）
身近な例： Webサイトのサーバーが使用するアドレス

サブネット計算の基礎

サブネット計算の手順（段階的解説）

例題： 192.168.1.0/24 のネットワークを4つのサブネットに分割
手順1： 必要なビット数を計算
4つに分割 → 2² = 4 → 2ビット必要
手順2： 新しいサブネットマスクを計算
/24 + 2ビット = /26 （255.255.255.192）
手順3： 各サブネットの範囲を計算

サブネット1: 192.168.1.0/26 （192.168.1.1 ～ 192.168.1.62）

サブネット2: 192.168.1.64/26 （192.168.1.65 ～ 192.168.1.126）

サブネット3: 192.168.1.128/26 （192.168.1.129 ～ 192.168.1.190）

サブネット4: 192.168.1.192/26 （192.168.1.193 ～ 192.168.1.254）

5. ネットワークセキュリティ基礎

身近な例で理解：家のセキュリティシステム

ネットワークセキュリティは、家のセキュリティと似ています。玄関の鍵（認証）、防犯カメラ（監視）、警備システム（ファイアウォール）、金庫（暗号化）など、多層的な防御で大切な情報を守ります。

主要なセキュリティ技術

技術名 役割 身近な例 実装場所 保護対象
ファイアウォール 不正アクセスの遮断 住宅の門番・警備員 ネットワークの入口 内部ネットワーク全体
暗号化（SSL/TLS） データの盗聴防止 重要書類を金庫に保管 通信経路上 通信内容
認証システム 正当なユーザーの確認 身分証明書での本人確認 システムログイン時 システムへのアクセス
IDS/IPS 攻撃の検知・防御 防犯カメラ・自動通報システム ネットワーク内各所 攻撃の早期発見
VPN 安全な通信トンネル 専用の暗号化された通話回線 拠点間・リモートアクセス 外部通信の安全性

👨‍🏫 セキュリティ指導のポイント

  • 現実性の強調：実際のサイバー攻撃事例を紹介し、セキュリティの重要性を実感させる
  • 多層防御の概念：一つの対策だけでは不十分で、複数の対策を組み合わせる必要性を説明
  • 身近な体験：HTTPSの鍵マークやパスワード設定など、日常で見るセキュリティ要素を具体例に
  • コスト意識：セキュリティは利便性とのトレードオフであることを理解させる

6. 演習問題

問題1: OSI参照モデル

以下の機能はOSI参照モデルのどの層に対応するか答えなさい。

  1. Webブラウザでのファイルダウンロード
  2. IPアドレスによる経路制御
  3. MACアドレスによるフレーム配送
  4. TCPによる再送制御

解答・解説：

  1. 第7層（アプリケーション層） - ユーザーが直接利用するアプリケーションサービス
  2. 第3層（ネットワーク層） - 異なるネットワーク間での経路選択
  3. 第2層（データリンク層） - 同一ネットワーク内での確実なデータ転送
  4. 第4層（トランスポート層） - 信頼性のあるデータ転送の保証

問題2: TCP/UDP選択

以下のアプリケーションには、TCPとUDPのどちらが適しているか理由とともに答えなさい。

  1. オンライン銀行取引システム
  2. ライブ動画配信
  3. メール送信
  4. オンラインゲーム

解答・解説：

  1. TCP - 金融取引は確実性が最重要。データの欠損や改ざんは許されない
  2. UDP - リアルタイム性重視。多少の画質劣化があっても遅延は避けたい
  3. TCP - メールは確実に届く必要がある。速度より信頼性を優先
  4. UDP - リアルタイム性が重要。わずかな遅延がゲーム体験に影響

問題3: サブネット計算

192.168.10.0/24のネットワークを8つのサブネットに分割した場合、各サブネットで利用可能なホスト数を求めなさい。

解答・解説：

手順1：8つに分割するために必要なビット数：2³ = 8 → 3ビット

手順2：新しいサブネットマスク：/24 + 3 = /27

手順3：ホスト部のビット数：32 - 27 = 5ビット

手順4：利用可能ホスト数：2⁵ - 2 = 30台

※-2は、ネットワークアドレスとブロードキャストアドレスを除く

問題4: ネットワーク機器選択

以下の状況で最適なネットワーク機器を選択し、理由を説明しなさい。

  1. オフィス内の20台のPCを相互接続したい
  2. 2つの異なる拠点のネットワークを接続したい
  3. 有線LANにスマートフォンやタブレットを接続したい

解答・解説：

  1. スイッチ - 同一ネットワーク内での効率的な通信。MACアドレステーブルによる学習機能で衝突を回避
  2. ルータ - 異なるネットワーク間の接続。IPアドレスによる経路制御で最適ルートを選択
  3. 無線アクセスポイント - 有線LANと無線LANの変換。Wi-Fi接続でモバイルデバイスにネットワークアクセスを提供

問題5: セキュリティ対策

以下の脅威に対して最も効果的なセキュリティ対策を選択しなさい。

  1. 外部からの不正侵入
  2. 通信内容の盗聴
  3. なりすましによる不正ログイン
  4. ウイルス感染

解答・解説：

  1. ファイアウォール - ネットワークの入口で不正なアクセスを遮断
  2. 暗号化（SSL/TLS） - 通信データを暗号化して第三者による解読を防止
  3. 多要素認証 - パスワード+生体認証等の複数要素で本人確認を強化
  4. アンチウイルスソフト - 既知のウイルスパターンを検出・駆除

7. 基本情報技術者試験 模擬問題

問題1

OSI参照モデルにおいて、IPアドレスによる経路制御を行う層はどれか。

正解：イ) ネットワーク層

ネットワーク層（第3層）は、異なるネットワーク間でのデータ転送と経路制御を担当する層です。IPアドレスを使用して最適な経路を決定します。

問題2

TCPとUDPの特徴に関する記述のうち、正しいものはどれか。

正解：ウ) TCPは接続型のプロトコルである

TCPは3ウェイハンドシェイクによる接続確立を行う接続型プロトコルです。確認応答、再送制御、順序制御により高い信頼性を実現します。

問題3

IPv4において、プライベートアドレスの範囲に含まれないものはどれか。

正解：ウ) 172.32.0.1

プライベートアドレスの範囲は、クラスBでは172.16.0.0～172.31.255.255です。172.32.0.1はこの範囲外のため、パブリックアドレスです。

問題4

192.168.1.0/26のネットワークで利用可能なホスト数はいくつか。

正解：イ) 62

/26は26ビットがネットワーク部、6ビットがホスト部。利用可能ホスト数は2⁶-2=64-2=62台（ネットワークアドレスとブロードキャストアドレスを除く）

問題5

LANにおいて、MACアドレスを学習してフレームを適切なポートに転送する機器はどれか。

正解：イ) スイッチ

スイッチは、MACアドレステーブルを構築してMACアドレスを学習し、該当するポートにのみフレームを転送することで効率的な通信を実現します。

8. ITエンジニア豆知識

インターネットの父：ヴィントン・サーフ

TCP/IPプロトコルの開発者の一人で「インターネットの父」と呼ばれます。現在もGoogleで技術開発に従事し、次世代インターネット技術の研究を続けています。

IPv4アドレス枯渇問題

IPv4アドレスは約43億個しかなく、2011年に枯渇しました。現在はIPv6（128ビット、約340澗個）への移行が進んでいますが、まだIPv4が主流です。

史上最大のDDoS攻撃

2020年にAmazonが受けた2.3Tbpsの攻撃が記録的です。これは全米のインターネット帯域の約半分に相当する規模で、現代のサイバー脅威の深刻さを物語っています。

パケットの旅行時間

光の速度でも、日本からアメリカまでパケットが届くのに約150ミリ秒かかります。これは物理的な距離による避けられない遅延で、オンラインゲームでは重要な要素です。
