2. エンティティマッピングとアノテーション完全理解

⚠️ 必須改良事項：

アノテーションの概念説明を強化：

  • 「@Entityは身分証明書のようなもの、このクラスがデータベーステーブルと対応すると宣言」
  • 「@Idは主キー指定、データの一意識別子（マイナンバーのような概念）」
  • 「@OneToManyは親子関係の定義（1つの部署に複数の社員）」
  • 「@JoinColumnは外部キー設定、テーブル間の橋渡し役」

コード例で理解する

// ユーザーエンティティの例
@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(name = "username", nullable = false, unique = true)
    private String username;
    
    @OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
    private List<Order> orders;
}

アノテーション比較表に「実用例」列を追加：

  • @Entity: 「ユーザー情報、商品マスタ」
  • @OneToMany: 「注文-注文明細、カテゴリ-商品」
  • @ManyToOne: 「社員-部署、商品-カテゴリ」

3. JPQL・Criteria API・クエリ最適化

JPQL vs SQL比較

// JPQL（オブジェクト指向）
SELECT u FROM User u WHERE u.age > :minAge

// 従来のSQL
SELECT * FROM users WHERE age > ?

Criteria APIの動的クエリ

// 条件に応じて動的にクエリを構築
CriteriaBuilder cb = em.getCriteriaBuilder();
CriteriaQuery<User> query = cb.createQuery(User.class);
Root<User> user = query.from(User.class);

if (name != null) {
    query.where(cb.equal(user.get("name"), name));
}

パフォーマンス最適化の実例

  • N+1問題回避：@BatchSize、fetch join活用
  • 遅延読み込み：@LazyLoading適切な設定
  • クエリキャッシュ：@QueryHint活用
