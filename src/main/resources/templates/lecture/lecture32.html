例題: 配列処理（よく出る形式）

public class ArrayProcessor {
    public static void main(String[] args) {
        int[] data = {3, 1, 4, 1, 5, 9, 2, 6};
        int result = process(data);
        System.out.println(result);  // この出力は？
    }
    
    static int process(int[] arr) {
        int max = arr[0];
        int count = 0;
        for (int i = 1; i < arr.length; i++) {
            if (arr[i] > max) {
                max = arr[i];
                count = i;
            }
        }
        return count;  // 最大値のインデックスを返す
    }
}

解法ポイント: 配列を順次チェックし、最大値とそのインデックスを追跡。最終的に最大値（9）のインデックス（5）が返される。

実践演習問題

プログラミング演習

問題1: 配列処理（基本）

public class Problem1 {
    public static void main(String[] args) {
        int[] numbers = {10, 5, 8, 3, 7};
        int result = calculate(numbers);
        System.out.println(result);
    }
    
    static int calculate(int[] arr) {
        int sum = 0;
        for (int i = 0; i < arr.length; i++) {
            if (arr[i] % 2 == 1) {  // 奇数の場合
                sum += arr[i];
            }
        }
        return sum;
    }
}

上記のプログラムの出力結果は何か。また、その理由を説明せよ。

正解: 15

解説: 配列 {10, 5, 8, 3, 7} から奇数のみを抽出して合計します。
• 10 % 2 = 0 (偶数) → 加算しない
• 5 % 2 = 1 (奇数) → sum = 0 + 5 = 5
• 8 % 2 = 0 (偶数) → 加算しない
• 3 % 2 = 1 (奇数) → sum = 5 + 3 = 8
• 7 % 2 = 1 (奇数) → sum = 8 + 7 = 15
したがって、出力は15となります。

問題2: 文字列処理（応用）

public class Problem2 {
    public static void main(String[] args) {
        String text = "Hello World";
        String result = process(text);
        System.out.println(result);
    }
    
    static String process(String str) {
        StringBuilder sb = new StringBuilder();
        for (int i = str.length() - 1; i >= 0; i--) {
            char c = str.charAt(i);
            if (c != ' ') {
                sb.append(c);
            }
        }
        return sb.toString();
    }
}

このプログラムの実行結果として正しいものはどれか。

正解: dlroWolleH

解説: このプログラムは文字列を逆順にしながらスペースを除去します。
• "Hello World" を末尾から処理
• 'd', 'l', 'r', 'o', 'W', ' '(スキップ), 'o', 'l', 'l', 'e', 'H'
• スペースを除去して結合すると "dlroWolleH"
StringBuilder により効率的な文字列結合が行われています。

問題3: 二分探索のトレース

ソート済み配列 [2, 5, 8, 12, 16, 23, 38, 45, 67, 78] に対して、値 23 を二分探索で検索する。 各ステップでの left, right, mid の値と比較結果を示せ。

トレース結果:

ステップ left right mid arr[mid] 比較 次のアクション
1 0 9 4 16 16 < 23 left = mid + 1 = 5
2 5 9 7 45 45 > 23 right = mid - 1 = 6
3 5 6 5 23 23 == 23 発見！ インデックス5を返す

結果: 値 23 はインデックス 5 で発見されました。計算量 O(log n) で効率的に検索できています。

問題4: スタック操作

空のスタックに対して以下の操作を順番に実行する。各操作後のスタックの状態を示せ。
操作: push(3), push(7), pop(), push(1), push(9), pop(), pop()

スタック操作のトレース:

操作手順:
  • 初期状態: [ ] (空)
  • push(3): [3]
  • push(7): [3, 7]
  • pop(): [3] (7を取り出し)
  • push(1): [3, 1]
  • push(9): [3, 1, 9]
  • pop(): [3, 1] (9を取り出し)
  • pop(): [3] (1を取り出し)
LIFO原則:

スタックは Last In, First Out (LIFO) の原則に従います。最後に追加された要素が最初に取り出されます。

最終状態: [3] - 要素3が残っています。

問題5: セキュリティ読解

「B社では、リモートワークの普及に伴い、VPN（Virtual Private Network）を導入してセキュアな通信環境を構築した。VPNは、公衆ネットワーク上に仮想的な専用ネットワークを構築する技術であり、暗号化により通信内容の機密性を保護する。B社が採用したIPsec VPNは、IP層での暗号化を提供し、エンドツーエンドの通信を保護する。」

「しかし、従業員の一部が個人所有のデバイスから接続しており、セキュリティポリシーの遵守が課題となっている。そこで、デバイス認証機能を追加し、事前に登録されたデバイスのみが接続できるよう制限を設けた。さらに、多要素認証（MFA: Multi-Factor Authentication）を導入し、パスワードに加えてワンタイムパスワードによる認証を必須とした。」

上記の文章を読んで、B社が実施したセキュリティ対策の目的と効果について説明せよ。

解答例:

1. VPN導入の目的と効果
  • • 目的: リモートワーク環境でのセキュアな通信確保
  • • 効果: IPsecによる暗号化で通信内容の機密性を保護
2. デバイス認証の追加
  • • 目的: 個人所有デバイスからの不正アクセス防止
  • • 効果: 事前登録されたデバイスのみ接続許可
3. 多要素認証（MFA）の導入
  • • 目的: パスワード単体認証の脆弱性補完
  • • 効果: ワンタイムパスワードによる追加認証で不正ログイン防止

総合効果: これらの対策により、リモートワーク環境における情報セキュリティの三要素（機密性、完全性、可用性）を包括的に保護することが可能となった。

問1: プログラミング（Java）【必須解答・20点】 

public class ArrayAnalyzer {
    public static void main(String[] args) {
        int[][] matrix = {
            {1, 4, 7, 2},
            {3, 6, 9, 5},
            {2, 8, 1, 4}
        };
        int result = findMaxSum(matrix);
        System.out.println("最大合計: " + result);
    }
    
    static int findMaxSum(int[][] arr) {
        int maxSum = Integer.MIN_VALUE;
        
        // 各行の合計を計算
        for (int i = 0; i < arr.length; i++) {
            int rowSum = 0;
            for (int j = 0; j < arr[i].length; j++) {
                rowSum += arr[i][j];
            }
            if (rowSum > maxSum) {
                maxSum = rowSum;
            }
        }
        
        // 各列の合計を計算
        for (int j = 0; j < arr[0].length; j++) {
            int colSum = 0;
            for (int i = 0; i < arr.length; i++) {
                colSum += arr[i][j];
            }
            if (colSum > maxSum) {
                maxSum = colSum;
            }
        }
        
        return maxSum;
    }
}

設問1: このプログラムの出力結果を答えよ。

設問2: findMaxSumメソッドの処理内容を説明せよ。

設問3: 計算量を O記法で表現せよ。

設問1の解答: 最大合計: 23

行の合計: [14, 23, 15]、列の合計: [6, 18, 17, 11]の中で最大値は23

設問2の解答:

2次元配列の各行と各列の要素の合計を計算し、その中で最大値を求める処理

設問3の解答: O(m×n)

m=行数、n=列数。全要素を2回走査するため線形時間

問2: 情報セキュリティ【必須解答・20点】

「C社では、クラウドサービスを活用したデータ管理システムを構築している。機密データの保護のため、データベース内の個人情報は AES-256 で暗号化し、暗号鍵は専用のキー管理システム（KMS: Key Management System）で管理している。また、データアクセス時には、ロールベースアクセス制御（RBAC: Role-Based Access Control）により、従業員の職務に応じた適切なアクセス権限を設定している。」

「しかし、最近のセキュリティ監査で、以下の課題が指摘された。(1)データベースのアクセスログが十分に記録されていない、(2)暗号鍵のローテーション（定期的な更新）が実施されていない、(3)従業員の退職時にアクセス権限の削除が遅れる場合がある。これらの課題を解決するため、C社は包括的なセキュリティ対策の見直しを行うことになった。」

設問1: AES-256暗号化の特徴と、KMSを使用する利点を説明せよ。

設問2: 指摘された3つの課題それぞれについて、適切な対策を提案せよ。

設問3: RBACの仕組みとその利点について説明せよ。

設問1の解答:

  • • AES-256: 256ビット鍵長の共通鍵暗号、高い安全性を提供
  • • KMS利点: 暗号鍵の集中管理、アクセス制御、監査ログ記録

設問2の解答:

  • • (1) SIEM導入によるリアルタイムログ監視、アクセス記録の詳細化
  • • (2) 自動化された暗号鍵ローテーションスケジュールの実装
  • • (3) IDライフサイクル管理システムの導入、自動的な権限削除

設問3の解答:

RBACは職務に基づく役割（ロール）に権限を割り当て、ユーザーは役割を通じて権限を取得する仕組み。最小権限の原則を実現し、管理効率を向上させる。
