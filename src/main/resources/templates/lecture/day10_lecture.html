2. Sealed Classes（密封クラス）

Sealed Classesとは

Sealed Classes（密封クラス）は、継承を指定したクラスのみに制限するJava17の新機能です。 図書館に例えると、「この本は指定された人だけが借りられる」ような仕組みです。

従来の問題点

どのクラスからでも継承可能 → 予期しない継承によるバグや設計の破綻

Sealed Classesの解決策

継承可能なクラスを明示的に指定 → API設計の堅牢性向上

実装例：決済システム

従来のコード（Java16以前）

// 問題：どのクラスからでも継承可能
public abstract class PaymentMethod {
    public abstract void pay(double amount);
}

// 予期しない継承が可能
class UnknownPayment extends PaymentMethod {
    @Override
    public void pay(double amount) {
        // 危険な実装が可能
    }
}

改善後のコード（Java17）

// 解決：継承を指定クラスのみに制限
public sealed class PaymentMethod 
    permits CreditCard, PayPal, BankTransfer {
    
    public abstract void pay(double amount);
}

// 許可されたクラスのみ継承可能
final class CreditCard extends PaymentMethod {
    @Override
    public void pay(double amount) {
        System.out.println("クレジットカードで" + amount + "円を決済");
    }
}

final class PayPal extends PaymentMethod {
    @Override
    public void pay(double amount) {
        System.out.println("PayPalで" + amount + "円を決済");
    }
}

final class BankTransfer extends PaymentMethod {
    @Override
    public void pay(double amount) {
        System.out.println("銀行振込で" + amount + "円を決済");
    }
}

// ❌ コンパイルエラー：許可されていないクラス
// class BitCoin extends PaymentMethod { }

3. Pattern Matching for instanceof

Pattern Matchingとは

Pattern Matching for instanceofは、instanceof後の自動キャスト機能です。 郵便配達に例えると、「この荷物が書留なら、書留として自動的に処理する」ような便利な仕組みです。

従来の冗長な処理

instanceof確認 → 明示的キャスト → 変数代入 → 処理

Pattern Matchingの簡略化

instanceof確認 + 自動キャスト + 変数代入を一気に実行

実装例：ユーザー入力処理

従来のコード（Java16以前）

// 冗長で間違いやすい処理
public void processInput(Object input) {
    if (input instanceof String) {
        String str = (String) input; // 明示的キャスト
        System.out.println("文字列：" + str.length() + "文字");
    } else if (input instanceof Integer) {
        Integer num = (Integer) input; // 明示的キャスト
        System.out.println("数値：" + num * 2);
    } else if (input instanceof Boolean) {
        Boolean bool = (Boolean) input; // 明示的キャスト
        System.out.println("真偽値：" + (bool ? "有効" : "無効"));
    }
}

改善後のコード（Java17）

// 簡潔で安全な処理
public void processInput(Object input) {
    if (input instanceof String str) { // 自動キャスト
        System.out.println("文字列：" + str.length() + "文字");
    } else if (input instanceof Integer num) { // 自動キャスト
        System.out.println("数値：" + num * 2);
    } else if (input instanceof Boolean bool) { // 自動キャスト
        System.out.println("真偽値：" + (bool ? "有効" : "無効"));
    }
}

4. Records（レコード）

Recordsとは

Recordsは、データホルダークラスの自動生成機能です。 設計図に例えると、「住所・名前・電話番号」を書くだけで、必要な機能が自動的に完成する便利な仕組みです。

従来の問題点

データクラス作成に大量のボイラープレートコード（getter, setter, equals, hashCode, toString）

Recordsの解決策

1行の宣言で必要なメソッドを自動生成、実装工数を大幅削減

実装例：顧客データ管理

従来のコード（Java16以前）

// 大量のボイラープレートコード
public class Customer {
    private final String name;
    private final String email;
    private final int age;
    
    public Customer(String name, String email, int age) {
        this.name = name;
        this.email = email;
        this.age = age;
    }
    
    public String getName() { return name; }
    public String getEmail() { return email; }
    public int getAge() { return age; }
    
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) 
            return false;
        Customer customer = (Customer) obj;
        return age == customer.age &&
               Objects.equals(name, customer.name) &&
               Objects.equals(email, customer.email);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(name, email, age);
    }
    
    @Override
    public String toString() {
        return "Customer{" +
               "name='" + name + '\'' +
               ", email='" + email + '\'' +
               ", age=" + age +
               '}';
    }
}

// 約50行のコード！

改善後のコード（Java17）

// たった1行でデータクラス完成！
public record Customer(String name, String email, int age) {}

// 使用例
public class CustomerService {
    public void demonstrateRecord() {
        // インスタンス作成
        var customer = new Customer("田中太郎", "tanaka@example.com", 30);
        
        // 自動生成されたメソッドが使用可能
        System.out.println("名前: " + customer.name());        // getter
        System.out.println("メール: " + customer.email());     // getter
        System.out.println("年齢: " + customer.age());         // getter
        
        // equals, hashCode, toString も自動実装
        var customer2 = new Customer("田中太郎", "tanaka@example.com", 30);
        System.out.println("同一判定: " + customer.equals(customer2)); // true
        System.out.println("文字列表現: " + customer.toString());
    }
}

// わずか数行で完成！

5. 模擬テスト②

📝 テスト概要

第1週（Day1-10）の学習内容とJava17新機能の理解度を確認します。 JavaSilver試験の出題形式に準拠した問題で実践力を測定します。

問題1: Java17新機能（Records）

以下のコードについて、正しい説明を選択してください。 

public record User(String name, int age) {
    public User {
        if (age < 0) {
            throw new IllegalArgumentException("年齢は0以上である必要があります");
        }
    }
}

正解: B

解説: Recordsではコンパクトコンストラクタを使用してバリデーション処理を記述できます。 このコンストラクタは通常のコンストラクタの前に実行され、フィールドの検証に使用されます。

  • A. 間違い - Record内でのコンパクトコンストラクタは有効な記法
  • C. 間違い - Recordsはgetterのみ自動生成（setterは生成されない：immutable）
  • D. 間違い - equals(), hashCode(), toString()は自動生成される

問題2: Pattern Matching

以下のコードの実行結果として正しいものを選択してください。 

Object obj = "Hello";
if (obj instanceof String str && str.length() > 3) {
    System.out.println("長い文字列: " + str);
} else if (obj instanceof String str) {
    System.out.println("短い文字列: " + str);
} else {
    System.out.println("文字列以外");
}

正解: A

解説: "Hello"の文字数は5文字なので、最初の条件（length() > 3）が真となり、 「長い文字列: Hello」が出力されます。

  • Pattern Matchingにより、obj instanceof String str で自動的にキャストされた変数strが使用可能
  • 論理AND演算子（&&）により、型チェックと長さチェックが同時に評価される
  • 最初の条件が真なので、else if以降は実行されない

問題3: Sealed Classes

以下のSealed Classの定義で、コンパイルエラーになるものを選択してください。

A.

public sealed class Shape permits Circle, Rectangle {}

B.

public sealed class Shape permits Circle, Rectangle {}
final class Circle extends Shape {}
final class Rectangle extends Shape {}

C.

public sealed class Shape permits Circle, Rectangle {}
final class Circle extends Shape {}
class Triangle extends Shape {}

D.

public sealed class Shape permits Circle {}
non-sealed class Circle extends Shape {}

正解: C

解説: Sealed Classでは、permits句で指定されていないクラス（Triangle）は継承できません。 また、継承するクラスは同一パッケージ内である必要があります。

  • A. 正常 - Sealed Classの正しい宣言
  • B. 正常 - permits句で指定したクラスがfinalで継承している
  • C. エラー - Triangleはpermits句で指定されていないため継承不可
  • D. 正常 - non-sealedキーワードで継承制限を解除

問題4: 総合問題（第1週復習）

以下のコードについて、正しい説明を選択してください。 

public class QuizApp {
    public static void main(String[] args) {
        var numbers = List.of(1, 2, 3, 4, 5);
        var result = numbers.stream()
                           .filter(n -> n % 2 == 0)
                           .map(n -> n * n)
                           .collect(Collectors.toList());
        System.out.println(result);
    }
}

正解: B

解説: Stream APIの処理順序に従って実行されます。

処理フロー:
  1. 元のリスト: [1, 2, 3, 4, 5]
  2. filter(n -> n % 2 == 0): 偶数のみ抽出 → [2, 4]
  3. map(n -> n * n): 各要素を二乗 → [4, 16]
  4. collect(): リストに収集 → [4, 16]
  • var キーワード（Java10新機能）により型推論
  • List.of()（Java9新機能）により不変リスト生成
  • Stream API（Java8導入）による関数型処理

問題5: JavaSilver対策

以下のコードの実行結果として正しいものを選択してください。 

public class Test {
    public static void main(String[] args) {
        String str1 = "Java17";
        String str2 = new String("Java17");
        System.out.println(str1 == str2);
        System.out.println(str1.equals(str2));
    }
}

正解: B

解説: Javaの文字列比較における「参照比較」と「値比較」の違いを理解する重要な問題です。

詳細解説:
  • str1 == str2: false - 参照比較（異なるオブジェクトを参照）
  • str1.equals(str2): true - 値比較（文字列の内容が同じ）
重要ポイント:
  • リテラル文字列はString Pool（文字列プール）に格納
  • new String()は毎回新しいオブジェクトを生成
  • 文字列比較は必ずequals()メソッドを使用
  • JavaSilver試験で最も頻出する引っかけ問題パターン
