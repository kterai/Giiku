1. レスポンシブデザインの基礎理論

レスポンシブデザインとは？

レスポンシブデザイン（Responsive Design：応答的設計）とは、ユーザーの画面サイズや向きに応じて、Webページのレイアウトが自動的に最適化される設計手法です。まるで水のように容器の形に合わせて変化するため、「流動的レイアウト」とも呼ばれます。

📱 実際の動作確認

ブラウザの幅を変更してみてください。この枠の色が変わります！

  • • 768px以下（スマホ）: 赤色背景
  • • 769px-1024px（タブレット）: オレンジ色背景
  • • 1025px以上（PC）: 緑色背景

レスポンシブデザインの3つの柱

1. フルードグリッド

固定サイズではなく、パーセンテージで幅を指定する流動的なレイアウト

/* ❌ 固定レイアウト */ .container { width: 960px; } /* ✅ フルードレイアウト */ .container { width: 100%; max-width: 1200px; }

2. メディアクエリ

画面サイズに応じて異なるCSSを適用する技術

/* スマホ用 */ @media (max-width: 768px) { .column { width: 100%; } } /* PC用 */ @media (min-width: 1024px) { .column { width: 50%; } }

3. フレキシブルメディア

画像や動画を画面サイズに応じて拡縮させる技術

/* レスポンシブ画像 */ img { max-width: 100%; height: auto; } /* レスポンシブ動画 */ video { width: 100%; height: auto; }

モバイルファースト設計の重要性

モバイルファースト（Mobile First）とは、最初にスマートフォン向けのデザインを作成し、その後により大きな画面向けに拡張していく設計手法です。

なぜモバイルファーストなのか？

  • • スマートフォンからのアクセスが全体の60%以上
  • • 小さい画面から設計すると本質的なコンテンツに集中できる
  • • パフォーマンスが軽量に保たれる
  • • SEOでモバイルフレンドリーが重視される

実装の流れ

  1. 1. スマホ用デザイン（320px～）を作成
  2. 2. タブレット用（768px～）に拡張
  3. 3. PC用（1024px～）に拡張
  4. 4. 大画面用（1200px～）に拡張

主要なブレークポイント設定

デバイス 画面幅 メディアクエリ 実用例
スマートフォン ～767px max-width: 767px 1カラムレイアウト、大きなタップ領域
タブレット 768px～1023px min-width: 768px 2カラムレイアウト、適度な余白
デスクトップ 1024px～1199px min-width: 1024px 3カラムレイアウト、リッチなUI
大画面 1200px～ min-width: 1200px 最大幅制限、センタリング

2. Bootstrap グリッドシステム完全習得

Bootstrapとは？

Bootstrap（ブートストラップ）とは、TwitterやMetaなどの大手企業で使われている、レスポンシブWebサイトを効率的に構築するためのCSSフレームワークです。まるで家を建てる時の設計図のように、あらかじめ用意されたパーツを組み合わせることで、美しく機能的なWebサイトを短時間で作成できます。

Bootstrapのメリット

  • • 開発速度の向上：既製パーツで効率化
  • • レスポンシブ対応：自動で全デバイス対応
  • • ブラウザ互換性：クロスブラウザ対応済み
  • • 保守性：統一されたコーディング規約

Bootstrap CDN読み込み

<!-- CSS --> <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet"> <!-- JavaScript --> <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>

12カラムシステムの数学的根拠

なぜ12なのか？数字の12は、1, 2, 3, 4, 6, 12で割り切れるため、レイアウトの組み合わせが非常に豊富になります。まるでレゴブロックのように、様々なパターンでコンテンツを配置できます。

分割パターン例

  • • 12 ÷ 1 = 12：全幅（ヘッダー・フッター）
  • • 12 ÷ 2 = 6：2分割（メイン・サイドバー）
  • • 12 ÷ 3 = 4：3分割（商品カード一覧）
  • • 12 ÷ 4 = 3：4分割（機能紹介）
  • • 12 ÷ 6 = 2：6分割（詳細情報）

実際のクラス名

<!-- 全幅 --> <div class="col-12">ヘッダー</div> <!-- 2分割 --> <div class="col-8">メイン</div> <div class="col-4">サイドバー</div> <!-- 3分割 --> <div class="col-4">商品1</div> <div class="col-4">商品2</div> <div class="col-4">商品3</div>

4. カスタマイズとベストプラクティス

CSS変数によるBootstrap 5カスタマイゼーション

Bootstrap 5ではCSS カスタムプロパティ（CSS変数）を活用して、リアルタイムでテーマカラーやサイズを変更できます。JavaScriptと組み合わせれば、ダークモード切り替えなども簡単に実装できます。

カスタムCSSでの上書き

:root { /* Bootstrapのデフォルト色を上書き */ --bs-primary: #6f42c1; --bs-secondary: #6c757d; --bs-success: #28a745; --bs-danger: #dc3545; --bs-warning: #ffc107; --bs-info: #17a2b8; /* カスタム変数の追加 */ --my-brand-color: #ff6b6b; --my-accent-color: #4ecdc4; --my-font-size: 1.1rem; } /* カスタム変数の使用 */ .custom-button { background-color: var(--my-brand-color); color: white; font-size: var(--my-font-size); border: none; padding: 0.75rem 1.5rem; border-radius: 0.5rem; transition: all 0.3s ease; } .custom-button:hover { background-color: var(--my-accent-color); transform: translateY(-2px); box-shadow: 0 4px 8px rgba(0,0,0,0.2); }

JavaScriptでの動的変更

// ダークモード切り替え function toggleDarkMode() { const root = document.documentElement; const isDark = root.getAttribute('data-theme') === 'dark'; if (isDark) { // ライトモード root.setAttribute('data-theme', 'light'); root.style.setProperty('--bs-body-bg', '#ffffff'); root.style.setProperty('--bs-body-color', '#212529'); } else { // ダークモード root.setAttribute('data-theme', 'dark'); root.style.setProperty('--bs-body-bg', '#212529'); root.style.setProperty('--bs-body-color', '#ffffff'); } } // テーマカラー変更 function changeThemeColor(color) { document.documentElement.style.setProperty('--bs-primary', color); // すべての.btn-primaryボタンに即座に反映 const primaryButtons = document.querySelectorAll('.btn-primary'); primaryButtons.forEach(btn => { btn.style.backgroundColor = color; }); } // 使用例 document.getElementById('darkModeToggle').addEventListener('click', toggleDarkMode); document.getElementById('colorPicker').addEventListener('change', (e) => { changeThemeColor(e.target.value); });

Sassによる高度なカスタマイゼーション

Sass（サス）を使用すると、変数のオーバーライドや独自のユーティリティクラス生成など、より高度なカスタマイゼーションが可能になります。

// custom.scss // 1. Bootstrapの変数をインポート前にオーバーライド $primary: #6f42c1; $secondary: #6c757d; $font-family-sans-serif: 'Noto Sans JP', sans-serif; $font-size-base: 1rem; $line-height-base: 1.6; // ブレークポイントのカスタマイズ $grid-breakpoints: ( xs: 0, sm: 576px, md: 768px, lg: 992px, xl: 1200px, xxl: 1600px // 超大画面を1600pxに変更 ); // カスタムカラーパレットの追加 $custom-colors: ( "brand": #ff6b6b, "accent": #4ecdc4, "neutral": #95a5a6 ); // 2. case 'price-low': return this.getPrice(a) - this.getPrice(b); case 'price-high': return this.getPrice(b) - this.getPrice(a); default: return 0; } }); // DOM順序を更新 productsArray.forEach(product => { container.appendChild(product);
