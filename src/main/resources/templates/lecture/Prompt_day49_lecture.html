📚 Spring Security概念の分かりやすさ改善指針

❌ 改善前の問題例（認証・認可表）

問題：「Authentication」「Authorization」「Principal」など、専門用語のみで初心者には理解困難

✅ 改善後の指針

Spring Security概念の説明強化

  • Authentication → 「認証（誰であるかの確認）」
  • Authorization → 「認可（何ができるかの許可）」
  • Principal → 「認証主体（ログインユーザー情報）」
  • SecurityContext → 「セキュリティ文脈（認証状態の保持）」

具体例の追加

  • 「認証」→「パスワードでログイン、身分証確認と同じ」
  • 「認可」→「管理者画面へのアクセス許可、VIP席への入場許可と同じ」
  • 「JWT」→「デジタル身分証明書、期限付き入場券のようなもの」
