✅ 改善後の指針

例外分類の説明強化

  • • checked例外 → 「事前対処必須例外（コンパイル時に対処を強制）」
  • • unchecked例外 → 「実行時例外（プログラマーのミス）」
  • • Error → 「システムレベル問題（対処不可能）」

具体例の併記

  • • IOException: 「ファイル読み込み失敗」
  • • NullPointerException: 「nullオブジェクトへのアクセス」
  • • OutOfMemoryError: 「メモリ不足」

対処方法の明示

  • • checked: try-catch必須またはthrows宣言
  • • unchecked: 事前チェックで予防（null判定等）
  • • Error: 基本的に対処不可（システム設定見直し）
