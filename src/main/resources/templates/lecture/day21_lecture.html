1. 計算量とオーダー記法

計算量（Computational Complexity）とは

計算量とは、アルゴリズムが問題を解くのに必要な時間やメモリの量を表す指標です。 データの量が増えたときにアルゴリズムの処理時間がどの程度増加するかを予測するために使用します。

身近な例で理解する計算量

📚 辞書で単語を探す場合：

  • • 最初から順番に探す → O(n) 線形時間
  • • 目次を使って効率的に探す → O(log n) 対数時間
  • • ページ番号を知っている場合 → O(1) 定数時間

主要なオーダー記法（Big O Notation）

記法 名称 性能評価 具体例 データ数1000件での処理回数 実例
O(1) 定数時間 最高 配列の要素取得 1回 array[5]で5番目の要素を取得
O(log n) 対数時間 優秀 二分探索 約10回 ソート済み配列から目的の要素を探索
O(n) 線形時間 普通 線形探索 1000回 配列の全要素を順番にチェック
O(n log n) 線形対数時間 良好 マージソート 約10000回 効率的なソートアルゴリズム
O(n²) 二次時間 注意 バブルソート 1000000回 二重ループの処理
O(2ⁿ) 指数時間 危険 フィボナッチ数列（再帰） 無限に近い 組み合わせの全探索

計算量分析の実践例

例1: 配列の最大値を求める（O(n)）

public static int findMax(int[] array) {
    int max = array[0];           // 1回の処理
    for (int i = 1; i < array.length; i++) {  // n-1回の繰り返し
        if (array[i] > max) {     // 各回で1回の比較
            max = array[i];       // 最悪の場合に1回の代入
        }
    }
    return max;                   // 1回の処理
}
// 総処理回数: 1 + (n-1) + (n-1) + (n-1) + 1 ≈ 3n
// 最高次の項のみ考慮: O(n)

例2: 二重ループ（O(n²)）

public static void printPairs(int[] array) {
    for (int i = 0; i < array.length; i++) {      // n回の繰り返し
        for (int j = 0; j < array.length; j++) {  // 各iに対してn回の繰り返し
            System.out.println(array[i] + ", " + array[j]);  // 1回の処理
        }
    }
}
// 総処理回数: n × n × 1 = n²
// 計算量: O(n²)

2. ソートアルゴリズム

ソートアルゴリズムとは

ソート（Sort）とは、データを一定の規則（昇順・降順）に従って並び替える処理です。 データベースの検索、プログラムの効率化、データ分析など、あらゆる場面で使用される基本的なアルゴリズムです。

🫧 バブルソート（Bubble Sort）

仕組み：隣接する要素を比較して交換を繰り返す

計算量：O(n²) - 非効率だが理解しやすい

特徴：安定ソート、実装が簡単

配列: [64, 34, 25, 12]
1回目: [34, 64, 25, 12] → [34, 25, 64, 12] → [34, 25, 12, 64]
2回目: [25, 34, 12, 64] → [25, 12, 34, 64]
3回目: [12, 25, 34, 64] (完了)

🎯 選択ソート（Selection Sort）

仕組み：最小値を見つけて先頭に移動を繰り返す

計算量：O(n²) - 交換回数が少ない

特徴：不安定ソート、メモリ効率が良い

配列: [64, 25, 12, 22]
1回目: 最小値12を先頭へ → [12, 25, 64, 22]
2回目: 残りの最小値22を2番目へ → [12, 22, 64, 25]
3回目: 残りの最小値25を3番目へ → [12, 22, 25, 64]

📥 挿入ソート（Insertion Sort）

仕組み：各要素を適切な位置に挿入していく

計算量：O(n²) - 部分的にソート済みの場合は高速

特徴：安定ソート、オンラインアルゴリズム

配列: [12, 11, 13, 5]
1回目: [11, 12, 13, 5]  (11を適切な位置に挿入)
2回目: [11, 12, 13, 5]  (13は既に正しい位置)
3回目: [5, 11, 12, 13]  (5を最初に挿入)

🔀 マージソート（Merge Sort）

仕組み：分割統治法により配列を分割→ソート→結合

計算量：O(n log n) - 常に安定した性能

特徴：安定ソート、大量データに適している

配列: [38, 27, 43, 3]
分割: [38, 27] [43, 3]
さらに分割: [38] [27] [43] [3]
結合: [27, 38] [3, 43]
最終結合: [3, 27, 38, 43]

ソートアルゴリズム性能比較

アルゴリズム 最良の場合 平均的な場合 最悪の場合 メモリ使用量 安定性 実用性
バブルソート O(n) O(n²) O(n²) O(1) 安定 学習用
選択ソート O(n²) O(n²) O(n²) O(1) 不安定 小データ向け
挿入ソート O(n) O(n²) O(n²) O(1) 安定 部分ソート済み
マージソート O(n log n) O(n log n) O(n log n) O(n) 安定 実用的
クイックソート O(n log n) O(n log n) O(n²) O(log n) 不安定 高速

3. 探索アルゴリズム

🔍 線形探索（Linear Search）

仕組み：先頭から順番に目的の要素を探す

計算量：O(n) - データ量に比例

適用条件：ソートされていない配列でも使用可能

Javaでの実装例

public static int linearSearch(int[] array, int target) {
    for (int i = 0; i < array.length; i++) {
        if (array[i] == target) {
            return i;  // 見つかった位置を返す
        }
    }
    return -1;  // 見つからない場合
}

// 使用例: [3, 7, 1, 9, 4] から 9 を探す
// → 0番目:3≠9, 1番目:7≠9, 2番目:1≠9, 3番目:9=9 → 結果: 3

実用例：電話帳を最初から順番にめくって名前を探す

⚡ 二分探索（Binary Search）

仕組み：中央値と比較して探索範囲を半分ずつ絞る

計算量：O(log n) - 非常に高速

適用条件：配列が事前にソートされている必要がある

Javaでの実装例

public static int binarySearch(int[] array, int target) {
    int left = 0;
    int right = array.length - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (array[mid] == target) {
            return mid;  // 見つかった
        } else if (array[mid] < target) {
            left = mid + 1;  // 右半分を探索
        } else {
            right = mid - 1; // 左半分を探索
        }
    }
    return -1;  // 見つからない
}

実用例：辞書で単語を探すときに、目次を使って効率的に探す方法

二分探索の動作例

ソート済み配列: [1, 3, 5, 7, 9, 11, 13, 15] から 7 を探す

ステップ1: left=0, right=7, mid=3 → array[3]=7 → 目標値発見！
結果: インデックス 3

別の例: 11 を探す場合
ステップ1: left=0, right=7, mid=3 → array[3]=7 < 11 → left=4
ステップ2: left=4, right=7, mid=5 → array[5]=11 → 目標値発見！
結果: インデックス 5

最大でも log2(8) = 3 回の比較で見つかる（線形探索は最大8回）

4. 木構造（Tree Structure）

木構造とは

木構造は、ノード（節点）と呼ばれる要素が階層的に結ばれたデータ構造です。 家系図やフォルダ構造のように、親子関係を持つデータを効率的に管理できます。

木構造の基本用語

  • ルート（根）：最上位のノード
  • 親ノード：上位のノード
  • 子ノード：下位のノード
  • 葉（リーフ）：子を持たないノード
  • 深さ：ルートからの距離
  • 高さ：木の最大深さ

二分木の例

        8     ← ルート
       / \
      3   10   ← 深さ1
     / \    \
    1   6    14 ← 深さ2（葉ノード）
       / \   /
      4   7 13  ← 深さ3（葉ノード）

🌲 二分探索木（Binary Search Tree）

特徴：各ノードで左の子 < 親 < 右の子 の関係を満たす

検索：O(log n) - 平均的な場合

挿入・削除：O(log n) - 平均的な場合

二分探索木の例

      8
     / \
    3   10
   / \    \
  1   6    14
     / \   /
    4   7 13

検索プロセス：9を探す場合

8 < 9 → 右へ → 10 > 9 → 左へ → 見つからない

📚 ヒープ（Heap）

特徴：親ノードが子ノードより大きい（最大ヒープ）

最大値取得：O(1) - ルートが最大値

挿入・削除：O(log n)

最大ヒープの例

     16    ← 最大値（ルート）
    /  \
   14   10
  /  \  / \
 8   7 9   3
/ \
2  4

用途：優先度付きキュー、ヒープソート

実例：タスクスケジューラ、最短経路探索

木構造の走査（Traversal）

前順走査（Pre-order）

ルート → 左 → 右

結果: 8, 3, 1, 6, 4, 7, 10, 14, 13

中順走査（In-order）

左 → ルート → 右

結果: 1, 3, 4, 6, 7, 8, 10, 13, 14

後順走査（Post-order）

左 → 右 → ルート

結果: 1, 4, 7, 6, 3, 13, 14, 10, 8

5. グラフ理論（Graph Theory）

グラフとは

グラフは、頂点（Vertex）と辺（Edge）からなるデータ構造です。 ネットワーク、道路、人間関係など、様々な関係性を表現できる汎用的な構造です。

グラフの基本用語

  • 頂点（Vertex/Node）：グラフの節点
  • 辺（Edge）：頂点同士を結ぶ線
  • 次数（Degree）：頂点から出る辺の数
  • 経路（Path）：頂点から頂点への道筋
  • サイクル：始点と終点が同じ経路

グラフの種類

  • 無向グラフ：辺に方向がない
  • 有向グラフ：辺に方向がある
  • 重み付きグラフ：辺にコスト値がある
  • 連結グラフ：全ての頂点が繋がっている
  • 完全グラフ：全ての頂点が直接接続

🔍 深さ優先探索（DFS: Depth-First Search）

仕組み：一つの経路を最後まで探索してからバックトラック

計算量：O(V + E) V:頂点数, E:辺数

実装：再帰またはスタックを使用

DFSの実装例（再帰）

public static void dfs(int[][] graph, boolean[] visited, int node) {
    visited[node] = true;
    System.out.print(node + " ");
    
    for (int i = 0; i < graph[node].length; i++) {
        if (graph[node][i] == 1 && !visited[i]) {
            dfs(graph, visited, i);  // 再帰的に探索
        }
    }
}

用途：迷路の探索、トポロジカルソート

📡 幅優先探索（BFS: Breadth-First Search）

仕組み：現在の階層を全て探索してから次の階層へ

計算量：O(V + E) V:頂点数, E:辺数

実装：キューを使用

BFSの実装例（キュー使用）

public static void bfs(int[][] graph, boolean[] visited, int start) {
    Queue<Integer> queue = new LinkedList<>();
    visited[start] = true;
    queue.offer(start);
    
    while (!queue.isEmpty()) {
        int node = queue.poll();
        System.out.print(node + " ");
        
        for (int i = 0; i < graph[node].length; i++) {
            if (graph[node][i] == 1 && !visited[i]) {
                visited[i] = true;
                queue.offer(i);
            }
        }
    }
}

用途：最短経路探索、レベル順探索

グラフの表現方法

隣接行列（Adjacency Matrix）

グラフ: A-B, A-C, B-C

  A B C
A 0 1 1
B 1 0 1  
C 1 1 0

メリット: 辺の存在確認がO(1)
デメリット: メモリ使用量がO(V²)

隣接リスト（Adjacency List）

グラフ: A-B, A-C, B-C

A: [B, C]
B: [A, C]
C: [A, B]

メリット: メモリ効率が良い O(V+E)
デメリット: 辺の存在確認がO(V)

演習問題

問題1: 計算量の分析

以下のコードの計算量を分析してください。

public static int findPairSum(int[] array, int target) {
    int count = 0;
    for (int i = 0; i < array.length; i++) {
        for (int j = i + 1; j < array.length; j++) {
            if (array[i] + array[j] == target) {
                count++;
            }
        }
    }
    return count;
}

解答と解説

計算量: O(n²)

分析過程:

  • 外側のループ: i が 0 から n-1 まで → n回
  • 内側のループ: j が i+1 から n-1 まで → 平均 n/2回
  • 総実行回数: n × (n/2) ≈ n²/2
  • 最高次の項のみ考慮: O(n²)

実用性: 配列のサイズが1000を超える場合は効率が悪い。ハッシュテーブルを使用すればO(n)で解決可能。

問題2: ソートアルゴリズムの選択

以下の状況で最適なソートアルゴリズムを選択し、理由を説明してください。

状況A: 10万件のデータを安定してソートしたい

状況B: ほぼソート済みの1000件のデータを効率的にソートしたい

状況C: メモリ使用量を最小限に抑えて小規模データをソートしたい

解答と解説

状況A: マージソート

理由: O(n log n)で安定した性能、安定ソート、大量データに適している

状況B: 挿入ソート

理由: 部分的にソート済みの場合はO(n)に近い性能、オンラインアルゴリズム

状況C: 選択ソート

理由: メモリ使用量O(1)、実装が簡単、小規模データなら性能も問題なし

問題3: 二分探索木の操作

以下の二分探索木に対して、値15を挿入する過程を説明してください。

      8
     / \
    3   10
   / \    \
  1   6    14
     / \
    4   7

解答と解説

挿入過程:

  1. ルート8と比較: 15 > 8 → 右の子(10)へ
  2. 10と比較: 15 > 10 → 右の子(14)へ
  3. 14と比較: 15 > 14 → 右の子がないので、14の右の子として挿入
挿入後の木構造:
      8
     / \
    3   10
   / \    \
  1   6    14
     / \     \
    4   7     15  ← 新しく挿入

計算量: 平均的にO(log n)、最悪の場合O(n)（木が一直線になった場合）

理解度クイズ

クイズ1: 計算量に関する問題

1000個の要素を持つソート済み配列から特定の値を探索する場合、二分探索では最大何回の比較が必要でしょうか？

正解: A) 10回

解説:

  • 二分探索の最大比較回数は ⌈log₂(n)⌉ で計算される
  • 1000個の要素の場合: ⌈log₂(1000)⌉ = ⌈9.97⌉ = 10回
  • 毎回探索範囲を半分にするため、効率的に探索できる
  • 線形探索の場合は最大1000回必要だが、二分探索なら10回で済む

クイズ2: ソートアルゴリズムに関する問題

安定ソートの特徴として正しいものはどれでしょうか？

正解: B) 同じ値の要素の相対位置が保持される

解説:

  • 安定ソートは、同じ値を持つ要素の元の順序を保持する
  • 例: [3₁, 5, 3₂] → [3₁, 3₂, 5] （3₁が3₂より前に保持される）
  • 安定ソートの例: バブルソート、挿入ソート、マージソート
  • 不安定ソートの例: 選択ソート、クイックソート

重要性: データベースの複数項目ソートや、すでにソート済みのデータを再ソートする際に重要

クイズ3: グラフ探索に関する問題

深さ優先探索（DFS）と幅優先探索（BFS）の違いとして正しいものはどれでしょうか？

正解: A) DFSはスタック、BFSはキューを使用する

解説:

  • DFS（深さ優先探索）: スタック（LIFO）を使用、深く探索してからバックトラック
  • BFS（幅優先探索）: キュー（FIFO）を使用、レベルごとに探索
  • 両者の計算量は同じO(V + E)
  • BFSは最短経路を保証するが、DFSは保証しない

用途の違い:

• DFS: 迷路探索、トポロジカルソート

• BFS: 最短経路探索、レベル順処理

アルゴリズム理解の促進方法

視覚的説明のコツ

  • • 具体的な数値: 抽象的な説明より実際の配列で動作を示す
  • • 段階的解説: アルゴリズムの各ステップを分解して説明
  • • 比較表活用: 複数のアルゴリズムを同時に比較
  • • 身近な例: 辞書検索、カード整理等で直感的理解を促進

実践的指導法

  • • 手動トレース: 学生に手でアルゴリズムを実行させる
  • • 計算量予測: 実行前に計算量を予想させる
  • • コード読解: 実装コードから動作を推測させる
  • • 場面設定: 実務での活用場面を具体的に提示

よくある躓きポイントと対策

計算量の理解不足

問題: O記法の意味が抽象的で理解困難

対策: 具体的なデータ量（100件、1000件、10000件）での実行回数を計算させ、実感を持たせる

アルゴリズムの選択基準

問題: どのアルゴリズムを使うべきか判断できない

対策: データの特性（サイズ、ソート状況、メモリ制約）と適用場面を関連付けて教える

木構造の概念理解

問題: 階層構造の抽象的概念が理解困難

対策: 家系図、組織図、フォルダ構造など身近な階層構造から説明を始める
