1. コレクションフレームワークとは

コレクションフレームワーク（Collection Framework）

複数のデータを効率的に管理するJavaの統一された仕組み。配列の制限（固定サイズ、型安全性の問題）を解決し、動的なデータ管理を可能にします。

配列 vs コレクション比較

項目 配列 コレクション
サイズ 固定（int[5]は5個固定） 動的拡張可能
型安全性 基本的（プリミティブ型のみ） ジェネリクスで完全保証
機能 基本操作のみ 豊富なメソッド（sort, filter等）

重要ポイント

  • • Collection: 複数要素を扱う共通インターフェース（add, remove, size等）
  • • Map: キー・値ペアを扱う独立インターフェース（Collectionを継承しない）
  • • 実装クラス: 用途に応じて最適な内部構造を提供

日常例による理解

• ArrayList = 図書館の本棚
番号で本を直接取れるが、途中に本を挿入すると後ろの本をすべて移動させる必要がある

• LinkedList = 電車の車両
隣の車両にしか移動できないが、車両の連結・切り離しは簡単

• HashSet = 名簿
同じ名前は一度しか載らず、特定の人がいるかすぐに確認できる

• HashMap = 電話帳
名前（キー）から電話番号（値）をすぐに調べられる

4. Stream API詳細解説

Stream API（ストリーム API）とは

コレクションの要素を宣言的に処理するJava 8以降の機能。「何をしたいか」を記述するだけで、「どうやるか」はStream APIが自動処理します。

従来のループ処理

List<String> names = Arrays.asList("田中", "佐藤", "鈴木");
List<String> result = new ArrayList<>();

// 手続き的記述（どうやるかを詳細に指定）
for (String name : names) {
    if (name.length() == 2) {
        result.add(name.toUpperCase());
    }
}
Collections.sort(result);

Stream API使用

List<String> names = Arrays.asList("田中", "佐藤", "鈴木");

// 宣言的記述（何をしたいかを記述）
List<String> result = names.stream()
    .filter(name -> name.length() == 2)  // フィルタリング
    .map(String::toUpperCase)            // 変換
    .sorted()                            // ソート
    .collect(Collectors.toList());       // 収集

主要なStream操作

操作 説明 実例
filter 条件に合う要素のみ抽出 .filter(x -> x > 10)
map 各要素を変換 .map(String::toUpperCase)
sorted 要素をソート .sorted()
collect 結果を収集 .collect(Collectors.toList())
groupingBy 条件でグループ化 .collect(groupingBy(Person::getAge))

実務でよく使うパターン

1. データ集計: 
// 部署ごとの平均年齢
Map<String, Double> avgAgeByDept = employees.stream()
    .collect(groupingBy(Employee::getDepartment, 
             averagingInt(Employee::getAge)));
2. 条件フィルタリング: 
// 30歳以上の営業部員
List<Employee> salesOver30 = employees.stream()
    .filter(e -> e.getAge() >= 30)
    .filter(e -> "営業部".equals(e.getDepartment()))
    .collect(toList());
3. データ変換: 
// 社員名のリストを作成
List<String> names = employees.stream()
    .map(Employee::getName)
    .sorted()
    .collect(toList());

📝 演習問題

演習1: ArrayList vs LinkedList 性能比較

以下のコードを実行して、ArrayListとLinkedListの性能差を確認してください。 

import java.util.*;

public class PerformanceTest {
    public static void main(String[] args) {
        int size = 100000;
        
        // ArrayList テスト
        List<Integer> arrayList = new ArrayList<>();
        long start = System.currentTimeMillis();
        for (int i = 0; i < size; i++) {
            arrayList.add(0, i); // 先頭に挿入
        }
        long arrayTime = System.currentTimeMillis() - start;
        
        // LinkedList テスト
        List<Integer> linkedList = new LinkedList<>();
        start = System.currentTimeMillis();
        for (int i = 0; i < size; i++) {
            linkedList.add(0, i); // 先頭に挿入
        }
        long linkedTime = System.currentTimeMillis() - start;
        
        System.out.println("ArrayList: " + arrayTime + "ms");
        System.out.println("LinkedList: " + linkedTime + "ms");
    }
}

予想結果と解説

予想結果: LinkedListの方が圧倒的に高速（ArrayListは数秒、LinkedListは数十ミリ秒）

理由:

  • ArrayList: 先頭挿入時、既存要素をすべて後ろにシフトする必要がある（O(n)）
  • LinkedList: 先頭ノードのポインタを変更するだけ（O(1)）
  • 実務への応用: 先頭・末尾の操作が多い場合はLinkedListを選択

演習2: HashSetによる重複除去処理

以下のデータから重複を除去し、アルファベット順でソートしてください。 

String[] cities = {"東京", "大阪", "名古屋", "東京", "福岡", "大阪", "札幌", "名古屋"};

// 1. HashSetを使って重複除去
// 2. TreeSetまたはStream APIでソート
// 3. 結果を表示

解答例

// 解答例1: HashSet + ArrayList
Set<String> uniqueCities = new HashSet<>(Arrays.asList(cities));
List<String> sortedList = new ArrayList<>(uniqueCities);
Collections.sort(sortedList);
System.out.println("HashSet使用: " + sortedList);

// 解答例2: TreeSet（自動ソート）
Set<String> sortedSet = new TreeSet<>(Arrays.asList(cities));
System.out.println("TreeSet使用: " + sortedSet);

// 解答例3: Stream API
List<String> result = Arrays.stream(cities)
    .distinct()  // 重複除去
    .sorted()    // ソート
    .collect(Collectors.toList());
System.out.println("Stream使用: " + result);

出力結果: [名古屋, 大阪, 東京, 札幌, 福岡]

ポイント: HashSetで高速重複除去、TreeSetで自動ソート、Stream APIで宣言的記述

演習3: Stream APIによる複雑なデータ変換

以下の社員データから、30歳以上の社員を部署別にグループ化し、各部署の平均年齢を求めてください。 

class Employee {
    private String name;
    private int age;
    private String department;
    
    // コンストラクタ、getter省略
}

List<Employee> employees = Arrays.asList(
    new Employee("田中", 25, "営業部"),
    new Employee("佐藤", 32, "営業部"),
    new Employee("鈴木", 28, "開発部"),
    new Employee("高橋", 35, "開発部"),
    new Employee("山田", 31, "営業部"),
    new Employee("小林", 29, "開発部")
);

解答例

// 30歳以上の社員を部署別にグループ化し、平均年齢を計算
Map<String, Double> avgAgeByDept = employees.stream()
    .filter(emp -> emp.getAge() >= 30)           // 30歳以上でフィルタ
    .collect(Collectors.groupingBy(              // 部署別にグループ化
        Employee::getDepartment,                 // グループ化のキー
        Collectors.averagingInt(Employee::getAge) // 平均年齢計算
    ));

// 結果表示
avgAgeByDept.forEach((dept, avgAge) -> 
    System.out.printf("%s: %.1f歳%n", dept, avgAge));

// 30歳以上の社員一覧も表示
List<String> over30Names = employees.stream()
    .filter(emp -> emp.getAge() >= 30)
    .map(Employee::getName)
    .sorted()
    .collect(Collectors.toList());
System.out.println("30歳以上の社員: " + over30Names);

出力結果:

  • 営業部: 31.5歳 （佐藤32歳、山田31歳の平均）
  • 開発部: 35.0歳 （高橋35歳のみ）
  • 30歳以上の社員: [佐藤, 山田, 高橋]

🤔 理解度クイズ

Q1: コレクション選択問題

以下の要件に最も適したコレクションを選択してください：
「重複を許可せず、常にソート順を維持したい」

正解: C) TreeSet

解説:

  • TreeSet: 重複不可 + 自動ソート機能 = 要件に完全一致
  • ArrayList: 重複可能（×）
  • HashSet: ソート順なし（×）
  • HashMap: キー・値ペア用（×）

実用例: 学生の成績ランキング、価格帯別商品一覧など

Q2: Stream API理解問題

以下のコードの実行結果は何でしょうか？ 

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
long count = numbers.stream()
    .filter(n -> n % 2 == 1)
    .map(n -> n * 2)
    .count();
System.out.println(count);

正解: A) 3

処理の流れ:

  1. 元データ: [1, 2, 3, 4, 5]
  2. filter(n → n % 2 == 1): 奇数のみ → [1, 3, 5]
  3. map(n → n * 2): 2倍に → [2, 6, 10]
  4. count(): 要素数をカウント → 3

ポイント: map()は要素を変換するが、count()は要素数を返すため、変換後の値は関係なし

Q3: JavaSilver試験対策

以下のコードは何を出力するでしょうか？ 

Set<String> set = new HashSet<>();
set.add("B");
set.add("A");
set.add("C");
set.add("A"); // 重複
System.out.println(set);

正解: D) 順序は保証されない

解説:

  • HashSet: 重複を自動除去するため、"A"は1個のみ
  • 順序保証なし: ハッシュ値で管理するため、挿入順序は保持されない
  • 実際の出力例: [A, C, B] や [C, A, B] など様々
  • 順序が必要なら: LinkedHashSet（挿入順）またはTreeSet（ソート順）を使用

試験のポイント: HashSetは順序保証なしがよく出題される

  • • ArrayList: インデックス検索が高速（O(1)）で、商品一覧表示に最適

  • • 性能特性理解: O(1), O(n), O(log n)の実用的な意味を把握
  • • 適切な選択: 用途に応じた最適なコレクション選択能力
  • • Stream API: 宣言的記述で複雑なデータ変換が簡潔に記述可能
  • • 実用パターン: groupingBy, filtering, mapping の実務での活用

🏆 JavaSilver試験対策ポイント

頻出問題:

  • 各コレクションの特徴（重複可否、順序保証）
  • Map はCollection を継承しない
  • HashSet の順序保証なし特性
  • Stream API の中間操作と終端操作

記憶のコツ:

  • 「Hash = 高速、Tree = ソート、Linked = 順序保持」
  • 「List = 重複OK、Set = 重複NG」
  • 「Stream は水の流れ = 一方向処理」
