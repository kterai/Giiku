✅ 改善後の指針

アルゴリズム名の説明強化

  • • バブルソート → 「バブルソート（泡のように大きい値が浮上する並び替え）」
  • • クイックソート → 「クイックソート（分割統治で高速並び替え）」
  • • 線形探索 → 「線形探索（先頭から順番に探す方法）」
  • • 二分探索 → 「二分探索（半分に分けて効率的に探す方法）」

計算量の説明強化

  • • O(1) → 「一定時間（データ量に関係なく高速）」
  • • O(n) → 「線形時間（データ数に比例、1000個なら1000回）」
  • • O(n²) → 「二乗時間（データ数の二乗、1000個なら1,000,000回）」
  • • O(log n) → 「対数時間（データ数が倍になっても処理時間は少し増えるだけ）」

適用場面の具体化

  • • 「小規模データ」→「100個以下の成績リスト、商品カタログ」
  • • 「大規模データ」→「10万件以上の顧客DB、在庫管理システム」
  • • 「リアルタイム処理」→「検索エンジン、ゲームのスコアランキング」
