2.1 2ヶ月間の学習総括

講師向け指導ポイント：第1ヶ月目と第2ヶ月目の学習内容を体系的に振り返り、受講者の成長を可視化する。具体的な成果物や習得スキルを列挙し、自信を持たせる。

第1ヶ月目の振り返り
  • • プログラミング基礎（Java、HTML/CSS、JavaScript）
  • • コンピュータ基礎（ハードウェア、ソフトウェア、OS）
  • • ネットワーク基礎（プロトコル、アーキテクチャ）
第2ヶ月目の振り返り
  • • データベース基礎・応用
  • • データ構造とアルゴリズム
  • • 情報セキュリティ、プロジェクトマネジメント

2.2 最終模擬試験の実施

講師向け指導ポイント：本番同様の環境で実施し、時間配分や心理的プレッシャーの管理方法も指導する。試験後は即座にフィードバックを行い、弱点を明確にする。

午前試験（80問・150分）

• テクノロジ系：50問（コンピュータシステム、ネットワーク、データベース、セキュリティ、アルゴリズム）

• マネジメント系：20問（プロジェクトマネジメント、サービスマネジメント、システム監査）

• ストラテジ系：10問（経営戦略、システム戦略、企業活動と法務）

午後試験（11問中5問選択・150分）

• 情報セキュリティ（必須）、データ構造・アルゴリズム（必須）

• ソフトウェア開発（Java/Python/C等から選択）

• システム開発・データベース（選択）

2.3 結果分析とフィードバック

講師向け指導ポイント：個人別の詳細分析結果を基に、具体的な改善策を提示する。強みは伸ばし、弱点は重点的に対策する方法を指導する。

分析項目
  • • 個人別・分野別の得点分布
  • • 正答率・誤答率の詳細分析
  • • 時間配分の適切性評価
  • • 弱点分野の特定と対策提案
フィードバック内容
  • • 詳細な解説（頻出問題・計算問題・読解問題）
  • • 時間短縮テクニック
  • • 個別の弱点に応じた学習方法指導
  • • 本番試験に向けた最終アドバイス

2.4 個人学習ポートフォリオの作成

講師向け指導ポイント：ポートフォリオ作成を通じて、学習成果を客観視させ、自己肯定感を高める。同時に、継続学習への動機づけを行う。

ポートフォリオ構成
  • • 2ヶ月間の学習記録と成長軌跡
  • • 習得したスキルと知識の整理
  • • 実習・演習プロジェクトの成果物
  • • 今後の学習目標と計画

2.5 第3ヶ月目の準備

講師向け指導ポイント：Webアプリケーション開発の魅力と実用性を伝え、次の段階への期待感を高める。基礎知識と実践開発の橋渡しを意識する。

Webアプリケーション開発の概要
  • • HTML/CSS/JavaScriptの実践応用
  • • フレームワーク（React/Vue.js等）の学習
  • • データベース連携とAPI開発
  • • プロジェクト型学習の進め方

演習問題・模擬試験要件

重要：演習・模擬試験の事前説明

演習問題・模擬試験に含まれる内容は、必ず講義内容で事前に説明してください。後述する全ての問題の答えとなる知識・技術・概念を講義パートで詳しく解説する必要があります。

演習問題（15問程度）

講師向け指導ポイント：2ヶ月間の学習内容を総復習できるよう、各分野から均等に出題。基礎から応用まで段階的な難易度設定で自信を段階的に構築する。

問題1-5: 2ヶ月間の学習内容総復習

  • • プログラミング基礎（Java、オブジェクト指向）
  • • データベース設計と正規化
  • • ネットワークとセキュリティ

問題6-10: 総合応用問題

  • • アルゴリズムと計算量解析
  • • プロジェクトマネジメント実践
  • • システム戦略と経営戦略

問題11-15: 実践問題

  • • 午前試験対策問題
  • • 午後試験対策問題
  • • 時間配分とテクニック

理解度確認テスト（最終模擬試験）

講師向け指導ポイント：本番同様の形式で実施し、実戦感覚を養う。制限時間を設けて緊張感のある環境で実施し、試験慣れを促進する。

午前試験（80問・150分）

基本レベル（50%）

基礎知識の確認

応用レベル（35%）

実践的な問題解決

発展レベル（15%）

複合的な応用問題

午後試験（11問中5問選択・150分）

• 必須問題：情報セキュリティ、アルゴリズム

• 選択問題：プログラミング、システム開発、データベース

評価基準

• 午前試験：60%以上（48問以上正解）

• 午後試験：60%以上

• 各分野：50%以上の理解度
