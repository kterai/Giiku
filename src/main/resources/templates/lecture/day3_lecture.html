重要ポイント

継承とポリモーフィズムはオブジェクト指向の核心概念です。JavaSilver試験でも頻出であり、実務では設計の質を大きく左右します。今日は基礎をしっかり固めましょう。

1. if文の基礎

if文（イフ文）は、条件によってプログラムの実行経路を変える最も基本的な制御構文です。 日常生活で「もし雨が降ったら傘を持つ」という判断をするように、プログラムでも条件によって処理を分岐させます。

基本構文

if (条件式) {
    // 条件が真（true）の場合に実行される処理
}

具体例：年齢による判定

int age = 20;
if (age >= 18) {
    System.out.println("成人です");
}

年齢が18歳以上なら「成人です」を表示

初心者が間違いやすいポイント

  • • 条件式の結果は必ずboolean型（true/false）
  • • C言語のように0/1では判定できない
  • • 波括弧{}は処理が1行でも付ける習慣を！

2. if-else文

if-else文は、条件が真の場合と偽の場合で異なる処理を実行する2分岐制御構文です。 「もし〜なら○○、そうでなければ××」という日常的な判断パターンをプログラムで表現できます。

基本構文

if (条件式) {
    // 条件が真（true）の場合の処理
} else {
    // 条件が偽（false）の場合の処理
}

具体例：合格/不合格判定

int score = 75;
if (score >= 60) {
    System.out.println("合格です");
} else {
    System.out.println("不合格です");
}

点数が60点以上なら「合格」、未満なら「不合格」を表示

JavaSilver試験のポイント

if-else文では、必ずどちらか一方の処理ブロックが実行されます。両方が実行されることも、両方とも実行されないこともありません。

3. else if文（多分岐）

else if文は、3つ以上の分岐処理を実装する際に使用します。 成績評価（A/B/C/D）や年齢層判定（子供/大人/高齢者）など、段階的な判定に適しています。

具体例：成績評価システム

int score = 85;
if (score >= 90) {
    System.out.println("優秀（A評価）");
} else if (score >= 80) {
    System.out.println("良好（B評価）");
} else if (score >= 70) {
    System.out.println("普通（C評価）");
} else if (score >= 60) {
    System.out.println("要努力（D評価）");
} else {
    System.out.println("不合格（F評価）");
}

重要な注意点

else if文は上から順番に評価されます。最初に真になった条件で処理が確定し、それ以降の条件は評価されません。

⚠️ 条件の順序が重要！「>= 90」を最後に書くと、正しく動作しません。

4. switch文

switch文は、1つの変数の値によって処理を分岐させる制御構文です。 曜日による営業時間変更、メニュー選択処理、エラーコード分岐など、値が決まっている場合の分岐に適しています。

基本構文

switch (式) {
    case 値1:
        // 処理1
        break;
    case 値2:
        // 処理2
        break;
    default:
        // デフォルト処理
        break;
}

具体例：曜日判定

int dayOfWeek = 3; // 1:月, 2:火, 3:水, 4:木, 5:金, 6:土, 7:日
switch (dayOfWeek) {
    case 1:
        System.out.println("月曜日 - 9:00-18:00営業");
        break;
    case 2:
        System.out.println("火曜日 - 9:00-18:00営業");
        break;
    case 3:
        System.out.println("水曜日 - 9:00-18:00営業");
        break;
    case 6:
    case 7:
        System.out.println("土日 - 10:00-17:00営業");
        break;
    default:
        System.out.println("不正な曜日です");
        break;
}

よくある間違い

break文の記述忘れは、JavaSilver試験でも実務でも頻出の間違いです。各case文の最後にbreak;を必ず確認しましょう。

  • • break文がないと次のcaseも実行される（フォールスルー）
  • • switch式で使える型は限定的（int, char, String, enum等）
  • • defaultは必須ではないが、通常は記述する

5. 三項演算子（条件演算子）

三項演算子（?:）は、簡単な条件分岐を1行で記述できる演算子です。 「条件 ? 真の場合の値 : 偽の場合の値」の形式で、値の選択に適しています。

基本構文

データ型 変数 = 条件式 ? 真の場合の値 : 偽の場合の値;

具体例：最大値選択

int a = 10, b = 20;
int max = a > b ? a : b;  // bが選ばれる（20）
System.out.println("最大値: " + max);

// if-else文で書いた場合の比較
int max2;
if (a > b) {
    max2 = a;
} else {
    max2 = b;
}

使い分けのコツ

  • • 簡単な条件＋値の選択 → 三項演算子
  • • 複数行の処理が必要 → if-else文
  • • 可読性を重視する場合 → if-else文

6. 論理演算子と複合条件

論理演算子は、複数の条件を組み合わせて複合条件を作る演算子です。 「年齢が18歳以上かつ運転免許を持っている」のような、AND・OR・NOTの論理演算を実現します。

論理演算子詳細一覧

演算子 読み方 意味 実用例 注意点
&& アンド 論理AND（両方が真） 年齢AND免許（age >= 18 && hasLicense） 短絡評価あり
|| オア 論理OR（どちらかが真） 平日OR祝日（isWeekday || isHoliday） 短絡評価あり
! ノット 論理NOT（否定） 未ログイン（!isLoggedIn） 単項演算子

具体例：複合条件での認証処理

int age = 20;
boolean hasLicense = true;
boolean isExperienced = false;

// 運転可能条件：18歳以上 かつ 免許あり
if (age >= 18 && hasLicense) {
    System.out.println("運転可能です");
}

// レンタカー利用条件：運転可能 かつ 経験者
if ((age >= 18 && hasLicense) && isExperienced) {
    System.out.println("レンタカー利用可能");
} else {
    System.out.println("レンタカー利用不可");
}

短絡評価（ショートサーキット）の重要性

int a = 5, b = 0;
// 短絡評価により、b != 0 が false なので a/b は実行されない
if (b != 0 && a/b > 2) {
    System.out.println("OK");
}
// エラーにならない！

&&演算子は左側が偽なら右側を評価しない。||演算子は左側が真なら右側を評価しない。

演習問題

問題1: 成績判定システム

点数に応じて以下の判定を行うプログラムを作成してください。

  • • 90点以上: "優秀"
  • • 80点以上90点未満: "良好"
  • • 70点以上80点未満: "普通"
  • • 60点以上70点未満: "要努力"
  • • 60点未満: "不合格"
int score = 85; // この値を変更してテスト
// ここにif-else if文を記述

回答例:

int score = 85;
if (score >= 90) {
    System.out.println("優秀");
} else if (score >= 80) {
    System.out.println("良好");
} else if (score >= 70) {
    System.out.println("普通");
} else if (score >= 60) {
    System.out.println("要努力");
} else {
    System.out.println("不合格");
}
実行結果: score = 85 の場合、「良好」が表示されます。
ポイント: 条件は上から順に評価されるため、>= 90, >= 80... の順序が重要です。

問題2: 曜日判定（switch文）

数値（1-7）を曜日名に変換し、平日/休日も判定してください。 

int dayNum = 3; // 1:月, 2:火, ..., 7:日
// switch文を使用して曜日名と平日/休日を表示

回答例:

int dayNum = 3;
switch (dayNum) {
    case 1:
        System.out.println("月曜日（平日）");
        break;
    case 2:
        System.out.println("火曜日（平日）");
        break;
    case 3:
        System.out.println("水曜日（平日）");
        break;
    case 4:
        System.out.println("木曜日（平日）");
        break;
    case 5:
        System.out.println("金曜日（平日）");
        break;
    case 6:
        System.out.println("土曜日（休日）");
        break;
    case 7:
        System.out.println("日曜日（休日）");
        break;
    default:
        System.out.println("不正な曜日番号です");
        break;
}
実行結果: dayNum = 3 の場合、「水曜日（平日）」が表示されます。
ポイント: 各caseの最後にbreak文を忘れずに記述することが重要です。

問題3: 三項演算子の活用

以下の条件を三項演算子で記述してください。

  • • 年齢が18歳以上なら"成人"、未満なら"未成年"
  • • 数値が偶数なら"even"、奇数なら"odd"
  • • 2つの数値の大きい方を取得

回答例:

// 1. 年齢判定
int age = 20;
String ageCategory = age >= 18 ? "成人" : "未成年";
System.out.println(ageCategory); // "成人"

// 2. 偶数・奇数判定
int number = 7;
String evenOdd = number % 2 == 0 ? "even" : "odd";
System.out.println(evenOdd); // "odd"

// 3. 最大値取得
int a = 15, b = 10;
int max = a > b ? a : b;
System.out.println("最大値: " + max); // "最大値: 15"
ポイント: 三項演算子は簡潔ですが、複雑な条件の場合はif-else文の方が可読性が高くなります。
使い分け: 単純な値の選択には三項演算子、複雑な処理にはif-else文を使用しましょう。

問題4: 複合条件の論理演算子

以下の条件を満たすログイン認証システムを作成してください。

  • • ユーザー名が"admin"かつパスワードが"password"の場合：管理者ログイン
  • • ユーザー名が"user"かつパスワードが"user123"の場合：一般ユーザーログイン
  • • それ以外：ログイン失敗

回答例:

String username = "admin";
String password = "password";

if (username.equals("admin") && password.equals("password")) {
    System.out.println("管理者としてログインしました");
} else if (username.equals("user") && password.equals("user123")) {
    System.out.println("一般ユーザーとしてログインしました");
} else {
    System.out.println("ログインに失敗しました");
}
実行結果: username = "admin", password = "password" の場合、「管理者としてログインしました」が表示されます。
重要: String比較では == ではなく .equals() メソッドを使用することが必須です。
論理演算子: && (AND) は両方の条件が真の場合のみ全体が真になります。

問題5: フォールスルー現象の理解

以下のswitch文の実行結果を予測し、その理由を説明してください。 

int grade = 2;
switch (grade) {
    case 1:
        System.out.println("1年生");
    case 2:
        System.out.println("2年生");
    case 3:
        System.out.println("3年生");
        break;
    default:
        System.out.println("その他");
}

実行結果:

2年生
3年生
理由: case 2でbreak文がないため、フォールスルー現象が発生します。
grade = 2なので case 2から実行が開始され、break文がないため case 3も実行されます。
case 3にbreak文があるため、そこで実行が終了します。

修正版: 
case 2:
    System.out.println("2年生");
    break; // この行を追加

理解度クイズ

クイズ1: switch文のフォールスルー

int x = 2;
switch (x) {
    case 1:
        System.out.print("A");
    case 2:
        System.out.print("B");
    case 3:
        System.out.print("C");
        break;
    default:
        System.out.print("D");
}

上記コードの出力結果は？

正解: BC

解説: x = 2なので case 2から実行開始。break文がないため case 3も実行される。 case 3にbreak文があるため、そこで終了。結果として「B」と「C」が出力される。

クイズ2: 論理演算子の短絡評価

int a = 5, b = 0;
if (b != 0 && a/b > 2) {
    System.out.println("OK");
}

このコードは実行時エラーになるでしょうか？

正解: エラーにならない

解説: &&演算子は短絡評価を行います。左側の条件 (b != 0) が false なので、 右側の条件 (a/b > 2) は評価されません。そのため、ゼロ除算エラーは発生しません。

クイズ3: 三項演算子の優先順位

int x = 10, y = 20;
int result = x > y ? x + y : x - y;

変数resultの値は？

正解: -10

解説: x (10) > y (20) は false なので、「:」の後の式 x - y が実行されます。 10 - 20 = -10 となります。

クイズ4: String比較の注意点

String str1 = "hello";
String str2 = "hello";
String str3 = new String("hello");

System.out.println(str1 == str2);     // ?
System.out.println(str1 == str3);     // ?
System.out.println(str1.equals(str3)); // ?

それぞれの出力結果は？

正解: true, false, true

解説:
• str1 == str2: 文字列リテラルは同じ参照を共有するため true
• str1 == str3: new String()で作成したオブジェクトは異なる参照のため false
• str1.equals(str3): equals()は内容を比較するため true
重要: String比較では必ず equals() を使用しましょう。

クイズ5: 代入と比較の区別

int x = 5;
if (x = 10) {  // 注意：= が1つ
    System.out.println("TRUE");
} else {
    System.out.println("FALSE");
}

このコードはコンパイルできるでしょうか？

正解: コンパイルエラー

解説: if文の条件式にはboolean型が必要ですが、x = 10 は代入式でint型を返します。 Javaでは自動的にint型をboolean型に変換できないため、コンパイルエラーになります。
正しい記述: if (x == 10) { ... }
