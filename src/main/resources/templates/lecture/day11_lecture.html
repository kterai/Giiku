メイン講義

📋 配列の課題とコレクションの利点

💡 なぜコレクションが必要なのか？

配列には以下のような制限があります：

  • サイズ固定 - 一度作成すると要素数を変更できない
  • 型制限 - 同じ型の要素しか格納できない
  • 機能不足 - 検索・ソート・重複除去などの便利メソッドがない

コレクションフレームワークは、これらの問題を解決し、柔軟で高機能なデータ管理を可能にします。

🏠 身近な例で理解しよう
  • 配列 = 決まった数の引き出しがある本棚（後から引き出しを増やせない）
  • ArrayList = 必要に応じて棚を増設できる本棚
  • HashSet = 同じ本は一冊しか置かない特別な本棚
  • HashMap = 番号札で本を管理する図書館システム

🏗️ コレクションフレームワークの全体像

📊 Collection階層
  • List - 順序あり、重複可能（ArrayList、LinkedList）
  • Set - 順序任意、重複不可（HashSet、TreeSet）
  • Queue - 先入先出（FIFO）の待ち行列
🗝️ Map階層
  • HashMap - 高速検索、順序なし
  • TreeMap - 自動ソート、順序あり
  • LinkedHashMap - 挿入順序保持
🔍 Collection と Map の違い
  • Collection - 単一の値を管理（例：名前のリスト）
  • Map - キーと値のペアを管理（例：学籍番号と名前の対応）

📝 List と Set

ArrayList vs LinkedList
ArrayList（配列リスト）
  • 内部的に配列で管理
  • インデックスでの高速アクセス
  • 末尾への追加は高速
  • 途中への挿入・削除は低速
  • 用途：頻繁な参照、少ない更新
LinkedList（連結リスト）
  • 内部的にノードを連結して管理
  • インデックスアクセスは低速
  • 任意位置への挿入・削除は高速
  • メモリ使用量が多い
  • 用途：頻繁な挿入・削除
💻 Listの基本コード例
// ArrayListの基本操作
List<String> names = new ArrayList<>();
names.add("Alice");        // 末尾に追加
names.add("Bob");
names.add(1, "Charlie");   // インデックス1に挿入
Collections.sort(names);   // ソート

// 結果：[Alice, Bob, Charlie] → [Alice, Bob, Charlie]
System.out.println(names);

// 要素アクセス
String first = names.get(0);       // インデックスでアクセス
boolean hasAlice = names.contains("Alice");  // 要素の存在確認
HashSet vs TreeSet
HashSet（ハッシュ集合）
  • ハッシュテーブルで管理
  • 順序は保証されない
  • 高速な重複チェック
  • add、contains、removeが高速
  • 用途：重複除去、高速な存在確認
TreeSet（木構造集合）
  • 二分探索木で管理
  • 自動的にソートされる
  • 範囲検索が可能
  • 操作は少し重い
  • 用途：ソート済み集合、範囲検索

🗝️ Map

🔗 キーと値のペア管理

Mapは「キー」と「値」のペアを管理するデータ構造です。

  • キー - 一意の識別子（重複不可）
  • 値 - キーに対応するデータ（重複可）
  • 例 - 学籍番号（キー）と学生名（値）
HashMap
  • ハッシュテーブル実装
  • 順序は保証されない
  • null値を許可
  • 高速な検索・追加・削除
  • 用途：一般的なキー・値管理
TreeMap
  • 二分探索木実装
  • キーで自動ソート
  • null値は不可
  • 範囲検索が可能
  • 用途：ソートされたマップ
💻 Mapの基本コード例
// HashMapの基本操作
Map<Integer, String> idToName = new HashMap<>();
idToName.put(1, "Alice");      // キーと値を登録
idToName.put(2, "Bob");
idToName.put(3, "Charlie");

// 値の取得
String name = idToName.get(1);  // "Alice"を取得
boolean hasKey = idToName.containsKey(2);  // キーの存在確認
boolean hasValue = idToName.containsValue("Bob");  // 値の存在確認

// 全要素の反復処理
for (Map.Entry<Integer, String> entry : idToName.entrySet()) {
    System.out.println("ID: " + entry.getKey() + ", Name: " + entry.getValue());
}

⚡ 基本操作まとめ

📥 追加・更新操作
  • List: add(element), add(index, element)
  • Set: add(element) - 重複は無視
  • Map: put(key, value) - 既存キーは上書き
🔍 検索操作
  • List: contains(element), get(index)
  • Set: contains(element)
  • Map: get(key), containsKey(key)
🗑️ 削除操作
  • List: remove(element), remove(index)
  • Set: remove(element)
  • Map: remove(key)
🔄 反復処理
  • 拡張for文: for(Type elem : collection)
  • Iterator: iterator.hasNext(), iterator.next()
  • Map: entrySet(), keySet(), values()

🚀 高速 O(1)

要素数に関係なく一定時間で処理完了

例：HashMap での検索、ArrayList のインデックスアクセス

⚡ 中程度 O(log n)

要素数の対数に比例して処理時間が増加

例：TreeSet での検索（1000要素→約10回の比較）

🐌 低速 O(n)

要素数に比例して処理時間が増加

例：ArrayList での値検索（全要素をチェック）

💡 選択の指針
  • 頻繁にインデックスでアクセス → ArrayList
  • リストの途中への挿入・削除が多い → LinkedList
  • 重複を許さず高速な存在確認 → HashSet
  • 重複を許さずソート済みが必要 → TreeSet
  • キーと値の対応付けが必要 → HashMap
  • キーでソートされたマップが必要 → TreeMap

演習問題

💡 演習：ArrayListで学生名簿作成

問題：ArrayList<Student> で学生名簿を管理し、名前で検索する findStudent メソッドを実装せよ。

要件：
  • Student クラス（id、name、age フィールド）を作成
  • 学生を追加する addStudent メソッド
  • 名前で検索する findStudent メソッド（見つからない場合は null 返却）
  • 全学生を表示する displayAllStudents メソッド
✅ 解答例
// Student クラス
class Student {
    private int id;
    private String name;
    private int age;
    
    public Student(int id, String name, int age) {
        this.id = id;
        this.name = name;
        this.age = age;
    }
    
    // ゲッター
    public int getId() { return id; }
    public String getName() { return name; }
    public int getAge() { return age; }
    
    @Override
    public String toString() {
        return "Student{id=" + id + ", name='" + name + "', age=" + age + "}";
    }
}

// 学生名簿管理クラス
class StudentRegistry {
    private List<Student> students;
    
    public StudentRegistry() {
        this.students = new ArrayList<>();
    }
    
    // 学生追加
    public void addStudent(Student student) {
        students.add(student);
        System.out.println("学生を追加しました: " + student.getName());
    }
    
    // 名前で検索
    public Student findStudent(String name) {
        for (Student student : students) {
            if (student.getName().equals(name)) {
                return student;
            }
        }
        return null; // 見つからない場合
    }
    
    // 全学生表示
    public void displayAllStudents() {
        System.out.println("=== 学生名簿 ===");
        if (students.isEmpty()) {
            System.out.println("登録されている学生はいません。");
            return;
        }
        
        for (Student student : students) {
            System.out.println(student);
        }
    }
}

// 使用例
public class Main {
    public static void main(String[] args) {
        StudentRegistry registry = new StudentRegistry();
        
        // 学生追加
        registry.addStudent(new Student(1, "田中太郎", 20));
        registry.addStudent(new Student(2, "佐藤花子", 19));
        registry.addStudent(new Student(3, "鈴木一郎", 21));
        
        // 全学生表示
        registry.displayAllStudents();
        
        // 名前で検索
        Student found = registry.findStudent("佐藤花子");
        if (found != null) {
            System.out.println("見つかりました: " + found);
        } else {
            System.out.println("学生が見つかりません。");
        }
    }
}
📝 解説ポイント
  • ArrayList<Student> - ジェネリクスで型安全性を確保
  • 拡張for文 - コレクションの反復処理で推奨される書き方
  • equals メソッド - 文字列比較では == ではなく equals を使用
  • null チェック - 見つからない場合の適切な処理
  • toString メソッド - オブジェクトの文字列表現をカスタマイズ
🚀 発展課題
  • 学籍番号（ID）で検索する findStudentById メソッドを追加
  • 年齢範囲で検索する findStudentsByAgeRange メソッドを追加
  • HashSet を使って重複する学生（同じID）の登録を防ぐ
  • HashMap を使ってID検索を高速化する

理解度確認クイズ

Q1 ArrayList と LinkedList の挿入・削除性能に関する説明で正しいものは？

  • A ArrayList の挿入は常に O(1)
  • B LinkedList は挿入削除が高速
  • C 両者とも挿入に O(n) かかる
  • D LinkedList はランダムアクセスが高速

正解: B

解説:

LinkedList は連結リスト構造のため、要素の挿入・削除時に他の要素をシフトする必要がなく、先頭や末尾への操作が O(1) で実行できます。一方、ArrayList は配列ベースなので途中への挿入時に後続要素のシフトが必要（O(n)）です。

詳細比較:

  • ArrayList: 末尾追加はO(1)、途中挿入はO(n)、ランダムアクセスO(1)
  • LinkedList: 先頭・末尾の挿入削除はO(1)、ランダムアクセスO(n)

Q2 HashSet が要素の重複を許さない理由は？

  • A equals と hashCode により判定するから
  • B 追加前にソートするから
  • C 内部で木構造を使うから
  • D 容量を固定しているから

正解: A

解説:

HashSet は内部でハッシュテーブルを使用し、要素の追加時に hashCode() でハッシュ値を計算し、同じハッシュ値の要素がある場合は equals() で内容を比較して重複をチェックします。

重複チェックの流れ:

  1. 新しい要素の hashCode() を計算
  2. 同じハッシュ値を持つ要素があるかチェック
  3. ある場合は equals() で内容を比較
  4. equals() が true なら重複として追加しない

Q3 Map でキーの存在を確認するメソッドは？

  • A containsValue
  • B hasKey
  • C containsKey
  • D existsKey

正解: C

解説:

containsKey(key) メソッドは、指定したキーがマップに登録されているかを boolean で返します。値の存在確認には containsValue(value) を使用します。

Map の主要な確認メソッド:

  • containsKey(key): 指定キーの存在確認
  • containsValue(value): 指定値の存在確認
  • isEmpty(): マップが空かどうかの確認
  • size(): キー・値ペアの数を取得

Q4 for-each ループでコレクションを走査する基本構文は？

  • A for(int i = 0; i < list.size(); i++)
  • B for(String s : list){}
  • C while(iterator.hasNext())
  • D list.forEach()

正解: B

解説:

拡張for文（enhanced for loop）は Java 5 から導入された構文で、コレクションの全要素を簡潔に走査できます。インデックスが不要で、要素に直接アクセスできます。

反復処理の種類:

  • 拡張for文: for(Type elem : collection) - 最も簡潔
  • 通常のfor文: for(int i=0; i<size; i++) - インデックス必要時
  • Iterator: while(iter.hasNext()) - 削除操作時に安全
  • Stream API: collection.forEach() - Java 8以降の関数型

Q5 List から指定要素を削除するメソッドは？

  • A delete
  • B remove
  • C erase
  • D cut

正解: B

解説:

remove メソッドで要素を削除します。List では remove(Object) で値を指定した削除と、remove(int) でインデックスを指定した削除の2つがあります。

remove メソッドの種類:

  • remove(Object o): 指定した値と等しい最初の要素を削除
  • remove(int index): 指定したインデックスの要素を削除
  • removeAll(Collection c): 指定コレクションに含まれる全要素を削除
  • clear(): 全要素を削除
