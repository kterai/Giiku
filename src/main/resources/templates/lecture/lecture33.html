問題1: 基数変換

10進数の25を2進数で表したものはどれか。

正解: a) 11001

解説:

10進数25を2進数に変換するには、2で割り続けて余りを下から読みます。

25 ÷ 2 = 12 余り 1

12 ÷ 2 = 6 余り 0

6 ÷ 2 = 3 余り 0

3 ÷ 2 = 1 余り 1

1 ÷ 2 = 0 余り 1

下から読むと: 11001

検算: 1×16 + 1×8 + 0×4 + 0×2 + 1×1 = 16 + 8 + 1 = 25 ✓

問題2: データ構造

スタック（stack）の特徴として正しいものはどれか。

正解: b) 最後に入れたデータを最初に取り出す（LIFO）

解説:

スタックは「LIFO（Last In, First Out）」の原則で動作するデータ構造です。本を積み重ねるイメージで、最後に置いた本を最初に取ることになります。

各選択肢の説明:

a) FIFO（First In, First Out）はキュー（queue）の特徴

b) 正解。スタックの基本的な動作原理

c) ランダムアクセスは配列の特徴

d) 優先度による取り出しは優先度付きキューの特徴

スタックの主な操作：push（データの追加）、pop（データの取り出し）

問題3: データベース

関係データベースの正規化に関する説明として適切なものはどれか。

正解: b) データの冗長性や矛盾を排除してデータの整合性を保つ

解説:

データベースの正規化は、データの冗長性（重複）や矛盾を取り除き、データの整合性を保つための技法です。

正規化の段階:

• 第1正規形（1NF）: 繰り返し項目を排除

• 第2正規形（2NF）: 部分関数従属を排除

• 第3正規形（3NF）: 推移関数従属を排除

各選択肢の説明:

a) 正規化は冗長性を「減らす」ことが目的

b) 正解。正規化の基本的な目的

c) 通常は適切に分割してテーブル数は増える

d) これはパフォーマンスチューニングの手法

問題4: ネットワーク

OSI参照モデルの第4層（トランスポート層）の役割として適切なものはどれか。

正解: d) エンドツーエンドの信頼性ある通信を提供する

解説:

OSI参照モデルの第4層（トランスポート層）は、送信側から受信側までのエンドツーエンドの信頼性ある通信を提供します。

OSI参照モデル7層:

第7層: アプリケーション層

第6層: プレゼンテーション層

第5層: セッション層

第4層: トランスポート層（TCP/UDP）

第3層: ネットワーク層（IP）

第2層: データリンク層

第1層: 物理層

各選択肢の説明:

a) 物理層（第1層）の役割

b) データリンク層（第2層）の役割

c) ネットワーク層（第3層）の役割

d) 正解。トランスポート層の主要な役割

問題5: セキュリティ

公開鍵暗号方式の特徴として正しいものはどれか。

正解: b) 暗号化と復号化に異なる鍵を使用する

解説:

公開鍵暗号方式では、暗号化用の鍵（公開鍵）と復号化用の鍵（秘密鍵）が異なります。これにより、鍵の配送問題を解決できます。

公開鍵暗号方式の仕組み:

1. 受信者が公開鍵と秘密鍵のペアを生成

2. 公開鍵を送信者に公開

3. 送信者が公開鍵でメッセージを暗号化

4. 受信者が秘密鍵でメッセージを復号化

代表的なアルゴリズム:

• RSA暗号

• 楕円曲線暗号（ECC）

• ElGamal暗号

共通鍵暗号方式（a）との違い：鍵の配送が不要で、大規模システムに適している

問題6: プロジェクトマネジメント

WBS（Work Breakdown Structure）の目的として適切なものはどれか。

正解: b) プロジェクトの作業を階層的に分解し、管理可能な単位にする

解説:

WBS（Work Breakdown Structure：作業分解構造）は、プロジェクトの作業を階層的に分解し、管理しやすい単位（ワークパッケージ）に分割するためのツールです。

WBSの効果:

• 作業の全体像が明確になる

• 作業の漏れを防げる

• 工数見積もりが正確になる

• 責任分担が明確になる

• 進捗管理がしやすくなる

各選択肢の説明:

a) これはガントチャートの目的

b) 正解。WBSの基本的な目的

c) これはリスク分析の目的

d) これは品質計画の目的

問題7: 経営戦略

SWOT分析における「S」が表すものはどれか。

正解: a) Strength（強み）

解説:

SWOT分析は、組織の内部環境と外部環境を4つの要素で分析する経営戦略手法です。

SWOT分析の4要素:

• Strength（強み）- 内部環境のプラス要因

• Weakness（弱み）- 内部環境のマイナス要因

• Opportunity（機会）- 外部環境のプラス要因

• Threat（脅威）- 外部環境のマイナス要因

分析の流れ:

1. 内部環境分析（強み・弱み）

2. 外部環境分析（機会・脅威）

3. クロス分析による戦略立案

4. 戦略の優先順位付けと実行計画

問題8: システム開発

アジャイル開発手法の特徴として適切なものはどれか。

正解: b) 短い期間での反復開発を行い、顧客との対話を重視する

解説:

アジャイル開発は、短い期間（スプリント）での反復開発を行い、顧客との継続的な対話を通じて価値のあるソフトウェアを早期に提供する開発手法です。

アジャイル開発の原則（アジャイル宣言）:

• プロセスやツールよりも個人と対話を

• 包括的なドキュメントよりも動くソフトウェアを

• 契約交渉よりも顧客との協調を

• 計画に従うことよりも変化への対応を

代表的なアジャイル手法:

• スクラム（Scrum）

• エクストリームプログラミング（XP）

• カンバン（Kanban）

ウォーターフォール開発とは対照的に、変化に柔軟に対応できる開発手法です。

演習問題

演習1: 計算量とアルゴリズム

以下のソートアルゴリズムのうち、最悪時間計算量がO(n²)でないものはどれか。

正解: c) マージソート

詳細解説:

マージソートは分割統治法を用いるソートアルゴリズムで、常にO(n log n)の時間計算量を持ちます。

各アルゴリズムの時間計算量:

• バブルソート: 最良O(n)、平均O(n²)、最悪O(n²)

• 選択ソート: 常にO(n²)

• マージソート: 常にO(n log n)

• 挿入ソート: 最良O(n)、平均O(n²)、最悪O(n²)

マージソートは安定ソートでもあり、大量のデータの処理に適しています。

演習2: データベース設計

関係データベースにおいて、主キーの制約として正しいものはどれか。

正解: c) 主キーの値は一意であり、NULLは許可されない

詳細解説:

主キー（Primary Key）は、テーブル内の各行を一意に識別するための制約です。

主キーの制約:

• 一意性制約: 同じ値を持つ行は存在できない

• NOT NULL制約: NULL値は許可されない

• 各テーブルに1つだけ定義可能

• 自動的にインデックスが作成される

主キーの種類:

• 自然キー: 業務上の意味を持つ値（商品コードなど）

• 代理キー: システムが自動生成する値（通し番号など）

• 複合キー: 複数のカラムの組み合わせ

演習3: セキュリティ対策

SQLインジェクション攻撃を防ぐために最も効果的な対策はどれか。

正解: b) プリペアードステートメント（パラメータ化クエリ）を使用する

詳細解説:

SQLインジェクション攻撃は、Webアプリケーションの入力値検証が不十分な場合に発生する攻撃です。プリペアードステートメントが最も効果的な対策です。

SQLインジェクション攻撃の仕組み:

1. 攻撃者が悪意のあるSQL文を入力値に混入

2. アプリケーションが入力値をそのままSQL文に結合

3. 意図しないSQL文が実行される

4. データベースの不正操作が可能になる

対策方法:

• プリペアードステートメント（最重要）

• 入力値検証とサニタイジング

• 最小権限の原則

• WAF（Web Application Firewall）の導入
