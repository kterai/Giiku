✅ 改善後の指針

マイクロサービス概念の説明強化

  • Service Discovery → 「サービス発見（各サービスの場所を自動で見つける仕組み）」
  • Circuit Breaker → 「サーキットブレーカー（電気のブレーカーのように障害を遮断）」
  • API Gateway → 「APIゲートウェイ（すべてのAPIリクエストの入り口）」
  • Load Balancer → 「負荷分散（複数サーバーに処理を振り分け）」

具体例の追加

  • 「マイクロサービス」→「レゴブロックのように、小さな機能を組み合わせてアプリを作る」
  • 「Service Mesh」→「サービス間の通信を管理する高速道路網」
  • 「分散トレーシング」→「宅配便の追跡のように、リクエストの流れを追跡」
