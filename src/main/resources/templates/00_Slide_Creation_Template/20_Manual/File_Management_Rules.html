1. 基本ルール

最重要ルール

新しいファイル: 元のファイル名を保持する
例: day1_lecture.html（変更なし）
古いファイル: ファイル名に_YYYYMMDD-HHMIを付加
例: day1_lecture_20250730-0612.html

適用理由

  • • リンク切れの防止
  • • ファイル履歴の保持
  • • 作業効率の向上
  • • チーム間の混乱回避

違反した場合

  • • リンクエラーの発生
  • • ファイル混乱
  • • 作業時間の無駄
  • • 品質低下のリスク

2. ファイル命名規則

新しいファイル

形式: 元のファイル名のまま
例: day1_lecture.html
✓ 既存のリンクが継続して機能
✓ 他のシステムとの互換性維持

古いファイル

形式: ファイル名_YYYYMMDD-HHMI.拡張子
例: day1_lecture_20250730-0612.html
✓ 作成/更新時刻の記録
✓ バージョン履歴の保持

日付時刻フォーマット詳細

YYYY: 4桁年
例: 2025
MM: 2桁月
例: 07（7月）
DD: 2桁日
例: 30
HH: 24時間制時
例: 06（午前6時）
MI: 2桁分
例: 12
区切り文字:
_ - ハイフン

3. バージョン管理

更新プロセス

1
既存ファイル確認

現在のファイルの状態と更新時刻を確認

2
古いファイルリネーム

元ファイル名_YYYYMMDD-HHMI.拡張子 形式でリネーム

3
新ファイル配置

新しいファイルを元のファイル名で配置

4
動作確認

リンクやページ表示の正常動作を確認

履歴保持期間

講義ファイル: 3ヶ月
プロンプトファイル: 6ヶ月
テンプレートファイル: 12ヶ月
設定ファイル: 無期限

削除タイミング

• 保持期間経過後
• プロジェクト完了時
• ストレージ容量不足時
• 明確に不要と判断時
注意: 削除前に関係者への確認必須

5. バックアップルール

自動バックアップ

• ファイル更新時に自動実行
• 古いファイルの日付時刻リネーム
• AIドライブ内での保持
• 履歴の自動管理

定期バックアップ

• 週次: 全講義ファイル
• 月次: プロジェクト全体
• 重要更新時: 即座に実行
• プロジェクト終了時: 最終アーカイブ

復旧手順

1. 問題ファイルの特定
2. 最新バックアップの確認
3. 日付時刻付きファイルから復旧
4. 動作テスト実施

新ファイルのダウンロード・配置

標準手順

1
既存ファイルの確認
2
日付時刻取得（更新時刻基準）
3
古いファイルのリネーム
4
新しいファイルの配置
5
動作確認とテスト

自動化スクリプト例

// ファイル更新の自動化例
function updateFile(oldPath, newFileUrl) {
  const timestamp = getCurrentTimestamp(); // YYYYMMDD-HHMI
  const backupPath = addTimestamp(oldPath, timestamp);
  aidrive_tool.move(oldPath, backupPath);
  aidrive_tool.download_file(newFileUrl, oldPath);
}

8. トラブルシューティング

よくある問題と解決方法

問題: ファイル名の重複エラー
原因: 同名ファイルが既に存在
解決: 既存ファイルの確認後、適切にリネーム
問題: リンク切れの発生
原因: ファイル名の不適切な更新
解決: 新ファイルは元の名前を維持、リンク確認
問題: バックアップファイルの混乱
原因: 日付時刻フォーマットの不統一
解決: YYYYMMDD-HHMI形式を厳密に適用

緊急対応手順

1. 問題の特定と影響範囲確認
2. 最新バックアップファイルの確認
3. 日付時刻付きファイルから復旧
4. リンクとページ表示の確認
5. 関係者への報告と記録

サポート連絡先

技術サポート: プロジェクトマネージャー
システム障害: システム管理者
ファイル復旧: バックアップ担当者
緊急時: チームリーダー

まとめ

重要ポイント

  • 新しいファイルは元の名前を保持
  • 古いファイルは日付時刻付きでリネーム
  • YYYYMMDD-HHMI形式を厳密に適用
  • リンク切れの防止を最優先

継続的改善

  • ルールの定期的な見直し
  • チーム間での共有と徹底
  • 自動化ツールの活用推進
  • トラブル事例の共有と対策

このルールを遵守することで、プロジェクトの品質と効率性を向上させましょう。
