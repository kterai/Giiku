### スライド5：カプセル化の概念 (10:15-11:45)
カプセル化の基本概念と重要性について説明:
- カプセル化の定義：データと操作をひとつのユニットにまとめ、データへの直接アクセスを制限する
- カプセル化の目的：データの整合性保護、実装詳細の隠蔽、変更に強い設計の実現
- カプセル化の実装方法：privateフィールドとpublicメソッド（アクセサ）の組み合わせ
- カプセル化の利点：データ検証の一元化、内部実装の変更が外部に影響しない

### スライド7：getter/setterメソッド (10:15-11:45 続き)

```java
public class Person {
    // privateフィールド（カプセル化）
    private String name;
    private int age;
    
    // getter - 読み取りアクセスを提供
    public String getName() {
        return name;
    }
    
    // setter - バリデーション付き書き込みアクセス
    public void setAge(int age) {
        if (age >= 0 && age <= 120) {
            this.age = age;
        } else {
            throw new IllegalArgumentException("年齢は0～120の範囲である必要があります");
        }
    }
}
```

### スライド10：継承の基本概念 (12:45-14:15)
継承の基本と利点について説明:
- 継承の定義：既存のクラス（スーパークラス）から特性を受け継ぎ、新しいクラス（サブクラス）を作る仕組み
- 「is-a」関係の表現：「サブクラスはスーパークラスの一種」
- 継承の利点：コードの再利用、階層的分類、拡張性
- 継承の基本構文：`public class 子クラス extends 親クラス {}`
- Javaの単一継承の特徴（多重継承との違い）

### スライド12：superキーワードの使い方 (12:45-14:15 続き)
superキーワードの様々な使用方法を例示:
```java
public class Animal {
    protected String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public void makeSound() {
        System.out.println("一般的な鳴き声");
    }
}

public class Dog extends Animal {
    private String breed;
    
    public Dog(String name, String breed) {
        super(name); // スーパークラスのコンストラクタ呼び出し
        this.breed = breed;
    }
    
    @Override
    public void makeSound() {
        super.makeSound(); // スーパークラスのメソッド呼び出し
        System.out.println(name + "がワンワン吠えています");
    }
}
```
- スーパークラスのコンストラクタ呼び出し: `super(引数)`
- スーパークラスのメソッド呼び出し: `super.メソッド名(引数)`
- スーパークラスのフィールドアクセス: `super.フィールド名`

### スライド13：メソッドオーバーライド (12:45-14:15 続き)
オーバーライドの概念と規則:
- オーバーライドの定義：サブクラスでスーパークラスのメソッドを再定義すること
- @Overrideアノテーションの意味と重要性
- オーバーライドの規則:
  1. メソッド名、パラメータ、戻り値の型が同じでなければならない
  2. アクセス修飾子は同等以上に緩くできる（制限は強化できない）
  3. スーパークラスのメソッドより多くの例外をスローできない
  4. staticメソッド、privateメソッド、finalメソッドはオーバーライドできない

### スライド14：継承の制限 (12:45-14:15 続き)
継承に関する制限事項:
- finalクラス：継承できないクラス（例：String, Integer など）
- finalメソッド：オーバーライドできないメソッド
- Javaの単一継承：1つのクラスは1つのクラスしか継承できない
- 代替策：インターフェースの実装（複数可能）
- 継承の乱用の危険性と「継承より合成」の原則

### スライド16：カプセル化と継承を組み合わせた実装例 (14:15-15:55 続き)
具体的な実装例:
```java
// 基本口座クラス
public class Account {
    private String accountNumber;
    private String ownerName;
    protected double balance; // サブクラスからのアクセス許可
    
    public Account(String accountNumber, String ownerName, double initialBalance) {
        this.accountNumber = accountNumber;
        this.ownerName = ownerName;
        if (initialBalance >= 0) {
            this.balance = initialBalance;
        } else {
            throw new IllegalArgumentException("初期残高は0以上である必要があります");
        }
    }
    
    public double getBalance() {
        return balance;
    }
    
    public boolean deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            return true;
        }
        return false;
    }
    
    public boolean withdraw(double amount) {
        if (amount > 0 && balance >= amount) {
            balance -= amount;
            return true;
        }
        return false;
    }
}

// 普通預金口座クラス
public class SavingsAccount extends Account {
    private double interestRate; // 金利
    
    public SavingsAccount(String accountNumber, String ownerName, 
                        double initialBalance, double interestRate) {
        super(accountNumber, ownerName, initialBalance);
        this.interestRate = interestRate;
    }
    
    public void addInterest() {
        double interest = balance * interestRate;
        deposit(interest);
    }
}
```
