### スライド7：クラス定義の実例

```java
public class Student {
    // フィールド（状態）
    String name;
    int age;
    int studentId;
    
    // コンストラクタ（初期化）
    public Student(String name, int age, int studentId) {
        this.name = name;
        this.age = age;
        this.studentId = studentId;
    }
    
    // メソッド（振る舞い）
    public void displayInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("学籍番号: " + studentId);
    }
}
```

### スライド14：コンストラクタの例
```java
public class Book {
    String title;
    String author;
    int pages;
    
    // デフォルトコンストラクタ
    public Book() {
        title = "不明";
        author = "不明";
        pages = 0;
    }
    
    // 引数付きコンストラクタ
    public Book(String title, String author, int pages) {
        this.title = title;
        this.author = author;
        this.pages = pages;
    }
    
    // 別のコンストラクタ
    public Book(String title, int pages) {
        this.title = title;
        this.author = "不明";
        this.pages = pages;
    }
}

// 使用例
Book book1 = new Book(); // デフォルトコンストラクタ
Book book2 = new Book("Java入門", "山田太郎", 350); // 引数付き
Book book3 = new Book("SQL基礎", 200); // タイトルとページ数のみ
```

### スライド15：thisキーワード (14:15-15:55 続き)
- thisキーワードの役割
  - 現在のオブジェクト自身への参照
  - ローカル変数とフィールドの区別
  - 他のコンストラクタの呼び出し
  - メソッドチェーンの実現
- thisの使用例と注意点

### スライド16：thisキーワードの使用例
```java
public class Person {
    String name;
    int age;
    
    // 名前のみ指定するコンストラクタ
    public Person(String name) {
        this(name, 0); // 他のコンストラクタを呼び出す
    }
    
    // 名前と年齢を指定するコンストラクタ
    public Person(String name, int age) {
        this.name = name; // thisでフィールドを参照
        this.age = age;
    }
    
    // メソッドチェーン用のメソッド
    public Person setAge(int age) {
        this.age = age;
        return this; // 自分自身を返す
    }
    
    public Person setName(String name) {
        this.name = name;
        return this; // 自分自身を返す
    }
}

// 使用例
Person p1 = new Person("田中");
Person p2 = new Person("山田", 25);
Person p3 = new Person("佐藤").setAge(30); // メソッドチェーン
```

### スライド17：static要素 (14:15-15:55 続き)
- staticキーワードの意味
- staticフィールド（クラス変数）の特徴
  - クラスに属し、全インスタンスで共有
  - クラス名.フィールド名 でアクセス可能
  - オブジェクト生成前でも使用可能
- staticメソッド（クラスメソッド）の特徴
  - インスタンス生成なしで呼び出し可能
  - インスタンス変数に直接アクセス不可
  - thisキーワードが使用不可
- staticの使用場面

### スライド18：static要素の例
```java
public class Counter {
    // インスタンス変数
    private int count;
    
    // クラス変数（static）
    private static int totalCount;
    
    public Counter() {
        this.count = 0;
    }
    
    // インスタンスメソッド
    public void increment() {
        this.count++;
        totalCount++; // クラス変数も更新
    }
    
    public int getCount() {
        return count;
    }
    
    // クラスメソッド（static）
    public static int getTotalCount() {
        return totalCount;
    }
    
    // クラスメソッド（static）- リセット機能
    public static void resetTotalCount() {
        totalCount = 0;
        // count = 0; // エラー：static メソッドからインスタンス変数にアクセス不可
    }
}

// 使用例
Counter c1 = new Counter();
Counter c2 = new Counter();
c1.increment();
c2.increment();
c2.increment();
System.out.println("c1のカウント: " + c1.getCount()); // 1
System.out.println("c2のカウント: " + c2.getCount()); // 2
System.out.println("総カウント: " + Counter.getTotalCount()); // 3
```
