### スライド5：例外の基本概念 (10:15-11:45)
以下の点を説明:
- 例外とは何か - 正常な処理の流れを中断する事象
- 例外処理の目的 - プログラムの堅牢性と回復性の向上
- Javaの例外階層構造
  * Throwable（基底クラス）
  * Error（回復不能な深刻な問題）
  * Exception（プログラムで処理可能な例外）

### スライド6：チェック例外と非チェック例外 (10:15-11:45 続き)
以下を表形式で比較:
- チェック例外（Exception継承、RuntimeException以外）
  * コンパイラによるチェック強制
  * try-catchまたはthrows宣言が必要
  * 例: IOException, SQLException
- 非チェック例外（RuntimeException継承）
  * コンパイラによるチェックなし
  * プログラミングエラーを表すことが多い
  * 例: NullPointerException, ArrayIndexOutOfBoundsException

### スライド7：try-catch-finally構文 (10:15-11:45 続き)
以下のコードと説明:
```java
try {
    // 例外が発生する可能性のあるコード
    FileReader file = new FileReader("sample.txt");
    BufferedReader reader = new BufferedReader(file);
    String line = reader.readLine();
    System.out.println(line);
} catch (FileNotFoundException e) {
    // FileNotFoundExceptionが発生した場合の処理
    System.err.println("ファイルが見つかりません: " + e.getMessage());
} catch (IOException e) {
    // その他のIOExceptionが発生した場合の処理
    System.err.println("入出力エラーが発生しました: " + e.getMessage());
} finally {
    // 例外発生の有無に関わらず実行される処理
    System.out.println("finally ブロックが実行されました");
    // リソースのクローズなど
}
```
- 各ブロックの役割と実行順序を説明
- マルチキャッチブロックの使い方
- finallyブロックの重要性

### スライド10：try-with-resources構文 (12:45-14:15)
以下のコードと説明:
```java
try (
    FileReader file = new FileReader("sample.txt");
    BufferedReader reader = new BufferedReader(file)
) {
    String line = reader.readLine();
    System.out.println(line);
} catch (IOException e) {
    System.err.println("エラーが発生しました: " + e.getMessage());
    e.printStackTrace();
}
// リソースは自動的にクローズされます
```
- Java 7で導入された構文
- AutoCloseableインターフェースの役割
- 従来のtry-finallyとの違いと利点

### スライド11：カスタム例外の作成 (12:45-14:15 続き)
以下のコード例と説明:
```java
// チェック例外の作成例
public class InvalidUserDataException extends Exception {
    private static final long serialVersionUID = 1L;
    
    public InvalidUserDataException() {
        super();
    }
    
    public InvalidUserDataException(String message) {
        super(message);
    }
    
    public InvalidUserDataException(String message, Throwable cause) {
        super(message, cause);
    }
}

// 非チェック例外の作成例
public class BusinessLogicException extends RuntimeException {
    private static final long serialVersionUID = 1L;
    private String errorCode;
    
    public BusinessLogicException(String message) {
        super(message);
    }
    
    public BusinessLogicException(String message, String errorCode) {
        super(message);
        this.errorCode = errorCode;
    }
    
    public String getErrorCode() {
        return errorCode;
    }
}
```
- カスタム例外作成の基本手順
- チェック例外と非チェック例外の選択基準
- カスタム例外に追加情報を含める方法

### スライド14：パッケージの宣言と使用方法 (14:15-15:00 続き)
以下のコード例と説明:
```java
// ファイル先頭にパッケージ宣言を記述
package com.example.myproject.util;

// インポート宣言
import java.util.List;
import java.util.ArrayList;
import static java.lang.Math.PI;  // static import

public class Calculator {
    // クラスの実装
    public double calculateCircleArea(double radius) {
        return PI * radius * radius;
    }
}
```
- package文の記法と配置
- importの種類と使い方
- static importの活用法
