```java
// 従来の方法（ボイラープレートが多い）
public class Point {
    private final int x;
    private final int y;
    
    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
    
    public int getX() { return x; }
    public int getY() { return y; }
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Point point = (Point) o;
        return x == point.x && y == point.y;
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(x, y);
    }
    
    @Override
    public String toString() {
        return "Point[x=" + x + ", y=" + y + "]";
    }
}

// レコードを使った方法（簡潔）
public record Point(int x, int y) {
    // コンパクト！すべてが自動生成される
    
    // コンパクトコンストラクタ（バリデーションなどを追加可能）
    public Point {
        if (x < 0 || y < 0) {
            throw new IllegalArgumentException("座標は正の値である必要があります");
        }
    }
    
    // 追加メソッドも定義可能
    public double distance() {
        return Math.sqrt(x * x + y * y);
    }
}
```
- Java 16で導入された機能
- レコードの特徴と自動生成される要素
- イミュータブルデータの簡潔な表現方法
- コンパクトコンストラクタと追加メソッド

### スライド7：パターンマッチング for instanceof (10:15-11:45 続き)
以下のコード比較と説明:
```java
// 従来の方法
if (obj instanceof String) {
    String s = (String) obj; // キャストが必要
    System.out.println(s.length());
}

// パターンマッチング for instanceof
if (obj instanceof String s) { // 型チェックとキャストを同時に行う
    System.out.println(s.length()); // sはString型として使用可能
}

// シールドクラスとパターンマッチングの組み合わせ
public double calculateArea(Shape shape) {
    if (shape instanceof Circle c) {
        return Math.PI * c.getRadius() * c.getRadius();
    } else if (shape instanceof Rectangle r) {
        return r.getWidth() * r.getHeight();
    } else if (shape instanceof Triangle t) {
        return 0.5 * t.getBase() * t.getHeight();
    } else {
        throw new IllegalArgumentException("未知の図形タイプです");
    }
}
```
- Java 16で導入された機能
- 型チェックとキャストの簡素化
- 条件式内での変数バインディング
- シールドクラスとの組み合わせによる網羅性チェック

### スライド9：スイッチ式 (10:15-11:45 続き)
以下のコード比較と説明:
```java
// 従来のスイッチ文
String dayType;
switch (day) {
    case "MONDAY":
    case "TUESDAY":
    case "WEDNESDAY":
    case "THURSDAY":
    case "FRIDAY":
        dayType = "Weekday";
        break;
    case "SATURDAY":
    case "SUNDAY":
        dayType = "Weekend";
        break;
    default:
        dayType = "Invalid day";
}

// 新しいスイッチ式
String dayType = switch (day) {
    case "MONDAY", "TUESDAY", "WEDNESDAY", "THURSDAY", "FRIDAY" -> "Weekday";
    case "SATURDAY", "SUNDAY" -> "Weekend";
    default -> "Invalid day";
};

// yieldを使った複数行の処理
int numLetters = switch (day) {
    case "MONDAY", "FRIDAY", "SUNDAY" -> {
        System.out.println(day + "の処理");
        yield 6;
    }
    case "TUESDAY" -> { yield 7; }
    case "THURSDAY", "SATURDAY" -> { yield 8; }
    case "WEDNESDAY" -> { yield 9; }
    default -> {
        throw new IllegalArgumentException("不正な曜日: " + day);
    }
};
```
- Java 14で導入された機能
- 式としての評価と値の返却
- アロー演算子（->）の使用
- yieldキーワードによる複数行処理
