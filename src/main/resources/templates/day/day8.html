テーマ詳細

ポリモーフィズム（Polymorphism）

ポリモーフィズム（多態性）は、オブジェクト指向プログラミングの重要な原則の1つで、同じインターフェースを持つ異なるオブジェクトが、それぞれ独自の方法で振る舞うことができる性質を指します。

ポリモーフィズムの主な特徴

  • 同じメソッド名で異なる実装が可能
  • 親クラスの参照で子クラスのオブジェクトを扱える（アップキャスト）
  • 実行時に適切なメソッドが動的に決定される（動的バインディング）
  • プログラムの拡張性と柔軟性が向上する
  • コードの結合度を低減させる

ポリモーフィズムの実装例

// 形状の基底クラス public abstract class Shape { // 抽象メソッド - サブクラスでの実装が必須 public abstract double calculateArea(); // 具象メソッド - 共通の振る舞い public void displayInfo() { System.out.println("この図形の面積は " + calculateArea() + " です"); } } // Circleクラス（Shapeのサブクラス） public class Circle extends Shape { private double radius; public Circle(double radius) { this.radius = radius; } @Override public double calculateArea() { return Math.PI * radius * radius; } } // Rectangleクラス（Shapeのサブクラス） public class Rectangle extends Shape { private double width; private double height; public Rectangle(double width, double height) { this.width = width; this.height = height; } @Override public double calculateArea() { return width * height; } } // ポリモーフィズムの活用例 public class Main { public static void main(String[] args) { // サブクラスのインスタンスを親クラスの型で参照（アップキャスト） Shape circle = new Circle(5.0); Shape rectangle = new Rectangle(4.0, 6.0); // 同じメソッド呼び出しだが、実際の実装はオブジェクトによって異なる circle.displayInfo(); // 円の面積計算メソッドが実行される rectangle.displayInfo(); // 長方形の面積計算メソッドが実行される // ポリモーフィズムを配列で活用 Shape[] shapes = { new Circle(3.0), new Rectangle(2.0, 5.0), new Circle(7.0) }; // 各図形の情報を表示（ループでの処理が簡潔になる） for (Shape shape : shapes) { shape.displayInfo(); // 各オブジェクトの適切なメソッドが呼び出される } } }

抽象クラス（Abstract Class）

抽象クラスは、直接インスタンス化できないクラスで、共通の振る舞いの一部を実装し、サブクラスに特定のメソッドの実装を強制するために使用されます。

抽象クラスの特徴

  • abstractキーワードを使って宣言する
  • 抽象メソッド（実装のないメソッド）を含むことができる
  • 具象メソッド（実装のあるメソッド）も含むことができる
  • 直接インスタンス化できない（new演算子で生成不可）
  • サブクラスは抽象メソッドを必ずオーバーライドして実装する必要がある
  • コンストラクタ、フィールド、staticメソッドを持つことができる

抽象クラスの実装例

// 抽象クラスの定義 public abstract class Animal { // フィールド protected String name; protected int age; // コンストラクタ public Animal(String name, int age) { this.name = name; this.age = age; } // 具象メソッド（実装あり） public void sleep() { System.out.println(name + "は眠っています"); } // 抽象メソッド（実装なし）- サブクラスでの実装が必須 public abstract void makeSound(); public abstract void move(); } // 抽象クラスを継承したサブクラス public class Dog extends Animal { private String breed; public Dog(String name, int age, String breed) { super(name, age); // 親クラスのコンストラクタを呼び出し this.breed = breed; } // 抽象メソッドの実装（必須） @Override public void makeSound() { System.out.println(name + "はワンワンと吠えています"); } @Override public void move() { System.out.println(name + "は四本足で走っています"); } // クラス独自のメソッド public void fetch() { System.out.println(breed + "の" + name + "はボールを取ってきました"); } }

インターフェース（Interface）

インターフェースは、クラスが実装すべきメソッドの仕様を定義するもので、多重実装が可能な点が特徴です。Java 8以降では、デフォルトメソッドやstaticメソッドも定義できるようになりました。

インターフェースの特徴

  • interfaceキーワードを使って宣言する
  • メソッドは基本的に抽象メソッド（Java 8以前は実装を持てなかった）
  • フィールドは定数（public static final）のみ定義可能
  • Java 8以降、デフォルトメソッド（default）とstaticメソッドが定義可能
  • 複数のインターフェースを実装できる（多重実装）
  • 実装クラスは全ての抽象メソッドを実装する必要がある

インターフェースの実装例

// インターフェースの定義 public interface Swimmable { // 抽象メソッド（実装なし） void swim(); // デフォルトメソッド（Java 8以降） default void dive() { System.out.println("水に潜っています"); } // staticメソッド（Java 8以降） static boolean isWaterSafe(int temperature) { return temperature >= 10 && temperature <= 40; } } // 別のインターフェース public interface Flyable { void fly(); default void land() { System.out.println("着陸しています"); } } // インターフェースを実装したクラス public class Duck extends Animal implements Swimmable, Flyable { public Duck(String name, int age) { super(name, age); } // Animalクラスの抽象メソッドの実装 @Override public void makeSound() { System.out.println(name + "はガーガーと鳴いています"); } @Override public void move() { System.out.println(name + "は歩いています"); } // Swimmableインターフェースのメソッド実装 @Override public void swim() { System.out.println(name + "は水面を泳いでいます"); } // Flyableインターフェースのメソッド実装 @Override public void fly() { System.out.println(name + "は空を飛んでいます"); } // デフォルトメソッドのオーバーライド（任意） @Override public void dive() { System.out.println(name + "は水中にダイブしました"); } }

抽象クラスとインターフェースの比較

特性 抽象クラス インターフェース
キーワード abstract class interface
多重継承/実装 単一継承のみ 複数実装可能
メンバ変数 あらゆる種類の変数を持てる 定数（public static final）のみ
メソッド 抽象メソッドと具象メソッドの両方 抽象メソッド、デフォルトメソッド、staticメソッド
コンストラクタ 定義可能 定義不可
アクセス修飾子 全て使用可能 メソッドは暗黙的にpublic abstract
用途 is-a関係（共通の基底クラス） can-do関係（機能の定義）

使い分けの指針

  • 抽象クラスを使う場合:
    • 共通の振る舞いを持つクラス階層を設計する場合
    • サブクラスで共通のフィールドやメソッドを持つ場合
    • 非publicなメンバやメソッドが必要な場合
    • インスタンス変数の利用や状態管理が必要な場合
  • インターフェースを使う場合:
    • 異なるクラス階層に共通の機能を追加する場合
    • 複数の型を実装する必要がある場合
    • 振る舞い（機能）のみを定義する場合
    • 完全に抽象化されたAPI定義が必要な場合
