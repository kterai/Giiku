クラスとオブジェクトの基本概念

オブジェクト指向プログラミングは、プログラムを「オブジェクト」という単位で構成する手法です。Javaでは、クラスがオブジェクトの設計図（テンプレート）となります。

クラスとオブジェクトの関係
  • クラス：オブジェクトの設計図、テンプレート
  • オブジェクト：クラスから生成された実体（インスタンス）
  • 一つのクラスから複数のオブジェクトを生成可能
  • 各オブジェクトは独自の状態（フィールド値）を持つ
クラスの基本構造
public class クラス名 {
    // フィールド（状態）
    データ型 フィールド名1;
    データ型 フィールド名2;
    
    // コンストラクタ（初期化）
    public クラス名() {
        // 初期化処理
    }
    
    // メソッド（振る舞い）
    戻り値の型 メソッド名1(引数リスト) {
        // 処理内容
    }
    
    戻り値の型 メソッド名2(引数リスト) {
        // 処理内容
    }
}
クラスとオブジェクトの例
// クラスの定義
public class Student {
    // フィールド
    String name;
    int age;
    int studentId;
    
    // コンストラクタ
    public Student(String name, int age, int studentId) {
        this.name = name;
        this.age = age;
        this.studentId = studentId;
    }
    
    // メソッド
    public void displayInfo() {
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age);
        System.out.println("学籍番号: " + studentId);
    }
}

// クラスの使用例（メインメソッド内）
public static void main(String[] args) {
    // オブジェクトの生成
    Student student1 = new Student("田中太郎", 18, 1001);
    Student student2 = new Student("山田花子", 19, 1002);
    
    // メソッドの呼び出し
    student1.displayInfo();
    student2.displayInfo();
}

コンストラクタとオブジェクトの初期化

コンストラクタはオブジェクト生成時に自動的に呼び出される特殊なメソッドで、オブジェクトの初期化を行います。

コンストラクタの特徴
  • クラス名と同じ名前を持つ
  • 戻り値の宣言を持たない（voidも記述しない）
  • new演算子でオブジェクト生成時に自動的に呼び出される
  • オーバーロード可能（引数の異なる複数のコンストラクタを定義可能）
  • 明示的なコンストラクタ定義がない場合、デフォルトコンストラクタが自動生成される
コンストラクタの例
public class Book {
    String title;
    String author;
    int pages;
    
    // デフォルトコンストラクタ
    public Book() {
        title = "不明";
        author = "不明";
        pages = 0;
    }
    
    // 引数付きコンストラクタ
    public Book(String title, String author, int pages) {
        this.title = title;
        this.author = author;
        this.pages = pages;
    }
    
    // タイトルとページ数のみ指定するコンストラクタ
    public Book(String title, int pages) {
        this.title = title;
        this.author = "不明";
        this.pages = pages;
    }
}

// 使用例
Book book1 = new Book(); // デフォルトコンストラクタ
Book book2 = new Book("Java入門", "山田太郎", 350); // 引数付きコンストラクタ
Book book3 = new Book("SQL基礎", 200); // タイトルとページ数のみ

staticメンバとthisキーワード

staticメンバ（クラス変数・メソッド）とインスタンスメンバの違い、thisキーワードの使用方法について説明します。

staticメンバの特徴
  • クラスに属し、全インスタンスで共有される
  • オブジェクト生成前でもアクセス可能
  • クラス名.メンバ名 でアクセス可能
  • 静的メソッドからは非静的メンバに直接アクセスできない
  • メモリ上に1つだけ存在する
thisキーワードの役割
  • 現在のオブジェクト（インスタンス）自身を参照する
  • ローカル変数とインスタンス変数の名前が同じ場合に区別する
  • コンストラクタ内で他のコンストラクタを呼び出す（this()）
  • メソッドチェーンを実装する際に利用（メソッドがthisを返す）
staticとthisの例
public class Counter {
    // インスタンス変数
    private int count;
    
    // クラス変数（static）
    private static int totalCount;
    
    public Counter() {
        this(0); // 他のコンストラクタを呼び出す
    }
    
    public Counter(int initialCount) {
        this.count = initialCount; // thisでインスタンス変数を参照
        totalCount += initialCount;
    }
    
    // インスタンスメソッド
    public void increment() {
        this.count++;
        totalCount++;
    }
    
    // インスタンスメソッド（thisを返す例）
    public Counter incrementAndGet() {
        this.increment();
        return this; // 自分自身を返す（メソッドチェーン用）
    }
    
    public int getCount() {
        return count;
    }
    
    // クラスメソッド（static）
    public static int getTotalCount() {
        return totalCount;
    }
}

// 使用例
Counter c1 = new Counter(5);
Counter c2 = new Counter(10);

c1.increment();
c2.incrementAndGet().incrementAndGet(); // メソッドチェーン

System.out.println("c1カウント: " + c1.getCount()); // 6
System.out.println("c2カウント: " + c2.getCount()); // 12
System.out.println("総カウント: " + Counter.getTotalCount()); // 18
