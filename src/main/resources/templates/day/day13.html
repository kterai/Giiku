テーマ詳細

ラムダ式とは？

ラムダ式は、匿名関数を簡潔に記述するための記法です。Java 8で導入され、関数型プログラミングのスタイルを可能にします。 従来の匿名クラスよりもはるかに読みやすく、簡潔なコードを書くことができます。

従来の匿名クラス vs ラムダ式

従来の匿名クラス

// ボタンクリック処理の従来の書き方 button.addActionListener(new ActionListener() { @Override public void actionPerformed(ActionEvent e) { System.out.println("ボタンがクリックされました"); } }); // Comparatorを使った並び替え Collections.sort(names, new Comparator<String>() { @Override public int compare(String a, String b) { return a.compareTo(b); } });

ラムダ式

// ラムダ式を使った書き方 button.addActionListener(e -> System.out.println("ボタンがクリックされました") ); // ラムダ式を使った並び替え Collections.sort(names, (a, b) -> a.compareTo(b)); // さらにメソッド参照で簡潔に Collections.sort(names, String::compareTo);

ラムダ式の基本記法

基本形式

(引数リスト) -> { 処理内容 }
引数なし
() -> System.out.println("Hello");
引数1つ
x -> x * 2 // または (x) -> x * 2
引数複数
(x, y) -> x + y

関数型インターフェース

ラムダ式は関数型インターフェース（抽象メソッドを1つだけ持つインターフェース）と組み合わせて使用されます。

主要な関数型インターフェース

Function<T, R>
引数Tを受け取り、結果Rを返す
Predicate<T>
引数Tを受け取り、boolean を返す
Consumer<T>
引数Tを受け取り、何も返さない
Supplier<T>
引数なしでTを返す

使用例

// Function: 文字列を整数に変換 Function<String, Integer> converter = s -> Integer.parseInt(s); // Predicate: 偶数かどうかをチェック Predicate<Integer> isEven = n -> n % 2 == 0; // Consumer: コンソール出力 Consumer<String> printer = s -> System.out.println(s); // Supplier: ランダムな数値を生成 Supplier<Double> randomSupplier = () -> Math.random();

メソッド参照

既存のメソッドを参照する場合、ラムダ式をさらに簡潔に書くことができます。

静的メソッド参照

// ラムダ式 Function<String, Integer> parser = s -> Integer.parseInt(s); // メソッド参照 Function<String, Integer> parser = Integer::parseInt;

インスタンスメソッド参照

// ラムダ式 Consumer<String> printer = s -> System.out.println(s); // メソッド参照 Consumer<String> printer = System.out::println;

コンストラクタ参照

// ラムダ式 Supplier<ArrayList<String>> listSupplier = () -> new ArrayList<>(); // コンストラクタ参照 Supplier<ArrayList<String>> listSupplier = ArrayList::new;

Stream APIの基礎

Stream APIは、コレクションのデータを関数型のスタイルで処理するためのAPIです。ラムダ式と組み合わせることで、非常に強力なデータ処理が可能になります。

基本的なStream操作の流れ

1
コレクション
2
Stream生成
3
中間操作
4
終端操作

中間操作（途中の処理）

filter() - 条件によるフィルタリング
map() - データの変換
sorted() - ソート
distinct() - 重複除去

終端操作（最終的な処理）

forEach() - 各要素に処理を適用
collect() - 結果をコレクションに収集
count() - 要素数のカウント
reduce() - 要素を集約

実用的な例

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David"); // 名前が5文字以上のものを大文字に変換して出力 names.stream() .filter(name -> name.length() >= 5) // 中間操作: フィルタ .map(String::toUpperCase) // 中間操作: 大文字変換 .forEach(System.out::println); // 終端操作: 出力 // 結果をListで取得 List<String> longNames = names.stream() .filter(name -> name.length() >= 5) .map(String::toUpperCase) .collect(Collectors.toList());
