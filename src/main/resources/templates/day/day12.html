テーマ詳細

ジェネリクスの基本概念

ジェネリクスとは

ジェネリクス（Generics）は、型をパラメータ化することで、クラス、インターフェース、メソッドを型に対して汎用的に定義する仕組みです。Java 5で導入され、型安全性の向上とコードの再利用性を高めることができます。

ジェネリクスを使わない場合の問題

// ジェネリクス使用前（Java 1.4以前）
List list = new ArrayList();
list.add("Hello");
list.add(123);  // 異なる型も追加可能

String str = (String) list.get(0);  // キャストが必要
String str2 = (String) list.get(1); // 実行時エラー（ClassCastException）

ジェネリクス使用後の改善

// ジェネリクス使用（Java 5以降）
List<String> list = new ArrayList<>();
list.add("Hello");
// list.add(123);  // コンパイルエラー：型が一致しない

String str = list.get(0);  // キャスト不要
// 型安全性が保証され、実行時エラーを防げる

ジェネリッククラス

ジェネリッククラスの定義

// ジェネリッククラスの定義
public class Box<T> {
    private T content;
    
    public Box(T content) {
        this.content = content;
    }
    
    public T getContent() {
        return content;
    }
    
    public void setContent(T content) {
        this.content = content;
    }
    
    // 型Tに依存しないメソッドも定義可能
    public boolean isEmpty() {
        return content == null;
    }
}

// 使用例
Box<String> stringBox = new Box<>("Hello, Generics!");
Box<Integer> intBox = new Box<>(42);
Box<List<String>> listBox = new Box<>(Arrays.asList("A", "B", "C"));

String text = stringBox.getContent();  // キャスト不要
Integer number = intBox.getContent();   // 型安全

複数の型パラメータ

// 複数の型パラメータを持つクラス
public class Pair<T, U> {
    private final T first;
    private final U second;
    
    public Pair(T first, U second) {
        this.first = first;
        this.second = second;
    }
    
    public T getFirst() {
        return first;
    }
    
    public U getSecond() {
        return second;
    }
    
    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}

// 使用例
Pair<String, Integer> nameAge = new Pair<>("Alice", 25);
Pair<Double, Boolean> scorePass = new Pair<>(85.5, true);

System.out.println(nameAge.getFirst());  // "Alice"
System.out.println(nameAge.getSecond()); // 25

ジェネリックメソッド

ジェネリックメソッドの定義

public class Utility {
    // ジェネリックメソッドの定義
    public static <T> void swap(T[] array, int i, int j) {
        T temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
    
    // 戻り値も型パラメータを使用
    public static <T> T getFirst(List<T> list) {
        if (list.isEmpty()) {
            return null;
        }
        return list.get(0);
    }
    
    // 複数の型パラメータを持つメソッド
    public static <T, U> Pair<T, U> makePair(T first, U second) {
        return new Pair<>(first, second);
    }
}

// 使用例
String[] names = {"Alice", "Bob", "Charlie"};
Utility.swap(names, 0, 2);  // Alice と Charlie を交換

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
Integer first = Utility.getFirst(numbers);  // 1

Pair<String, Double> result = Utility.makePair("Score", 95.5);

ワイルドカード

ワイルドカードの種類

  • ? （非境界ワイルドカード）: 任意の型
  • ? extends T （上界ワイルドカード）: Tまたはそのサブタイプ
  • ? super T （下界ワイルドカード）: Tまたはそのスーパータイプ

ワイルドカードの使用例

// 上界ワイルドカード (? extends T)
public static double sumOfList(List<? extends Number> list) {
    double sum = 0.0;
    for (Number n : list) {
        sum += n.doubleValue();
    }
    return sum;
}

// 下界ワイルドカード (? super T)
public static void addNumbers(List<? super Integer> list) {
    for (int i = 1; i <= 10; i++) {
        list.add(i);
    }
}

// 使用例
List<Integer> integers = Arrays.asList(1, 2, 3, 4, 5);
List<Double> doubles = Arrays.asList(1.1, 2.2, 3.3);

// ? extends Number なので、NumberのサブタイプならOK
double sum1 = sumOfList(integers);  // Integer は Number のサブタイプ
double sum2 = sumOfList(doubles);   // Double は Number のサブタイプ

// ? super Integer なので、IntegerのスーパータイプならOK
List<Number> numbers = new ArrayList<>();
List<Object> objects = new ArrayList<>();
addNumbers(numbers);  // Number は Integer のスーパータイプ
addNumbers(objects);  // Object は Integer のスーパータイプ

PECS原則

Producer Extends, Consumer Super

  • Producer Extends: データを取り出す（生産する）場合は ? extends T
  • Consumer Super: データを追加する（消費する）場合は ? super T

境界付き型パラメータ

境界付き型パラメータの使用

// 上界境界付き型パラメータ
public class NumberBox<T extends Number> {
    private T number;
    
    public NumberBox(T number) {
        this.number = number;
    }
    
    public T getNumber() {
        return number;
    }
    
    // Numberのメソッドが使用可能
    public double getDoubleValue() {
        return number.doubleValue();
    }
    
    // 他のNumberBoxと比較
    public boolean isGreaterThan(NumberBox<? extends Number> other) {
        return this.getDoubleValue() > other.getDoubleValue();
    }
}

// 複数の境界を指定
public class SortableList<T extends Comparable<T> & Serializable> {
    private List<T> list = new ArrayList<>();
    
    public void add(T item) {
        list.add(item);
    }
    
    public void sort() {
        Collections.sort(list);  // Comparableなのでソート可能
    }
    
    public List<T> getList() {
        return new ArrayList<>(list);  // Serializableなので安全にコピー
    }
}

// 使用例
NumberBox<Integer> intBox = new NumberBox<>(42);
NumberBox<Double> doubleBox = new NumberBox<>(3.14);

System.out.println(intBox.isGreaterThan(doubleBox));  // false

SortableList<String> stringList = new SortableList<>();
stringList.add("Charlie");
stringList.add("Alice");
stringList.add("Bob");
stringList.sort();  // アルファベット順にソート

型消去（Type Erasure）

型消去とは

Javaのジェネリクスは、コンパイル時のみに存在し、実行時には型情報が消去（erasure）されます。これは後方互換性を保つための仕組みですが、いくつかの制限を生み出します。

型消去による制限と対処法

public class GenericArray<T> {
    private T[] array;
    
    // これはコンパイルエラー：new T[size] はできない
    // public GenericArray(int size) {
    //     array = new T[size];  // Cannot create a generic array of T
    // }
    
    // 回避策1: Object配列を使用してキャスト
    @SuppressWarnings("unchecked")
    public GenericArray(int size) {
        array = (T[]) new Object[size];
    }
    
    // 回避策2: Class<T>を受け取ってリフレクションを使用
    @SuppressWarnings("unchecked")
    public GenericArray(Class<T> type, int size) {
        array = (T[]) Array.newInstance(type, size);
    }
    
    public T get(int index) {
        return array[index];
    }
    
    public void set(int index, T value) {
        array[index] = value;
    }
    
    // 実行時の型チェック
    public boolean isInstance(Object obj, Class<T> type) {
        return type.isInstance(obj);
    }
}

// 実行時型情報を取得する方法
public abstract class TypeReference<T> {
    private final Type type;
    
    protected TypeReference() {
        Type superClass = getClass().getGenericSuperclass();
        this.type = ((ParameterizedType) superClass).getActualTypeArguments()[0];
    }
    
    public Type getType() {
        return type;
    }
}

// 使用例
TypeReference<List<String>> typeRef = new TypeReference<List<String>>() {};
Type type = typeRef.getType();
System.out.println(type);  // java.util.List<java.lang.String>
