MVCアーキテクチャの概念

MVCパターンとは

MVC（Model-View-Controller）は、アプリケーションの構造を3つの役割に分離する設計パターンです。 これにより、コードの保守性と再利用性が向上します。

Model

  • • データとビジネスロジック
  • • データベースアクセス
  • • データの検証・処理

View

  • • ユーザーインターフェース
  • • データの表示
  • • テンプレート・画面

Controller

  • • リクエスト処理
  • • ModelとViewの仲介
  • • 処理の制御・振り分け

MVCの利点

  • 関心の分離：各層が独立した責任を持つ
  • 保守性向上：変更が他の層に与える影響を最小化
  • テストしやすい：各層を独立してテスト可能
  • 再利用性：コンポーネントの再利用が容易

Spring MVCの基礎

Spring MVCの構成要素

@Controller

リクエストを処理するコントローラークラスを定義

@Controller
public class UserController {
    
    @GetMapping("/users")
    public String listUsers(Model model) {
        List<User> users = userService.findAll();
        model.addAttribute("users", users);
        return "users/list"; // users/list.htmlを返す
    }
    
    @GetMapping("/users/new")
    public String newUserForm(Model model) {
        model.addAttribute("user", new User());
        return "users/form";
    }
}

@RequestMapping / @GetMapping / @PostMapping

URLパスとHTTPメソッドをマッピング

@Controller
@RequestMapping("/api/users")
public class UserApiController {

    public String getUser(@PathVariable Long id, Model model) {

Viewにデータを渡すためのModelオブジェクト

@GetMapping("/dashboard")
public String dashboard(Model model, HttpSession session) {

Thymeleafテンプレートエンジン

Thymeleafとは

Thymeleafは、Spring Bootで推奨されるテンプレートエンジンです。 HTMLファイルにThymeleaf属性を追加することで、動的なWebページを作成できます。

基本的な構文

<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title th:text="${title}">Default Title</title>
</head>
<body>
    <h1 th:text="${message}">Hello World</h1>
    
    <!-- 条件分岐 -->
    <div th:if="${user != null}">
        <p>ようこそ、<span th:text="${user.name}"></span>さん！</p>
    </div>
    <div th:unless="${user != null}">
        <p>ログインしてください</p>
    </div>
    
    <!-- ループ処理 -->
    <ul>
        <li th:each="item : ${items}" th:text="${item.name}"></li>
    </ul>
</body>
</html>

レイアウト機能

共通レイアウトを使用してページを統一

<!-- layout.html -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title th:text="${title}">App</title>
    <link th:href="@{/css/bootstrap.min.css}" rel="stylesheet">
</head>
<body>
    <nav class="navbar">
        <a th:href="@{/}">ホーム</a>
        <a th:href="@{/users}">ユーザー一覧</a>
    </nav>
    
    <main th:fragment="content">
        <!-- ここにページ固有のコンテンツが入る -->
    </main>
</body>
</html>

<!-- users/list.html -->
<html xmlns:th="http://www.thymeleaf.org" 
      th:replace="layout :: html">
<main th:fragment="content">
    <h1>ユーザー一覧</h1>
    <table class="table">
        <tr th:each="user : ${users}">
            <td th:text="${user.name}"></td>
            <td th:text="${user.email}"></td>
        </tr>
    </table>
</main>
</html>

フォーム処理と検証

Modelクラスの作成

import javax.validation.constraints.*;

public class User {
    private Long id;
    
    @NotBlank(message = "名前は必須です")
    @Size(min = 2, max = 50, message = "名前は2文字以上50文字以下で入力してください")
    private String name;
    
    @NotBlank(message = "メールアドレスは必須です")
    @Email(message = "正しいメールアドレス形式で入力してください")
    private String email;
    
    @NotNull(message = "年齢は必須です")
    @Min(value = 0, message = "年齢は0以上で入力してください")
    @Max(value = 120, message = "年齢は120以下で入力してください")
    private Integer age;
    
    // コンストラクタ、ゲッター、セッター
}

コントローラーでの処理

@Controller
@RequestMapping("/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping("/new")
    public String newUserForm(Model model) {
        model.addAttribute("user", new User());
        return "users/form";
    }
    
    @PostMapping
    public String createUser(@Valid @ModelAttribute User user,
                           BindingResult result,
                           Model model,
                           RedirectAttributes redirectAttributes) {
        
        // バリデーションエラーがある場合
        if (result.hasErrors()) {
            return "users/form";
        }
        
        try {
            userService.save(user);
            redirectAttributes.addFlashAttribute("successMessage", 
                "ユーザーを正常に作成しました");
            return "redirect:/users";
        } catch (Exception e) {
            model.addAttribute("errorMessage", "作成に失敗しました");
            return "users/form";
        }
    }
}

理解度クイズ

問題1: MVCアーキテクチャについて

Spring MVCにおけるControllerの主な役割は何ですか？

正解: c) HTTPリクエストの処理とModelとViewの仲介
Controllerは、クライアントからのリクエストを受け取り、適切なビジネスロジック（Model）を呼び出し、 結果をView（テンプレート）に渡す役割を担います。

問題2: Thymeleafについて

Thymeleafでオブジェクトのプロパティを表示する正しい構文はどれですか？

正解: b) <span th:text="${user.name}"></span>
th:text 属性は、要素のテキストコンテンツにThymeleaf式の結果を設定します。 これにより、user.nameの値が表示されます。

問題3: バリデーションについて

Spring Bootでフォームのバリデーションエラーがある場合の適切な処理方法は？

正解: b) BindingResultを確認してフォーム画面を再表示
@Valid アノテーションとBindingResultを使用して、バリデーションエラーがある場合は result.hasErrors()で確認し、エラー情報をフォーム画面に返して再表示します。
