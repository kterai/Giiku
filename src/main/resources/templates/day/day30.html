テーマ詳細

探索アルゴリズム

データの集合から特定の値を効率的に見つけ出すためのアルゴリズムです。探索の効率性は、データの性質と選択するアルゴリズムによって大きく異なります。

線形探索（Linear Search）

配列の先頭から順番に目的の値を探すシンプルなアルゴリズムです。

function linearSearch(arr, target) { for (int i = 0; i < arr.length; i++) { if (arr[i] == target) { return i; // 見つかった位置を返す } } return -1; // 見つからない場合 }
特徴：
  • 時間計算量: O(n)
  • 空間計算量: O(1)
  • ソート済みでないデータでも使用可能
  • 実装が簡単

二分探索（Binary Search）

ソート済みの配列で、中央の値と比較して探索範囲を半分ずつ絞り込むアルゴリズムです。

function binarySearch(arr, target) { int left = 0, right = arr.length - 1; while (left <= right) { int mid = (left + right) / 2; if (arr[mid] == target) { return mid; } else if (arr[mid] < target) { left = mid + 1; } else { right = mid - 1; } } return -1; }
特徴：
  • 時間計算量: O(log n)
  • 空間計算量: O(1)
  • データが事前にソートされている必要がある
  • 大量のデータで効率的

ソートアルゴリズム

データを特定の順序（昇順・降順）に並び替えるアルゴリズムです。用途に応じて最適なアルゴリズムを選択することが重要です。

基本ソートアルゴリズム
バブルソート

隣接する要素を比較し、順序が逆なら交換を繰り返す

O(n²)
選択ソート

未ソート部分から最小値を選択し、先頭に移動

O(n²)
挿入ソート

要素を一つずつ取り出し、適切な位置に挿入

O(n²)
高速ソートアルゴリズム
クイックソート

基準値（ピボット）で分割し、再帰的にソート

平均: O(n log n)
最悪: O(n²)
マージソート

分割統治法を使用し、安定ソートを実現

O(n log n)
ヒープソート

ヒープ構造を利用した効率的なソート

O(n log n)

計算量解析（O記法）

アルゴリズムの効率性を評価するための数学的記法です。入力サイズに対する実行時間やメモリ使用量の増加率を表現します。

主要な計算量クラス
  • O(1) - 定数時間: ハッシュテーブルのアクセス
  • O(log n) - 対数時間: 二分探索
  • O(n) - 線形時間: 線形探索
  • O(n log n) - 線形対数時間: 効率的なソート
  • O(n²) - 二次時間: 基本的なソート
