テーマ詳細

コレクションフレームワーク概要

Javaコレクションフレームワークは、データの集合を効率的に格納・操作するための統一されたアーキテクチャです。 JDK 1.2から導入され、オブジェクトの集合を扱うための標準的な方法を提供します。

主要な利点

  • 動的なサイズ変更
  • 型安全性（ジェネリクス）
  • 豊富な操作メソッド
  • 統一されたAPI
  • 高性能な実装
  • 相互運用性

Listインターフェース

Listは順序付きのコレクションで、重複する要素を許可します。要素にはインデックスを使用してアクセスできます。

ArrayList

// ArrayListの使用例 List<String> list = new ArrayList<>(); // 要素の追加 list.add("Java"); list.add("Python"); list.add("JavaScript"); // インデックスアクセス String first = list.get(0); // 拡張for文による反復処理 for (String lang : list) { System.out.println(lang); }
特徴: 配列ベース、インデックスアクセスが高速、末尾への追加が高速

LinkedList

// LinkedListの使用例 List<Integer> linkedList = new LinkedList<>(); // 要素の追加 linkedList.add(10); linkedList.add(20); linkedList.addFirst(5); // 先頭に追加 // 要素の削除 linkedList.removeFirst(); // 先頭削除 linkedList.removeLast(); // 末尾削除 // サイズ確認 int size = linkedList.size();
特徴: ノードベース、挿入・削除が高速、先頭・末尾操作が得意

Setインターフェース

Setは重複を許可しないコレクションです。数学的な集合概念を実装しており、一意性が保証されます。

HashSet

// HashSetの使用例 Set<String> fruits = new HashSet<>(); // 要素の追加 fruits.add("apple"); fruits.add("banana"); fruits.add("apple"); // 重複は無視される // 含有チェック if (fruits.contains("apple")) { System.out.println("りんごが含まれています"); } // サイズは2（重複が除かれるため） System.out.println(fruits.size()); // 2
特徴: ハッシュテーブルベース、高速な検索・追加・削除、順序保証なし

TreeSet

// TreeSetの使用例 Set<Integer> numbers = new TreeSet<>(); // 要素の追加（自動的にソートされる） numbers.add(30); numbers.add(10); numbers.add(20); numbers.add(10); // 重複は無視される // 順序付けられた出力 for (Integer num : numbers) { System.out.println(num); // 10, 20, 30 }
特徴: 赤黒木ベース、要素が自動ソート、範囲検索が可能

Mapインターフェース

Mapはキーと値のペアを格納するコレクションです。キーは一意である必要がありますが、値は重複可能です。

HashMap

// HashMapの使用例 Map<String, Integer> scoreMap = new HashMap<>(); // キーと値のペアを追加 scoreMap.put("Alice", 95); scoreMap.put("Bob", 87); scoreMap.put("Charlie", 92); // 値の取得 Integer aliceScore = scoreMap.get("Alice"); // キーの存在確認 if (scoreMap.containsKey("Bob")) { System.out.println("Bobの成績: " + scoreMap.get("Bob")); } // 全エントリの反復処理 for (Map.Entry<String, Integer> entry : scoreMap.entrySet()) { System.out.println(entry.getKey() + ": " + entry.getValue()); }
特徴: ハッシュテーブルベース、高速なキー検索、順序保証なし

性能特性の比較

コレクション 追加 検索 削除 特徴
ArrayList O(1)* O(n) O(n) インデックスアクセス高速
LinkedList O(1) O(n) O(1) 先頭・末尾操作高速
HashSet O(1) O(1) O(1) 重複排除、高速操作
TreeSet O(log n) O(log n) O(log n) 自動ソート、範囲検索
HashMap O(1) O(1) O(1) キー・値ペア、高速

*末尾への追加の場合。配列サイズ拡張時はO(n)。
