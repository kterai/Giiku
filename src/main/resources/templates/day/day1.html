Day 1: オリエンテーション・Java概要

ITエンジニア育成カリキュラムの第一歩として、Javaプログラミングの基礎を学びます

講義スライドを開く

学習目標

  • ITエンジニア育成カリキュラムの全体像と進め方を理解する
  • Javaプログラミング言語の歴史と特徴を理解する
  • Java開発環境（JDK, IDE）を構築する
  • Hello, World! プログラムを作成・実行できる

達成基準

  • カリキュラムの全体構成と学習目標を説明できる
  • Java SE, EE, MEの違いや、Javaの特徴を3つ以上挙げられる
  • JDK, JRE, JVMの関係性を図示できる
  • 自分のPCでJavaプログラムをコンパイル・実行できる

詳細スケジュール

10:00-10:15 オリエンテーション

ITエンジニア育成カリキュラムの概要説明、3ヶ月間の学習内容と目標、カリキュラムの進め方についての説明。

本日はカリキュラム初日です。学習の進め方や教材の使い方についても説明します。
10:15-11:00 午前講義パート1: Javaの歴史と特徴

Javaプログラミング言語の誕生から現在までの発展の歴史、プラットフォーム独立性、オブジェクト指向、ガベージコレクションなどのJavaの主要な特徴について学習します。

講義資料: 「Javaプログラミング入門」第1章
11:00-11:45 午前演習パート1: Java言語の特徴を理解する

Javaと他のプログラミング言語（C++, Python, JavaScript等）との違いを比較し、それぞれの用途や特性を理解するグループワーク。

演習問題
  1. Javaの「Write Once, Run Anywhere」の意味と仕組みを説明してください。
  2. プログラミング言語を3つ挙げ、Javaとの違いを表にまとめてください。
  3. Java SEとJava EEの違いと使用例を調べてください。
11:45-12:45 昼休憩

休憩時間です。昼食をとり、リフレッシュしましょう。

12:45-13:30 午後講義パート1: Java開発環境の構築

JDK (Java Development Kit)のインストール方法、統合開発環境（Eclipse, IntelliJ IDEA, Visual Studio Code）の紹介と設定方法、環境変数の設定などを学習します。

必要なツール: AdoptOpenJDK, Eclipse IDE
13:30-14:15 午後演習パート1: 開発環境のセットアップ

各自のPCにJDKと統合開発環境（Eclipse）をインストールし、環境変数の設定を行います。トラブルシューティングも含めて実施します。

インストール中に問題が発生した場合は、すぐに質問してください。
14:15-15:00 午後講義パート2: 最初のJavaプログラム

プログラムの基本構造、クラスとメソッド、mainメソッド、コンパイルと実行の流れなど、Javaプログラミングの基礎概念を学習します。

サンプルコード: Hello World
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
15:00-15:10 午後休憩

短い休憩時間です。リフレッシュしましょう。

15:10-15:55 午後演習パート2: Hello, World!プログラミング

統合開発環境を使用して最初のJavaプログラム（Hello, World!）を作成し、コンパイル・実行します。さらに簡単なプログラムの修正を行います。

演習課題
  1. 「Hello, World!」と表示するプログラムを作成して実行する
  2. プログラムを修正して、自分の名前も表示するようにする
  3. 複数行の出力をするプログラムに拡張する
15:55-17:00 理解度チェック

本日学習した内容の理解度を確認するためのクイズと小テストを実施します。Javaの基本概念と環境構築に関する問題が出題されます。

このテストは学習の進捗を確認するためのものであり、評価のためではありません。
17:00-17:45 振り返り・質疑応答

本日の学習内容を振り返り、疑問点や不明点について質問する時間です。翌日の学習内容についての予告も行います。

Javaの概要

Javaプログラミング言語とは
Javaの歴史

Javaは1995年にSun Microsystems（現在はOracleが買収）によって開発されたプログラミング言語です。当初は家電製品用の組み込み言語として開発されましたが、Webアプリケーションの普及とともに広く使われるようになりました。

Javaの特徴
プラットフォーム独立性

「Write Once, Run Anywhere」の原則に基づき、JVMが動作するどのプラットフォームでも同じコードが実行可能。

オブジェクト指向

すべてがオブジェクトとして表現され、カプセル化、継承、ポリモーフィズムをサポート。

自動メモリ管理

ガベージコレクション機能により、メモリリークやダングリングポインタの問題が軽減。

堅牢性と安全性

強い型付け、例外処理、セキュリティマネージャによる保護。

Java実行環境の構成
  • JDK (Java Development Kit): Javaアプリケーションの開発に必要なツールセット
  • JRE (Java Runtime Environment): Javaアプリケーションの実行に必要な環境
  • JVM (Java Virtual Machine): バイトコードを実行する仮想マシン
