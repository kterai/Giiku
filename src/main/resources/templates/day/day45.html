Spring Frameworkの概要

Springとは？

Spring Frameworkは、Java/JVMベースのエンタープライズアプリケーション開発のための 包括的なプログラミング・設定モデルを提供するオープンソースフレームワークです。

複雑な企業システム開発を簡素化し、テスタブルで保守性の高いコードの作成を支援します。

軽量性

POJOベースの開発により、重いフレームワークに依存しない軽量なアプリケーション作成が可能

非侵入性

ビジネスロジックにフレームワーク固有のクラスやインターフェースを強制しない設計

疎結合

依存性注入により、オブジェクト間の結合度を低減し、テストしやすいコード構造を実現

Springの主要モジュール

  • Spring Core: DI/IoC機能
  • Spring MVC: Webアプリケーション開発
  • Spring Data: データアクセス抽象化
  • Spring Security: 認証・認可
  • Spring Boot: 自動設定・スターター
  • Spring Test: テストサポート

依存性注入（DI）と制御の逆転（IoC）

依存性注入とは？

依存性注入（Dependency Injection）は、 オブジェクトが自身の依存オブジェクトを直接作成するのではなく、外部から注入してもらうデザインパターンです。

これにより、オブジェクト同士の結合度が低くなり、テストしやすく、保守性の高いコードが実現できます。

❌ DIを使わない場合

public class OrderService {
    private PaymentService paymentService;
    
    public OrderService() {
        // 依存オブジェクトを直接作成（結合度が高い）
        this.paymentService = new PaymentService();
    }
    
    public void processOrder(Order order) {
        paymentService.processPayment(order);
    }
}

問題点：PaymentServiceの実装が変わると、OrderServiceも変更が必要

✅ DIを使う場合

@Service
public class OrderService {
    private final PaymentService paymentService;
    
    // コンストラクタインジェクション
    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
    
    public void processOrder(Order order) {
        paymentService.processPayment(order);
    }
}

利点：PaymentServiceの実装に依存せず、テスト時にモックを注入可能

SpringのDI注入方法

コンストラクタ注入
@Service
public class UserService {
    private final UserRepository repo;
    
    public UserService(UserRepository repo) {
        this.repo = repo;
    }
}

推奨：必須依存関係に使用

フィールド注入
@Service
public class UserService {
    @Autowired
    private UserRepository repo;
}

簡潔だがテストが困難

セッター注入
@Service
public class UserService {
    private UserRepository repo;
    
    @Autowired
    public void setUserRepository(UserRepository repo) {
        this.repo = repo;
    }
}

オプション依存関係に使用

制御の逆転（IoC）コンテナ

SpringのIoCコンテナ（ApplicationContext）は、Bean定義を読み取り、オブジェクトの生成・管理・依存関係の解決を自動化します。 

@Configuration
public class AppConfig {
    
    @Bean
    public PaymentService paymentService() {
        return new CreditCardPaymentService();
    }
    
    @Bean
    public OrderService orderService(PaymentService paymentService) {
        return new OrderService(paymentService);
    }
}

Spring Boot入門

Spring Bootとは？

Spring Bootは、Springベースのアプリケーションを 迅速に作成するためのフレームワークです。「設定よりも規約」の原則に基づき、 最小限の設定でプロダクション対応のアプリケーションを構築できます。

自動設定

クラスパス上のjarファイルに基づいて、自動的に適切な設定を行う

組み込みサーバー

Tomcat、Jetty、Undertowなどのサーバーを内蔵し、standalone実行が可能

スターター依存関係

目的別にパッケージ化された依存関係で、簡単にライブラリを導入

プロジェクト構造

src/
├── main/
│   ├── java/
│   │   └── com/example/demo/
│   │       ├── DemoApplication.java      # メインクラス

└── test/                                 # テストコード

メインクラスの解説

@SpringBootApplication

@SpringBootApplication アノテーションの構成要素：

  • @ComponentScan: コンポーネントの自動検出を有効化

主要なスターター依存関係

RESTful API開発

RESTコントローラーの作成

@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    // 全ユーザー取得
    @GetMapping
    public List<User> getAllUsers() {
        return userService.findAll();
    }
    
    // ユーザー詳細取得
    @GetMapping("/{id}")
    public ResponseEntity<User> getUserById(@PathVariable Long id) {
        User user = userService.findById(id);
        return user != null ? 
            ResponseEntity.ok(user) : 
            ResponseEntity.notFound().build();
    }
    
    // ユーザー作成
    @PostMapping
    public ResponseEntity<User> createUser(@RequestBody User user) {
        User createdUser = userService.save(user);
        return ResponseEntity.status(HttpStatus.CREATED).body(createdUser);
    }
    
    // ユーザー更新
    @PutMapping("/{id}")
    public ResponseEntity<User> updateUser(@PathVariable Long id, @RequestBody User user) {
        User updatedUser = userService.update(id, user);
        return updatedUser != null ? 
            ResponseEntity.ok(updatedUser) : 
            ResponseEntity.notFound().build();
    }
    
    // ユーザー削除
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        boolean deleted = userService.delete(id);
        return deleted ? 
            ResponseEntity.noContent().build() : 
            ResponseEntity.notFound().build();
    }
}

主要アノテーション

@RestController

RESTfulなWebサービスのコントローラーを示す

@RequestMapping

HTTPリクエストをメソッドにマッピング

@PathVariable

URLパスから変数を抽出

@RequestBody

HTTPリクエストボディをJavaオブジェクトに変換

HTTPメソッドマッピング

@GetMapping

データの取得（READ操作）

@PostMapping

データの作成（CREATE操作）

@PutMapping

データの更新（UPDATE操作）

@DeleteMapping

データの削除（DELETE操作）

サービス層の実装

@Service
public class UserService {
    
    private final UserRepository userRepository;
    
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
    
    public List<User> findAll() {
        return userRepository.findAll();
    }
    
    public User findById(Long id) {
        return userRepository.findById(id).orElse(null);
    }
    
    public User save(User user) {
        return userRepository.save(user);
    }
    
    public User update(Long id, User userDetails) {
        return userRepository.findById(id)
            .map(user -> {
                user.setName(userDetails.getName());
                user.setEmail(userDetails.getEmail());
                return userRepository.save(user);
            })
            .orElse(null);
    }
    
    public boolean delete(Long id) {
        if (userRepository.existsById(id)) {
            userRepository.deleteById(id);
            return true;
        }
        return false;
    }
}

演習2: Book管理APIの作成

書籍管理のためのCRUD操作が可能なREST APIを実装する

要件

  • Book エンティティ（id, title, author, price）
  • 全書籍取得: GET /api/books
  • 書籍詳細取得: GET /api/books/{id}
  • 書籍作成: POST /api/books
  • 書籍更新: PUT /api/books/{id}
  • 書籍削除: DELETE /api/books/{id}
// Book.java
public class Book {
    private Long id;
    private String title;
    private String author;
    private Double price;
    
    // コンストラクタ、ゲッター、セッター省略
}

// BookController.java
@RestController
@RequestMapping("/api/books")
public class BookController {
    
    @Autowired

演習3: 理解度クイズ

問題1: @SpringBootApplicationアノテーションが含む3つのアノテーションは？

問題2: DIの主な利点は？

問題1の答え： A) @Configuration, @EnableAutoConfiguration, @ComponentScan

@SpringBootApplicationは3つのアノテーションを組み合わせたメタアノテーションです。

問題2の答え： B) オブジェクト間の結合度が低くなり、テストしやすくなる

DIにより疎結合な設計が実現でき、モックオブジェクトを使ったテストが容易になります。
