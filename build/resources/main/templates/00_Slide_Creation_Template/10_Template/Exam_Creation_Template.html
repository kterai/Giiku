模擬試験作成プロンプトテンプレート

Google Formでの模擬試験作成、自動採点、スプレッドシート管理のための包括的ガイド

1. 模擬試験の基本設定

テンプレートパラメータ

以下のパラメータを試験要件に合わせて調整してください：

パラメータ 設定値 説明
試験名 Java Silver模擬テスト① Google Form・スプレッドシートのタイトル
問題数 50問 合計出題数
試験時間 75分 制限時間
配点 1問2点（合計100点満点） 自動採点用の配点設定
合格ライン 70%以上（35問以上正解） 合否判定の基準
難易度分布 基本:35問、中級:10問、難問:5問 難易度別の問題数

出題範囲

  • 基本構文と変数の宣言
  • 基本データ型と参照型
  • 演算子と式の評価
  • 制御構文（if-else, switch, for, while）
  • 配列の宣言と操作

2. Google Form作成手順

Google Formの作成と基本設定

  • Googleアカウントにログインし、Google Formsにアクセス
  • 「＋」ボタンをクリックして新規フォームを作成
  • フォームタイトルを「Java Silver模擬テスト①」に設定
  • 説明欄に試験の概要、時間、合格ラインなどを記入

受験者情報の収集設定

  • 「質問を追加」ボタンをクリックし、以下の項目を作成：
  • 名前（短文回答）- 必須項目
  • 学籍番号/社員ID（短文回答）- 必須項目
  • メールアドレス（メールアドレス）- 必須項目

試験時間の制限設定

  • 「設定」（⚙️アイコン）をクリックし、「プレゼンテーション」タブを選択
  • 「タイマーを表示」にチェックを入れ、時間を75分に設定
  • 「時間切れ時の動作」で「回答を自動的に送信する」を選択

セクションの作成（オプション）

内容が多い場合は、カテゴリ別にセクションを分けるとよいでしょう：

  • 「セクションを追加」アイコンをクリック
  • 各セクションに適切なタイトルと説明を追加（例：「基本データ型に関する問題」）
  • 問題は難易度順ではなく、トピック順に配置することを推奨

3. 問題作成テンプレート

問題テンプレート構造

以下のテンプレートを使用して各問題を作成してください。各問題は「選択式」形式を使用します。

# 問題【問題番号】（難易度：基本/中級/難問） 【問題文】 ```java // 必要に応じてコードスニペットを含める ``` * 【選択肢A】 * 【選択肢B】 * 【選択肢C】 * 【選択肢D】 * 【選択肢E】（必要に応じて） 正解：【正解の選択肢】 解説：【解説文】

問題例

問題1（難易度：基本）

次のうち、Javaの基本データ型（プリミティブ型）でないものはどれですか？

  • int
  • boolean
  • String
  • char
  • float

正解：String

解説：Stringはプリミティブ型ではなく、参照型（クラス）です。Javaの基本データ型は、byte, short, int, long, float, double, char, booleanの8つです。

問題2（難易度：中級）

次のコードを実行した場合、出力される結果はどれですか？

public class ArrayTest {
    public static void main(String[] args) {
        int[] array = {1, 2, 3, 4, 5};
        for(int i = 0; i < array.length; i += 2) {
            System.out.print(array[i] + " ");
        }
    }
}
  • 1 2 3 4 5
  • 1 3 5
  • 2 4
  • コンパイルエラーが発生する

正解：1 3 5

解説：for文の増分式が i += 2 となっているため、iは0, 2, 4と変化します。そのため、array[0], array[2], array[4]の値である1, 3, 5が出力されます。

4. 自動採点の設定

Google Formの回答キー設定

  • フォームエディタの右上にある「設定」（⚙️アイコン）をクリック
  • 「テスト」タブを選択し、「テストにする」にチェックを入れる
  • 各問題に対して「回答キー」設定を有効にし、正解の選択肢を選択
  • 各問題に2点を配点（合計100点になるよう設定）

回答後の表示設定

  • 「回答者が表示できる項目」で必要なオプションを選択：
  • 「回答者が間違えた問題」にチェック
  • 「正解」にチェック
  • 「点数」にチェック
  • 必要に応じて「解説」にもチェックを入れる

合格/不合格の判定設定

合格ラインを設定するには：

  • 「合格点」欄に70点と入力（70%の基準）
  • 合格メッセージと不合格メッセージをそれぞれ設定

5. Googleスプレッドシートでの結果管理

スプレッドシートの連携設定

  • フォームエディタの「回答」タブをクリック
  • 右上の緑色のスプレッドシートアイコン「Googleスプレッドシートで回答を表示」をクリック
  • 「新しいスプレッドシートを作成」を選択
  • スプレッドシート名を「Java Silver模擬テスト①_結果」などに設定

スプレッドシートのカスタマイズ

以下の追加カラムを作成して分析を容易にします：

  • 受験日時（自動記録される）
  • 受験者名（自動記録される）
  • メールアドレス（自動記録される）
  • 合計点数（自動記録される）
  • 所要時間 - 計算式で算出（終了時間 - 開始時間）
  • 合否判定 - IF関数で判定（=IF(合計点数>=70, "合格", "不合格")）
  • カテゴリ別正答率 - 各カテゴリの問題ごとに集計

分析用シートの作成

スプレッドシートに以下の分析用シートを追加します：

  • 受験者統計シート - 平均点、最高点、最低点、標準偏差を計算
  • 問題分析シート - 問題ごとの正答率を計算し、難易度評価
  • カテゴリ分析シート - カテゴリ別の平均正答率を集計
  • 進捗管理シート - 受験者の学習進捗状況を視覚化

定期レポート設定（オプション）

Google Apps Scriptを使用して、定期的に結果サマリーをメール送信するよう設定できます：

  • スプレッドシートで「拡張機能」→「Apps Script」を選択
  • 週次/月次レポートを生成して送信するスクリプトを作成
  • トリガーを設定して定期実行するよう設定

6. ベストプラクティスとTips

テスト配布のポイント

  • 短縮URL（Bitlyなど）を使用して共有しやすくする
  • QRコードを生成して印刷物やスライドに掲載
  • 受験可能期間を設定（締め切り日時を明記）
  • 回答制限を1人1回に設定して不正防止

問題作成のコツ

  • 紛らわしい選択肢を含め、理解度を正確に測定
  • 実際のJava Silver試験形式に近づける
  • 似た問題を複数出題し、一貫性をチェック
  • 解説は簡潔かつ明確に、参考資料へのリンクを含める

採点後のフォローアップ

  • 間違えた問題の詳細解説資料を提供
  • カテゴリ別の弱点を特定して復習計画を提案
  • 共通の誤解や間違いについて補足説明を全体に共有
  • 次回の模擬テストの予告と準備アドバイスを提供

不正防止対策

  • 問題と選択肢の順序をランダム化
  • 一度に表示する問題数を制限（戻れないように設定）
  • テスト中の画面共有や録画を禁止するポリシーを明示
  • 同一時間帯に一斉実施する方式も検討
