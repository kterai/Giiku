- メソッドのオーバーライド：サブクラスでの再定義
- instanceof演算子：オブジェクトの型チェック
- ダウンキャスト：親クラス型から子クラス型への変換（注意点を含む）

### スライド7：ポリモーフィズムのコード例 (10:15-11:45 続き)
次のコード例と解説を含める:
```java
// 親クラス
class Animal {
    public void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

// 子クラス
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Dog barks");
    }
    
    public void fetch() {
        System.out.println("Dog fetches");
    }
}

class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Cat meows");
    }
}

// ポリモーフィズムの活用
Animal animal1 = new Dog(); // アップキャスト
Animal animal2 = new Cat(); // アップキャスト

animal1.makeSound(); // "Dog barks" と出力
animal2.makeSound(); // "Cat meows" と出力

// animal1.fetch(); // コンパイルエラー（Animal型にはfetch()メソッドがない）

// instanceofとダウンキャスト
if (animal1 instanceof Dog) {
    Dog dog = (Dog) animal1; // ダウンキャスト
    dog.fetch();             // "Dog fetches" と出力
}
```

### スライド9：抽象クラス (12:45-14:15)
抽象クラスの定義と特徴:
- `abstract` キーワードを使用して宣言
- インスタンス化できない（new演算子で生成不可）
- 抽象メソッド（実装なし）と具象メソッド（実装あり）を含むことができる
- サブクラスは抽象メソッドをすべて実装しなければならない
- コンストラクタ、フィールド変数、通常メソッドを持つことができる
- is-a関係の表現に適している

### スライド10：抽象クラスの実装例 (12:45-14:15 続き)
次のコード例と解説を含める:
```java
// 抽象クラス
public abstract class Shape {
    // フィールド
    protected String color;
    
    // コンストラクタ
    public Shape(String color) {
        this.color = color;
    }
    
    // 具象メソッド
    public void display() {
        System.out.println("This is a " + color + " shape");
    }
    
    // 抽象メソッド
    public abstract double calculateArea();
    public abstract double calculatePerimeter();
}

// 具象サブクラス
public class Circle extends Shape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}
```

### スライド11：インターフェース (12:45-14:15 続き)
インターフェースの定義と特徴:
- `interface` キーワードを使用して宣言
- メソッドは基本的に抽象メソッド（Java 8以前）
- Java 8以降はデフォルトメソッドとstaticメソッドも定義可能
- フィールドは定数（public static final）のみ
- 複数のインターフェースを実装可能（多重実装）
- can-do関係の表現に適している
- 共通の振る舞いを異なるクラス階層に提供する手段

### スライド12：インターフェースの実装例 (12:45-14:15 続き)
次のコード例と解説を含める:
```java
// インターフェース
public interface Drawable {
    // 抽象メソッド（暗黙的にpublic abstract）
    void draw();
    
    // 定数（暗黙的にpublic static final）
    String TOOL = "Pencil";
    
    // デフォルトメソッド（Java 8以降）
    default void displayInfo() {
        System.out.println("Drawing with " + TOOL);
    }
    
    // staticメソッド（Java 8以降）
    static boolean isValidColor(String color) {
        return color != null && !color.isEmpty();
    }
}

// インターフェースの実装
public class Rectangle extends Shape implements Drawable {
    private double width;
    private double height;
    
    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    @Override
    public double calculateArea() {
        return width * height;
    }
    
    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
    
    @Override
    public void draw() {
        System.out.println("Drawing a " + color + " rectangle");
    }
}
```

### スライド13：抽象クラスとインターフェースの比較 (12:45-14:15 続き)
以下の比較表を含める:
| 特性 | 抽象クラス | インターフェース |
| --- | --- | --- |
| 継承/実装 | 単一継承のみ | 複数実装可能 |
| フィールド | あらゆる種類の変数 | 定数（public static final）のみ |
| メソッド | 抽象・具象両方可能 | 抽象、デフォルト、staticメソッド |
| コンストラクタ | 定義可能 | 定義不可 |
| アクセス修飾子 | 全て使用可能 | メソッドは暗黙的にpublic |
| 用途 | is-a関係 | can-do関係 |
| 状態管理 | 状態を持てる | 状態を持てない（基本的に） |

### スライド14：使い分けの指針 (12:45-14:15 続き)
抽象クラスとインターフェースの適切な使い分け:
- **抽象クラスを使う場合**:
  - 密接に関連したクラス間で機能を共有する場合
  - 共通の状態（フィールド）やメソッド実装を提供する場合
  - 非publicメンバやメソッドが必要な場合
  - サブクラスに対する継承の「骨格」を提供する場合
- **インターフェースを使う場合**:
  - 異なるクラス階層に共通の機能を追加する場合
  - 多重継承に近い機能が必要な場合
  - API契約を定義するだけの場合
  - 実装より「できること」を強調したい場合

### スライド16：instanceof演算子とダウンキャスト (14:15-15:55 続き)
型チェックと安全なダウンキャストの方法:
- instanceof演算子の使い方と注意点
- ダウンキャストの基本と危険性
- Java 16以降のパターンマッチングを活用した安全なキャスト
```java
// 従来の方法
if (shape instanceof Circle) {
    Circle circle = (Circle) shape;
    System.out.println("半径: " + circle.getRadius());
}

// Java 16以降のパターンマッチング
if (shape instanceof Circle circle) {
    System.out.println("半径: " + circle.getRadius());
}
```
- ダウンキャストに関する一般的なベストプラクティス
