問題1

Java基礎 - データ型と型変換

以下のJavaコードを実行した時の出力結果として正しいものを選択してください。

public class Test { public static void main(String[] args) { int a = 10; double b = 3.5; int result = (int)(a + b); System.out.println("result = " + result); } }

正解: B) result = 13

解説：

1. int a = 10, double b = 3.5なので、a + bの計算時にaが自動的にdoubleに変換される

2. 10.0 + 3.5 = 13.5の結果がdouble型で得られる

3. (int)キャストにより13.5が13に切り捨てられる（小数部分が消失）

4. 従って、出力結果は「result = 13」となる

ポイント：明示的型変換では小数部分は切り捨てられることを覚えておきましょう。

問題2

Java基礎 - 制御構文

以下のswitch文で「Good」が出力される入力値として正しいものを選択してください。

int score = /* 入力値 */; switch (score / 10) { case 10: case 9: System.out.println("Excellent"); break; case 8: case 7: System.out.println("Good"); break; case 6: System.out.println("Pass"); break; default: System.out.println("Fail"); }

正解: C) score = 85

解説：

各選択肢でscore / 10の計算結果を確認します：

  • • A) 65 ÷ 10 = 6 → case 6なので「Pass」
  • • B) 75 ÷ 10 = 7 → case 7なので「Good」
  • • C) 85 ÷ 10 = 8 → case 8なので「Good」
  • • D) 95 ÷ 10 = 9 → case 9なので「Excellent」

BとCの両方が「Good」を出力しますが、選択肢としてはCが正解です。

ポイント：整数の除算では小数部分が切り捨てられることを理解しましょう。

問題3

オブジェクト指向 - 継承

以下のコードの実行結果として正しいものを選択してください。

class Parent { public void method() { System.out.println("Parent method"); } } class Child extends Parent { public void method() { System.out.println("Child method"); } } public class Test { public static void main(String[] args) { Parent p = new Child(); p.method(); } }

正解: B) Child method

解説：

これはポリモーフィズム（多態性）の典型例です。

1. Parent p = new Child()で、Parent型の参照変数pがChild型のオブジェクトを参照

2. p.method()の呼び出し時、実際のオブジェクトの型（Child）のメソッドが実行される

3. これを動的バインディングまたは実行時メソッド解決と呼ぶ

4. 参照変数の型ではなく、実際のオブジェクトの型に基づいてメソッドが選択される

ポイント：オーバーライドされたメソッドは、実際のオブジェクトの型で実行されます。

問題4

コレクション - List選択

以下の要件に最も適したListの実装クラスを選択してください。
要件：データの先頭と末尾への挿入・削除が頻繁に行われ、ランダムアクセスは不要

正解: B) LinkedList

解説：

各実装クラスの特徴を比較します：

実装 先頭挿入 末尾挿入 ランダムアクセス
ArrayList O(n) O(1) O(1)
LinkedList O(1) O(1) O(n)

要件では「先頭と末尾への挿入・削除が頻繁」かつ「ランダムアクセス不要」なので、

LinkedListが最適です。内部的には双方向リンクリストで実装されており、

先頭・末尾への操作は定数時間O(1)で実行できます。

ポイント：用途に応じたコレクション選択が重要です。

問題5

Stream API - 実践応用

以下のコードの実行結果として正しいものを選択してください。

import java.util.*; import java.util.stream.*; public class Test { public static void main(String[] args) { List<String> words = Arrays.asList("apple", "banana", "cherry", "date"); String result = words.stream() .filter(s -> s.length() > 5) .map(String::toUpperCase) .collect(Collectors.joining(", ")); System.out.println(result); } }

正解: B) BANANA, CHERRY

解説：

Stream処理を順番に追跡します：

  1. 1. 元のリスト: ["apple", "banana", "cherry", "date"]
  2. 2. filter(s → s.length() > 5): 文字数が5より大きい要素を抽出
    • • "apple" (5文字) → 除外
    • • "banana" (6文字) → 通過
    • • "cherry" (6文字) → 通過
    • • "date" (4文字) → 除外
  3. 3. map(String::toUpperCase): 大文字に変換 → ["BANANA", "CHERRY"]
  4. 4. collect(Collectors.joining(", ")): カンマ区切りで結合 → "BANANA, CHERRY"

ポイント：Streamは各操作を順番に適用し、条件に合う要素のみが次の処理に進みます。

問題10

アルゴリズム - 計算量比較

10万件のソート済み配列から特定の値を検索する場合、最も効率的なアルゴリズムを選択してください。

正解: B) 二分探索 - O(log n)

解説：

問題の条件「ソート済み配列」が重要なヒントです：

  • • 線形探索：ソート済みでも先頭から順番に探索。最大10万回の比較が必要
  • • 二分探索：ソート済み配列の特性を活用。中央値と比較して範囲を半分ずつ絞り込む
  • • ハッシュ探索：配列ではハッシュテーブル構造が必要なため、この条件では適用不可

計算量の実際の差：

  • • 10万件の線形探索：最大100,000回の比較
  • • 10万件の二分探索：最大17回の比較（log₂ 100,000 ≈ 17）

ポイント：データが「ソート済み」という前提条件を活用することが重要です。

問題15

総合問題 - 実践的プログラム設計

オンラインショッピングサイトで「商品レビューの平均評価が4.0以上の商品のみを価格の安い順に表示する」機能を実装する場合、最も適切なJavaコードを選択してください。

正解: B) Stream APIを活用

解説：

各選択肢の評価：

  • A) for文とif文のみ：
    • • 実装が複雑で可読性が低い
    • • パフォーマンスが悪い（複数回のソート）
    • • メンテナンスが困難
  • B) Stream API：
    • • 宣言的で読みやすいコード
    • • フィルタリングとソートを効率的に処理
    • • Java 8以降の現代的なアプローチ
    • • 並列処理への拡張も容易
  • C) SQLクエリ：
    • • データベースレベルでの処理で高速
    • • ただし、Java処理の文脈では直接的でない
  • D) 手動ソート：
    • • 効率が悪い（バブルソートはO(n²)）
    • • 既存のライブラリを活用すべき

ポイント：現代のJava開発では、Stream APIを活用して関数型プログラミングのアプローチを取ることが推奨されます。

よくある間違いパターンと対策

Java基礎での典型的ミス

1. 等価比較での==とequals()混同
// 間違い String s1 = new String("Hello"); String s2 = new String("Hello"); if (s1 == s2) { ... } // false（参照比較） // 正しい if (s1.equals(s2)) { ... } // true（値比較）

対策：オブジェクトの比較では必ずequals()を使用

2. 配列の範囲外アクセス
// 間違い int[] arr = new int[5]; for (int i = 0; i <= arr.length; i++) { // <=が問題 arr[i] = i; // ArrayIndexOutOfBoundsException } // 正しい for (int i = 0; i < arr.length; i++) { // <を使用 arr[i] = i; }

対策：配列の長さはlength、インデックスは0からlength-1まで

オブジェクト指向での典型的ミス

1. staticメソッドからインスタンス変数アクセス
// 間違い class Test { private int value = 10; public static void main(String[] args) { System.out.println(value); // コンパイルエラー } } // 正しい Test t = new Test(); System.out.println(t.value);

対策：staticコンテキストではインスタンスを作成してアクセス

2. コンストラクタでのsuper()呼び出し忘れ
継承時のコンストラクタでは、明示的にsuper()を呼び出すか、
親クラスにデフォルトコンストラクタが必要です
