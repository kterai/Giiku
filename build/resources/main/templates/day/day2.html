Day 2: Java基本型・変数・演算子

プログラミングの基本となるJavaの基本データ型、変数の宣言と使用方法、様々な演算子を学ぶ1日

講義スライドを開く

学習目標

  • Javaの基本データ型と参照型の違いを理解する
  • 変数宣言、初期化、スコープの概念を習得する
  • リテラル表記と型変換のルールを理解する
  • 算術演算子、比較演算子、論理演算子を使いこなす
  • 演算子の優先順位を理解して正しく式を評価できる

達成基準

  • 基本データ型と参照型を適切に選択して変数宣言ができる
  • 異なる型間の変換（キャスト）を適切に実装できる
  • 複合代入演算子を効果的に活用できる
  • 演算子の優先順位を考慮した式を記述できる
  • 理解度チェックで80%以上の正答率を達成する

詳細スケジュール

Javaの概要、JVM、JDKなどの前日の内容を簡単に復習し、本日の学習内容と目標を確認します。

Java言語で使用される8種類の基本データ型（プリミティブ型）について学びます。

  • 整数型：byte, short, int, long
  • 浮動小数点型：float, double
  • 文字型：char
  • 論理型：boolean

それぞれの型のサイズ、範囲、デフォルト値について詳しく解説します。

各データ型を使った変数の宣言と初期化を行う演習問題に取り組みます。

// サンプルコード
byte b = 10;
short s = 20;
int i = 30;
long l = 40L;
float f = 12.3f;
double d = 12.3;
char c = 'A';
boolean bool = true;

各型の値の範囲やリテラルの書き方の違いを実際に確認します。

休憩時間です。昼食をとり、リフレッシュしましょう。

変数の宣言、初期化、スコープについて学びます。

  • 変数の命名規則とベストプラクティス
  • ローカル変数とインスタンス変数、クラス変数の違い
  • 変数のスコープとライフサイクル
  • リテラル表記（10進数、16進数、2進数、文字リテラル、文字列リテラル）

変数の宣言と初期化、スコープについての演習問題に取り組みます。

// サンプルコード
int decimalValue = 100; // 10進数リテラル
int hexValue = 0x64; // 16進数リテラル (100)
int binaryValue = 0b1100100; // 2進数リテラル (100)
int withUnderscore = 1_000_000; // 数値リテラルの読みやすさ向上

// スコープの確認
{
  int localVar = 10; // このブロック内でのみ有効
}
// ここではlocalVarは使えない

Javaで使用される各種演算子について学びます。

  • 算術演算子 (+, -, *, /, %)
  • 単項演算子 (+, -, ++, --)
  • 比較演算子 (==, !=, >, <, >=, <=)
  • 論理演算子 (&&, ||, !)
  • ビット演算子 (&, |, ^, ~, <<, >>, >>>)
  • 代入演算子 (=, +=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=, >>>=)
  • 条件演算子 (? :)
  • 演算子の優先順位

短い休憩時間です。リフレッシュしましょう。

各種演算子を使用した演習問題に取り組みます。

// サンプルコード
int a = 10, b = 3;

// 算術演算子
int sum = a + b; // 13
int diff = a - b; // 7
int product = a * b; // 30
int quotient = a / b; // 3 (整数除算)
int remainder = a % b; // 1 (剰余)

// インクリメント・デクリメント
int c = 5;
System.out.println(c++); // 5を出力し、その後cは6になる
System.out.println(++c); // cを7にしてから7を出力

// 条件演算子
int max = (a > b) ? a : b; // aとbの大きい方を選択

テーマ詳細

Javaの基本データ型

Java言語は静的型付け言語であり、変数を使用する前に型を宣言する必要があります。基本データ型（プリミティブ型）には以下の8種類があります：

型 サイズ 範囲 デフォルト値
byte 8ビット -128 ~ 127 0
short 16ビット -32,768 ~ 32,767 0
int 32ビット -2,147,483,648 ~ 2,147,483,647 0
long 64ビット -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807 0L
float 32ビット 約±3.40282347E+38F（6-7桁の精度） 0.0f
double 64ビット 約±1.79769313486231570E+308（15桁の精度） 0.0d
char 16ビット 0 ~ 65,535（Unicode文字） '\u0000'
boolean 1ビット true / false false

これらの基本データ型はスタックに格納され、直接値を保持します。一方、参照型（クラス、インターフェース、配列など）はヒープに格納され、その参照（アドレス）をスタックに保持します。

型変換とキャスト

Javaでは、異なる型間でのデータ変換が頻繁に行われます。変換には、自動的に行われる「暗黙の型変換」と、明示的に指定する「明示的な型変換（キャスト）」の2種類があります。

暗黙の型変換（自動的に行われる変換）

小さい型から大きい型への変換は自動的に行われます：

byte → short → int → long → float → double
char → int → long → float → double
// 例
byte b = 10;
int i = b; // 自動的に型変換される
long l = i; // 自動的に型変換される
float f = l; // 自動的に型変換される

明示的な型変換（キャスト）

大きい型から小さい型への変換は、明示的なキャストが必要です：

// 例
double d = 100.5;
int i = (int)d; // 明示的なキャスト（小数部分は切り捨てられる）
byte b = (byte)i; // 明示的なキャスト（値が範囲外の場合は上位ビットが失われる）
注意: 大きい型から小さい型へのキャストでは、データの損失（精度の損失や値の切り捨て）が発生する可能性があります。

演算子の優先順位

Javaの式では、演算子の優先順位に従って評価が行われます。優先順位が高いものから順に評価されます。

優先順位 演算子 結合性
1 後置 ++, -- 左から右
2 前置 ++, --, +, -, !, ~ 右から左
3 *, /, % 左から右
4 +, - 左から右
5 <<, >>, >>> 左から右
6 <, >, <=, >=, instanceof 左から右
7 ==, != 左から右
8 & 左から右
9 ^ 左から右
10 | 左から右
11 && 左から右
12 || 左から右
13 ? : 右から左
14 =, +=, -=, *=, /=, %=, &=, ^=, |=, <<=, >>=, >>>= 右から左
ヒント: 括弧 () を使用すると、演算子の優先順位を無視して、括弧内の式を先に評価させることができます。
